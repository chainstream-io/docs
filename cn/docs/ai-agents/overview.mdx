ChainStream 向 AI Agent 暴露的 [数据产品](/cn/docs/data-products/overview) 与执行原语和其他客户端完全一致——只是在鉴权、工具发现与按次付费这三个场景上提供了 Agent 真正需要的形态。

<Note>
  **AI Agents 是消费面，不是独立产品。** Agent 能做的所有事情都可以通过 REST、GraphQL、
  WebSocket、Kafka、SDK、CLI 获取。本分区文档讲的是让 LLM 能够顺畅消费能力目录的那层外形。
</Note>

## 两大主要接入面

<CardGroup cols={2}>
  <Card title="MCP Server" icon="robot" href="/cn/docs/ai-agents/mcp-server/introduction">
    托管（也可自托管）的 Model Context Protocol 服务，把 ChainStream 能力目录暴露为类型化工具。
    Claude、Cursor、ChatGPT、Windsurf 等 MCP 兼容客户端开箱即用。
  </Card>
  <Card title="Agent Skills" icon="brain" href="/cn/docs/ai-agents/agent-skills/introduction">
    可安装的 SKILL.md 能力包，教 Agent <em>怎么</em> 用 ChainStream——代币分析、
    Smart Money 跟踪、DeFi 执行等模式，面向 Cursor、Claude Code、Codex。
  </Card>
</CardGroup>

## 为什么这件事对 Agent 重要

传统 LLM 接入区块链有三个老问题：模型有知识截止日期、没有鉴权过的数据源、无法在未预注册账户的情况下付费读取。ChainStream 逐个解决：

<AccordionGroup>
  <Accordion title="实时数据，天生带 schema 好给 tool call 用" icon="bolt">
    每个数据产品都能以结构化 tool call 接入，返回 JSON schema。
    Agent 拿到的是规范化 ID，而不是被截屏出来的文本——后续调用可以直接串起来，
    不必每一步都重解析。
  </Accordion>
  <Accordion title="鉴权贴合 Agent 的运行循环" icon="shield-halved">
    API Key 可用、OAuth 可用；自治 Agent 可用
    [x402 / MPP](/cn/docs/platform/billing-payments/x402-payments)——这是一种按次付费的 HTTP 支付协议，
    Agent 用 USDC 付款、立即调用，无需事先开户。
  </Accordion>
  <Accordion title="可选的写路径带用户级保护" icon="right-left">
    DEX 报价、路由、swap 提交都可用。写操作必须由用户控制的钱包签名——
    Agent 可以构造交易，但不会代表别人的钱默默发送。
  </Accordion>
</AccordionGroup>

## 我该选哪个？

| 情况 | 接入面 |
|:--|:--|
| 你在用 Claude / Cursor / ChatGPT，想接入 ChainStream 工具 | [MCP Server](/cn/docs/ai-agents/mcp-server/introduction) |
| 你想装一个现成的 Agent Skill 做代币研究 | [Agent Skills](/cn/docs/ai-agents/agent-skills/chainstream-data) |
| 你自己用 Python / TS 写 Agent，想直接调 | [SDK](/cn/sdks/overview) |
| 你想按次用 USDC 付费，不注册账号 | [x402 支付](/cn/docs/platform/billing-payments/x402-payments) |
| 你想用 Tempo / Stripe / 稳定币付费 | [MPP 支付](/cn/docs/platform/billing-payments/mpp-payments) |
| 你想给 Agent 一套 shell 原语 | [CLI](/cn/docs/access-methods/cli) |

## 面向 Agent 的 Quickstart

<Steps>
  <Step title="启动 MCP Server">
    ```bash
    CHAINSTREAM_API_KEY=your_api_key npx @chainstream-io/mcp
    ```
  </Step>
  <Step title="把你的 Agent 指过来">
    Claude Desktop、Cursor、ChatGPT 的完整配置片段见
    [MCP Server 安装](/cn/docs/ai-agents/mcp-server/setup)。
  </Step>
  <Step title="提问">
    *"列出 24 小时内 Solana 交易量 Top 10 的代币，并标出有 KYT 告警的那些。"*
    Agent 会自己选工具、串起调用。
  </Step>
</Steps>
