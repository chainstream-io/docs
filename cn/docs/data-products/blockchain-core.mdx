**Blockchain Core** 数据产品是 ChainStream 发布的所有其他数据的底层基础。当规整产品（Tokens、Trades、Wallets）表达不了你要的事实，或者你需要端到端核对 / 审计时，用它。

<Note>
  多数集成应该先从 Tokens / Trades / Wallets 入手——上层产品更便宜、已经去噪。
</Note>

## 包含什么

| 数据集 | 关键字段 |
|:--|:--|
| Blocks | `number`, `hash`, `parentHash`, `timestamp`, `gasUsed`, `baseFeePerGas`, `miner` |
| Transactions | `hash`, `from`, `to`, `value`, `input`, `status`, `gasUsed`, `effectiveGasPrice` |
| Logs / Events | `txHash`, `logIndex`, `address`, `topics[]`, `data`, `removed` |
| Gas 指标 | `blockGasUsage`, `chainGasOracle`（EIP-1559 base / priority 费） |
| 链状态 | `latestBlock`, `finality`, `reorgsLast1h` |

## 覆盖与新鲜度

- 所有支持链的原始数据，尽可能回填到 genesis。
- 延迟：区块与日志在最终确认后 1–2s 内投递。

## 常见用例

- 独立核对某个上层数据点
- 需要原始日志的自定义索引管道
- gas price 预言机与手续费估算服务
- 链健康看板（重组率、叔块率、base fee 漂移）

## 计费

原始数据端点按条数计费。Kafka 主题按投递条数计费，流处理档位有折扣。
