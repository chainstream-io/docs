**Trades** 数据产品是 ChainStream 索引的每一笔 DEX swap 的规整版本——你可以按代币、钱包、池子、路由器查询，不用关心事件来自哪个 AMM。

## 包含什么

| 字段组 | 举例 |
|:--|:--|
| 身份 | `chainId`, `txHash`, `logIndex`, `blockNumber`, `timestamp` |
| 双方 | `makerAddress`, `takerAddress`, `router`, `dex` |
| 经济 | `tokenIn`, `tokenOut`, `amountIn`, `amountOut`, `priceUsd`, `sizeUsd` |
| 上下文 | `poolAddress`, `feeTier`, `slippageBps`, `gasUsedUsd` |
| 分类 | `side` (buy/sell)、`type` (open/close/add/remove)、`isFirstBuy` |

## 覆盖与新鲜度

- **链**：[支持的链](/cn/docs/supported-chains) 里全部。
- **DEX**：Uniswap v2/v3/v4、Raydium、Meteora、PancakeSwap、Aerodrome、Jupiter 路由等——详见 [分链支持表](/cn/api-reference/kafka-topics/evm)。
- **延迟**：区块级——Kafka 在确认后 1–2s 投递，REST 5–10s。

## 常见用例

- 渲染某代币的实时成交 tape
- 计算钱包的已实现 PnL
- 在首笔买入 / 大单上触发告警
- 为交易机器人提供 maker/taker 上下文
- 自定义周期聚合成 OHLC（或直接用 [Candles](/cn/docs/data-products/candles-ohlc)）

## 计费

REST / GraphQL 按 `limit` 分页计费；WebSocket 与 Kafka 按投递条数，流量越大阶梯折扣越多。
