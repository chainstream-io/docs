## 你将学到

完成本教程后，你将能够：

- 使用 API Key 或 OAuth Token 认证 ChainStream
- 调用 API 获取代币元数据
- 使用 CLI 和 SDK 完成相同的查询
- 理解 API 响应结构

<Note>
**预计耗时**：5 分钟
</Note>

---

## 前提条件

- 已注册 ChainStream 账号（[点此注册](https://www.chainstream.io/dashboard)）

---

## 第一步：获取 API Key

1. 登录 [ChainStream Dashboard](https://www.chainstream.io/dashboard)
2. 进入 **Applications**
3. 点击 **Create New App**
4. 复制你的 **API Key**（以 `cs_live_...` 开头）

<Tip>
API Key 是最快的认证方式。如需 OAuth 2.0 JWT 流程，参见[认证](/cn/guides/getting-started/authentication)。
</Tip>

---

## 第二步：发起第一个请求

查询 Solana 上的 SOL 代币元数据。选择你偏好的方式：

<Tabs>
  <Tab title="cURL">
```bash
curl "https://api.chainstream.io/v2/token/sol/So11111111111111111111111111111111111111112" \
  -H "X-API-KEY: your_api_key"
```
  </Tab>
  <Tab title="CLI">
```bash
# 使用 npx 一行命令（无需安装）
npx @chainstream-io/cli token info --chain sol \
  --address So11111111111111111111111111111111111111112
```

CLI 首次运行会提示认证。或者提前设置 key：

```bash
npx @chainstream-io/cli config set --key apiKey --value your_api_key
```
  </Tab>
  <Tab title="SDK (TypeScript)">
```typescript
import { ChainStreamClient } from "@chainstream-io/sdk";

const cs = new ChainStreamClient({ apiKey: "your_api_key" });

const token = await cs.token.getToken(
  "So11111111111111111111111111111111111111112",
  "solana"
);
console.log(token.name, token.symbol, token.decimals);
```
  </Tab>
  <Tab title="MCP (AI Agent)">
将你的 AI Agent 连接到 ChainStream 数据 — 无需编码：

```bash
CHAINSTREAM_API_KEY=your_api_key npx @chainstream-io/mcp
```

然后向你的 AI 提问：*"Solana 上的 SOL 代币是什么？"*

参见 [MCP Server 设置](/cn/guides/ai-infrastructure/mcp-server/setup-guide) 了解 Claude/Cursor 配置。
  </Tab>
  <Tab title="OAuth 2.0 (JWT)">
适用于使用 OAuth 2.0 Client Credentials 流程的应用：

```bash
# 第一步：获取 access token
curl -X POST "https://dex.asia.auth.chainstream.io/oauth/token" \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET",
    "audience": "https://api.dex.chainstream.io",
    "grant_type": "client_credentials"
  }'

# 第二步：使用 token
curl "https://api.chainstream.io/v2/token/sol/So11111111111111111111111111111111111111112" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```
  </Tab>
</Tabs>

<Tip>
将 `YOUR_ACCESS_TOKEN` 替换为你生成的 token。
</Tip>

### 响应示例

```json
{
  "chain": "sol",
  "decimals": 9,
  "name": "Wrapped SOL",
  "symbol": "SOL",
  "address": "So11111111111111111111111111111111111111112",
  "imageUrl": "https://raw.githubusercontent.com/solana-labs/token-list/main/assets/mainnet/So11111111111111111111111111111111111111112/logo.png",
  "tokenCreatedAt": 1710417600000
}
```

---

## 第三步：理解响应

| 字段 | 类型 | 说明 |
|:--|:--|:--|
| `chain` | string | 区块链网络标识 |
| `decimals` | integer | 代币小数位数 |
| `name` | string | 代币名称 |
| `symbol` | string | 代币符号 |
| `address` | string | 代币合约地址 |
| `imageUrl` | string | 代币 Logo URL |
| `tokenCreatedAt` | integer | 代币创建时间戳（毫秒） |

<Note>
完整字段说明参见 [Token Metadata API 参考](/cn/api-reference/endpoint/data/token/v2/token-chain-tokenaddress-metadata-get)。
</Note>

## 常见问题

<AccordionGroup>
  <Accordion title="401 Unauthorized 错误">
    请检查：
    - Access Token 是否正确且未过期？
    - Header 格式是否为 `Authorization: Bearer YOUR_ACCESS_TOKEN` 或 `X-API-KEY: your_key`？
    - 尝试使用 Client ID 和 Client Secret 重新生成 token
  </Accordion>
  <Accordion title="429 Too Many Requests 错误">
    你已达到请求速率限制。免费版限制为每秒 10 次请求。你可以升级套餐以提高限制。
    
    详见 [API 安全](/cn/guides/data-concepts/api-security)。
  </Accordion>
  <Accordion title="如何查询其他链上的代币？">
    修改 URL 中的 chain 参数，例如：
    
    `GET /v2/token/ethereum/{address}` 查询以太坊
    
    `GET /v2/token/bsc/{address}` 查询 BSC
    
    参见[支持的链](/cn/guides/getting-started/supported-chains)获取完整列表。
  </Accordion>
</AccordionGroup>
