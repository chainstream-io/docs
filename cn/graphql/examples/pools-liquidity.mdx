本页涵盖用于 DEX 池与流动性分析的 Cube：

- **DEXPoolEvents** (DWD) — 添加/移除流动性事件及逐事件储备与价格
- **DEXPools** (DWS) — 池子**快照**（当前流动性与价格，定期刷新）
- **TokenSupplyUpdates** (DWD) — 代币供应量、市值与 FDV 更新

<Note>
示例使用 **Solana** 与 **EVM** [链分组](/cn/graphql/schema/chain-groups)：`Solana { ... }` 查询 Solana，`EVM(network: eth | bsc | polygon) { ... }` 查询 EVM 链（含 Polygon）。
</Note>

## How do I get liquidity pools for a token?

查找快照中该代币作为 **token A** 的 DEX 池。若代币只作为 token B 出现，需再用 `tokenB` 查一次（或使用 `where`）；需要**逐事件**的完整交易对上下文时，请用 **DEXPoolEvents**。

```graphql
query {
  Solana {
    DEXPools(
      limit: {count: 10}
      tokenA: {is: "TOKEN_ADDRESS"}
    ) {
      Pool {
        Address
        ProgramAddress
        TokenAAddress
        TokenBAddress
        LiquidityUSD
        PriceAtoB
        PriceBtoA
        LastUpdated
      }
    }
  }
}
```

<Tip>
[在 GraphQL IDE 中打开](https://ide.chainstream.io) — 将上方查询粘贴到 IDE 中即可交互式运行，享受自动补全和 Schema 探索。
</Tip>

<Tip>
将 `TOKEN_ADDRESS` 替换为代币 mint（Solana）或合约地址（EVM）。若代币常作为 quote 侧，请再用 `tokenB: {is: "TOKEN_ADDRESS"}` 查询。
</Tip>

<AccordionGroup>
  <Accordion title="关键字段">
    | Field | Description |
    |:------|:------------|
    | `Pool.Address` | 流动性池 / 市场地址 |
    | `Pool.ProgramAddress` | DEX program 或工厂合约地址 |
    | `Pool.TokenAAddress` | Base 侧代币地址 |
    | `Pool.TokenBAddress` | Quote 侧代币地址 |
    | `Pool.LiquidityUSD` | 池子总流动性（USD，快照） |
    | `Pool.LastUpdated` | 该快照行上次更新时间 |
  </Accordion>
  <Accordion title="自定义技巧">
    - **按流动性过滤**：增加 `where: {Pool: {LiquidityUSD: {gt: 10000}}}` 筛选深度足够的池
    - **指定 DEX**：增加 `where: {Pool: {ProgramAddress: {is: "DEX_PROGRAM_OR_FACTORY_ADDRESS"}}}`
    - **更多池**：提高 `count` 以发现更多相关池
    - **逐事件储备**：用 **DEXPoolEvents**，配合 `tokenAddress` / `poolAddress` 与 `orderBy: Block_Time_DESC` 查看加减仓历史
  </Accordion>
</AccordionGroup>

## How do I get details of a specific pool?

按地址读取该池的**最新快照**。**DEXPools** 每行不是区块时间线；储备**历史序列**请用 **DEXPoolEvents**。

```graphql
query {
  Solana {
    DEXPools(
      limit: {count: 1}
      poolAddress: {is: "POOL_ADDRESS"}
    ) {
      Pool {
        Address
        TokenAAddress
        TokenBAddress
        ProgramAddress
        LiquidityUSD
        PriceAtoB
        PriceBtoA
        LastUpdated
      }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="关键字段">
    | Field | Description |
    |:------|:------------|
    | `Pool.TokenAAddress` / `Pool.TokenBAddress` | 交易对代币地址 |
    | `Pool.LiquidityUSD` | 总流动性（USD，快照） |
    | `Pool.LastUpdated` | 该池快照最近刷新时间 |
  </Accordion>
  <Accordion title="自定义技巧">
    - **储备历史**：在 **DEXPoolEvents** 上使用 `poolAddress: {is: "POOL_ADDRESS"}` 与 `orderBy: Block_Time_DESC`
    - **事件时间范围**：在 **DEXPoolEvents** 上增加 `where: {Block: {Time: {since: "2025-03-01T00:00:00Z"}}}` 限定加减仓时段
  </Accordion>
</AccordionGroup>

<Info>
`poolAddress` 选择器按 `Pool.Address` 过滤，在 **DEXPools**（快照）与 **DEXPoolEvents**（事件）上均可用。
</Info>

## How do I rank pools or read snapshot liquidity?

**DEXPools** (DWS) 存**当前状态**行（通常数分钟级刷新）。适合按 `LiquidityUSD` 排名、按 DEX program 对比，而无需扫原始事件。

```graphql
query {
  Solana {
    DEXPools(
      limit: {count: 20}
      tokenA: {is: "TOKEN_ADDRESS"}
    ) {
      Pool {
        Address
        ProgramAddress
        TokenAAddress
        TokenBAddress
        LiquidityUSD
        LastUpdated
      }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="关键字段">
    | Field | Description |
    |:------|:------------|
    | `Pool.Address` | 池地址 |
    | `Pool.ProgramAddress` | DEX program / 工厂 |
    | `Pool.TokenAAddress` / `Pool.TokenBAddress` | 交易对代币 |
    | `Pool.LiquidityUSD` | 快照流动性（USD） |
    | `Pool.LastUpdated` | 快照新鲜度 |
  </Accordion>
  <Accordion title="自定义技巧">
    - **按流动性排名**：可在客户端按 `LiquidityUSD` 排序，或在 Schema 支持时对快照使用 `orderBy`
    - **活跃池**：优先看 `LastUpdated` 较新的池
    - **对比 DEX**：按 `Pool.ProgramAddress` 分组或过滤
  </Accordion>
</AccordionGroup>

<Tip>
**时间序列**储备与每笔流动性变动用 **DEXPoolEvents** (DWD)；**最新**池状态与发现用 **DEXPools** (DWS)。
</Tip>

## How do I get token supply and market cap?

用 **TokenSupplyUpdates** 获取与供应事件相关的供应量、市值、价格与 FDV。

```graphql
query {
  Solana {
    TokenSupplyUpdates(
      limit: {count: 1}
      tokenAddress: {is: "TOKEN_ADDRESS"}
      orderBy: Block_Time_DESC
    ) {
      TokenSupplyUpdate {
        Currency { MintAddress, Decimals }
        PostBalance
        MarketCapInUSD
        PriceInUSD
        FDVInUSD
        TotalSupply
      }
      Block { Time }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="关键字段">
    | Field | Description |
    |:------|:------------|
    | `TokenSupplyUpdate.Currency.MintAddress` | 代币地址 |
    | `TokenSupplyUpdate.Currency.Decimals` | 代币小数位 |
    | `TokenSupplyUpdate.PostBalance` | 事件后当前供应量 |
    | `TokenSupplyUpdate.MarketCapInUSD` | 市值（USD） |
    | `TokenSupplyUpdate.PriceInUSD` | 供应事件时的代币价格（USD） |
    | `TokenSupplyUpdate.FDVInUSD` | 完全稀释估值 |
    | `TokenSupplyUpdate.TotalSupply` | 代币总供应 |
  </Accordion>
  <Accordion title="自定义技巧">
    - **供应历史**：提高 `count` 查看供应量随时间变化（mint/burn）
    - **市值历史**：查询多条供应更新以绘制市值曲线
    - **与价格结合**：用 `PriceInUSD` 与 `TotalSupply` 交叉校验市值计算
  </Accordion>
</AccordionGroup>

<Info>
本页以 **TokenSupplyUpdates** 提供与供应相关的市值、价格、FDV。更多代币指标与汇总见 [OHLC 与统计](/cn/graphql/examples/ohlc-stats)。
</Info>
