## 什么是查询模板

查询模板是预写好的 GraphQL 查询，覆盖 Solana、Ethereum 与 BSC 上最常见的分析场景。每个模板针对特定 Cube，并预先填好正确的查询结构、字段与筛选参数——替换占位符后即可运行。

模板帮助你：

- **通过示例学习 schema**——在实际查询中看到字段、筛选与排序如何配合
- **快速上手**，无需从零编写查询
- **发现**你可能尚未了解的 Cube

---

## 如何访问模板

<Steps>
  <Step title="打开侧边栏">
    点击侧边栏开关或按 **Ctrl/Cmd+B** 打开左侧面板。
  </Step>
  <Step title="选择「查询」标签">
    侧边栏有两个标签：**Queries**（模板）与 **Saved**（已保存查询）。请选择 **Queries**。
  </Step>
  <Step title="按链与 Cube 浏览">
    模板以树形组织：**链**（Solana / Ethereum / BSC）→ **Cube**（DEXTrades、Transfers 等）→ **模板**。
  </Step>
  <Step title="点击加载">
    点击任意模板即可加载到查询编辑器。占位符会高亮显示，便于替换。
  </Step>
</Steps>

---

## 模板组织方式

模板采用两级层级：

```
Chain (Solana / Ethereum / BSC)
 └── Cube
      └── Template
```

每条链包含同一套 Cube 与模板，并已配置正确的 `network` 值（`sol`、`eth` 或 `bsc`）。

## 占位符

模板使用大写占位符，表示执行前需替换为真实链上地址。

| 占位符 | 说明 | 示例 |
|:--|:--|:--|
| `TOKEN_ADDRESS` | 代币合约 / mint 地址 | `EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v`（Solana 上 USDC） |
| `WALLET_ADDRESS` | 钱包公钥地址 | `5Q544fKrFoe6tsEbD7S8EmxGTJYAKtTVhAW5Q5pge4j1` |
| `POOL_ADDRESS` | DEX 流动性池地址 | `58oQChx4yWmvKdwLLZzBi4ChoCc2fqCUWBkwMihLYQo2` |

<Note>
占位符在查询中以字面字符串出现（例如 `MintAddress: {is: "TOKEN_ADDRESS"}`）。请将整段占位符字符串（含引号）替换为你的真实地址。
</Note>

## 示例：使用模板

以下说明如何在 Solana 上使用 **DEXTrades → Latest Trades** 模板：

<Steps>
  <Step title="打开侧边栏并选择模板">
    进入 **Solana → DEXTrades → Latest Trades** 并点击加载。
  </Step>
  <Step title="检查已加载的查询">
    编辑器会填入：
    ```graphql
    query {
      Solana {
        DEXTrades(
          limit: {count: 10}
          orderBy: {descending: Block_Time}
        ) {
          Block { Time Slot }
          Transaction { Hash }
          Trade {
            Buy {
              Currency { MintAddress Symbol Name }
              Amount
              PriceInUSD
            }
            Sell {
              Currency { MintAddress Symbol Name }
              Amount
            }
            Dex { ProtocolName }
          }
        }
      }
    }
    ```
  </Step>
  <Step title="执行">
    该模板无占位符——会拉取最近 10 笔成交。按 **Ctrl/Cmd+Enter** 运行。
  </Step>
  <Step title="迭代">
    可修改查询以添加筛选、调整 `limit` 或选择不同字段。schema 自动补全有助于发现可用选项。
  </Step>
</Steps>
