## 概述

Stdio 是 ChainStream MCP Server 的**默认传输方式**。它通过标准输入/输出通信，是本地 IDE 与桌面集成的常规选择。

```bash
chainstream-mcp
```

或未全局安装时：

```bash
npx @chainstream-io/mcp
```

无需额外标志；未指定 `--transport` 时会自动使用 stdio。

## Claude Desktop

将下列内容加入 Claude Desktop 的 MCP 配置文件：

<Tabs>
  <Tab title="macOS">
    编辑 `~/Library/Application Support/Claude/claude_desktop_config.json`：

    ```json
    {
      "mcpServers": {
        "chainstream": {
          "command": "npx",
          "args": ["@chainstream-io/mcp"],
          "env": {
            "CHAINSTREAM_API_KEY": "your-key"
          }
        }
      }
    }
    ```
  </Tab>
  <Tab title="Windows">
    编辑 `%APPDATA%\Claude\claude_desktop_config.json`：

    ```json
    {
      "mcpServers": {
        "chainstream": {
          "command": "npx",
          "args": ["@chainstream-io/mcp"],
          "env": {
            "CHAINSTREAM_API_KEY": "your-key"
          }
        }
      }
    }
    ```
  </Tab>
</Tabs>

若已全局安装该包，可直接使用二进制：

```json
{
  "mcpServers": {
    "chainstream": {
      "command": "chainstream-mcp",
      "env": {
        "CHAINSTREAM_API_KEY": "your-key"
      }
    }
  }
}
```

保存配置后重启 Claude Desktop。ChainStream 工具会自动出现在工具列表中。

## Claude Code

使用 CLI 将 MCP 服务器添加到 Claude Code：

```bash
claude mcp add chainstream -- npx @chainstream-io/mcp
```

在启动 Claude Code 前将 API 密钥设为环境变量：

```bash
export CHAINSTREAM_API_KEY=your-key
```

也可将服务器配置写入项目的 `.mcp.json`：

```json
{
  "mcpServers": {
    "chainstream": {
      "command": "npx",
      "args": ["@chainstream-io/mcp"],
      "env": {
        "CHAINSTREAM_API_KEY": "your-key"
      }
    }
  }
}
```

## Cursor

打开 Cursor 设置，进入 **Features > MCP Servers**。点击 **Add new MCP server**，填写如下：

| 字段 | 值 |
|-------|-------|
| Name | `chainstream` |
| Type | `stdio` |
| Command | `npx @chainstream-io/mcp` |

然后在 Cursor 的环境配置中设置 API 密钥：

```
CHAINSTREAM_API_KEY=your-key
```

也可在项目级 `.cursor/mcp.json` 中添加：

```json
{
  "mcpServers": {
    "chainstream": {
      "command": "npx",
      "args": ["@chainstream-io/mcp"],
      "env": {
        "CHAINSTREAM_API_KEY": "your-key"
      }
    }
  }
}
```

<Tip>
  Cursor 使用与 Claude Desktop 相同的 MCP 配置格式。若你已有可用的 Claude Desktop 配置，可直接复用相同的 `mcpServers` 块。
</Tip>

## 故障排除

<AccordionGroup>
  <Accordion title="工具列表中看不到服务器">
    确认 `CHAINSTREAM_API_KEY` 已在 `env` 中正确设置。若密钥缺失或无效，工具调用会因 API 错误失败。可在终端手动运行命令进行测试：

    ```bash
    CHAINSTREAM_API_KEY=your-key npx @chainstream-io/mcp
    ```
  </Accordion>
  <Accordion title="npx 启动较慢">
    首次使用 `npx` 会下载包，可能需要数秒。执行 `npm install -g @chainstream-io/mcp` 全局安装可加快启动，然后将命令改为 `chainstream-mcp`。
  </Accordion>
  <Accordion title="macOS 上出现权限错误">
    若全局 npm 安装遇到权限问题，可使用 `npx`，或将 npm 配置为用户级前缀：

    ```bash
    mkdir -p ~/.npm-global
    npm config set prefix '~/.npm-global'
    export PATH=~/.npm-global/bin:$PATH
    ```
  </Accordion>
</AccordionGroup>
