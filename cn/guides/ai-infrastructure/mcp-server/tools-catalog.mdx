ChainStream MCP Server 支持所有 API Reference 中定义的接口，AI 可以直接调用这些工具获取链上数据。

<Note>
MCP 工具的参数和返回值与 REST API 完全一致，详细字段说明请参考 [API Reference](/cn/api-reference/authentication/authenticate)。
</Note>

## Token 工具

### token_search

搜索代币。

```
搜索 PEPE 相关的代币
```

### token_get

获取代币详细信息。

```
查看代币 So11111111111111111111111111111111111111112 的详细信息
```

### token_get_metadata

获取代币元数据（名称、符号、Logo 等）。

```
获取这个代币的 Logo 和社交媒体链接
```

### token_get_stats

获取代币统计数据（交易量、买卖次数等）。

```
查看这个代币过去 24 小时的交易统计
```

### token_get_price

获取代币当前价格。

```
BONK 现在的价格是多少？
```

### token_get_prices

获取代币历史价格。

```
查看 ETH 过去一周的价格走势
```

### token_get_candles

获取代币 K 线数据。

```
获取 SOL 最近 100 根 1 小时 K 线
```

### token_get_holders

获取代币持有者信息。

```
这个代币有多少持有者？
```

### token_get_top_holders

获取代币 Top 持有者列表。

```
查看这个代币的前 10 大持有者
```

### token_get_pools

获取代币相关的流动池。

```
这个代币在哪些 DEX 有流动池？
```

### token_get_security

获取代币安全检查结果。

```
帮我检查这个代币是否安全
```

### token_get_creation

获取代币创建信息。

```
这个代币是什么时候创建的？创建者是谁？
```

### token_get_mint_and_burn

获取代币铸造和销毁记录。

```
查看这个代币的铸造和销毁历史
```

### token_list

获取代币列表（支持分页和筛选）。

```
列出 Solana 上最近创建的代币
```

### token_get_traders_tag

获取代币交易者标签分布。

```
分析这个代币的交易者都是什么类型的
```

### token_get_liquidity_snapshots

获取代币流动性快照。

```
查看这个代币过去一周的流动性变化
```

### token_dev_get_tokens

获取开发者创建的所有代币。

```
查看这个开发者还创建过哪些代币
```

## WebSocket 订阅

MCP 服务也支持 WebSocket 实时订阅，可以获取以下实时数据：

### 订阅类型

| 类型 | 说明 | 示例提问 |
|------|------|----------|
| K线 | 实时价格 K 线 | "订阅这个代币的 1 分钟 K 线" |
| 代币统计 | 实时交易统计 | "实时监控这个代币的交易数据" |
| 新代币 | 新代币创建通知 | "通知我 Solana 上的新代币" |
| 代币元数据 | 新代币元数据 | "获取新代币的详细信息" |
| 钱包余额 | 余额变动 | "监控这个钱包的余额变化" |
| 钱包交易 | 交易通知 | "这个钱包有新交易时通知我" |
| 交易活动 | 实时交易流 | "显示这个代币的实时交易" |
| 流动池 | 池子状态 | "监控这个池子的流动性变化" |
| 排行榜 | 热门代币更新 | "实时更新热门代币排行" |

<Note>
WebSocket 订阅的详细参数和返回格式请参考 [WebSocket API 文档](/cn/api-reference/endpoint/websocket/api)。
</Note>

## 提示词最佳实践

### 明确指定链

```
✅ 好：查询 0x... 在以太坊上的余额
❌ 差：查询 0x... 的余额
```

### 提供完整地址

```
✅ 好：分析 0x1234567890abcdef1234567890abcdef12345678
❌ 差：分析 0x1234...5678
```

### 指定时间范围

```
✅ 好：查看最近 7 天的交易记录
❌ 差：查看最近的交易
```

### 分步骤提问

复杂任务拆分为多个简单问题：

```
✅ 好：
1. 首先查询这个地址的持仓
2. 然后分析它的交易历史
3. 最后判断它的投资风格

❌ 差：
告诉我这个地址的一切信息
```

### 利用上下文追问

```
用户：查询 0x... 的持仓
AI：[返回持仓列表]
用户：这些代币中哪个涨幅最大？
用户：分析持有 PEPE 的原因
```
