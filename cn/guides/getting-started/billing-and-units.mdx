## 概述

ChainStream 采用 **Unit（计费单位）** 模式计费。不同 API 消耗不同数量的 Units，你可以根据业务需求选择合适的套餐。

### 套餐选择指南

<AccordionGroup>

  <Accordion title="Starter — 适合小型项目">
    **适用场景：**
    - 个人开发者的小型 dApp
    - 交易机器人原型
    - 小型行情展示网站

    **典型用量估算：**
    - 行情看板（10 个代币，每分钟刷新）→ 约 43,200 Units/月

## Units 计算

### 什么是 Unit

Unit 是 ChainStream 的计费单位。不同 API 消耗不同数量的 Units，复杂度越高的 API 消耗越多。

## WebSocket 实时订阅计费

WebSocket 实时数据订阅独立计费，按推送数据的字节数计费：

| 计费项 | 费率 | 说明 |
|:--|:--|:--|
| WebSocket 数据 | **0.005 Unit/byte** | 按推送数据字节数计费 |
| WebSocket 连接 | 免费 | 建立连接不计费 |
| 心跳消息 | 免费 | 心跳包不计费 |

### WebSocket 用量估算示例

```
场景：监控 10 个代币的实时K线（1分钟周期）

计算：
- K线消息约 200 bytes/条
- 每个代币每分钟约 1 条K线消息
- 10 代币 × 1 消息/分钟 × 60 分钟 × 24 小时 × 30 天 = 432,000 条消息
- 总数据量：432,000 × 200 = 86,400,000 bytes
- 月消耗：86,400,000 × 0.005 = 432,000 Units

场景：监控 50 个代币统计数据（实时推送，约每秒更新）

计算：
- 代币统计消息约 500 bytes/条
- 50 代币 × 1 消息/秒 × 86,400 秒 × 30 天 = 129,600,000 条消息
- 总数据量：129,600,000 × 500 = 64,800,000,000 bytes (约 64.8 GB)
- 月消耗：64,800,000,000 × 0.005 = 324,000,000 Units (约 324M)
```

<Note>
批量 API 通常有折扣。具体消耗以 API Reference 中的说明为准。WebSocket 消息频率取决于市场活跃度，高波动时消息量会增加。
</Note>

## KYT/KYA 服务计费

<Warning>
KYT/KYA API **不消耗套餐 Units**，而是从独立的 KYT 账户余额扣费（美元计价）。请在 Dashboard → KYT Service 页面进行充值。
</Warning>

| API | 费用 | 端点 |
|:--|:--|:--|
| 充值风险评估 | $0.25/次 | `POST /v1/kyt/transfer` |
| 提现风险评估 | $0.25/次 | `POST /v1/kyt/withdrawal` |
| 注册地址 | $1.25/次 | `POST /v1/kyt/address` |

### 费用估算示例

<Tabs>
  <Tab title="行情展示应用">
    **场景：** 一个 DeFi 行情网站，展示 50 个代币价格，每 10 秒刷新
    ```
    计算：
    - 请求频率：50 代币 × 6 次/分钟 = 300 次/分钟
    - 月请求量：300 × 60 × 24 × 30 = 12,960,000 次
    - 月消耗：12,960,000 × 1 Unit = 12,960,000 Units

    推荐：Enterprise 套餐
    ```
  </Tab>
  <Tab title="交易机器人">
    **场景：** 一个套利机器人，监控 5 个交易对，每秒查询价格
    ```
    计算：
    - 请求频率：5 × 1 次/秒 = 5 次/秒
    - 月请求量：5 × 86400 × 30 = 12,960,000 次
    - 月消耗：12,960,000 × 1 Unit = 12,960,000 Units

    推荐：Enterprise 套餐，或使用 WebSocket 降低消耗

    使用 WebSocket 优化后：
    - 5 交易对 × 0.1 Unit/消息 × 86400 × 30 = 1,296,000 Units
    - 推荐：Pro 套餐 + 超额
    ```
  </Tab>
  <Tab title="交易所风控">
    **场景：** 小型交易所，每天 1000 笔充值需要 KYT 检测
    ```
    计算：
    - 日请求量：1000 笔
    - 月请求量：1000 × 30 = 30,000 次
    - 月消耗：30,000 × 5 Units = 150,000 Units

    推荐：Pro 套餐
    ```

### 额度预警设置

可以设置额度预警，在使用量达到阈值时收到通知：

| 预警级别 | 触发条件 | 通知方式 |
|:--|:--|:--|
| 提醒 | 50% 使用量 | 邮件 |

<Tip>

</Tip>

## 额度用尽处理

### 用量不足时会发生什么

当月额度用尽后，API 将返回 403 错误，无法继续调用。付费套餐用户可手动购买额外用量恢复服务。

| 套餐 | 处理方式 |
|:--|:--|
| Free | API 返回 403 错误，等待下月额度重置 |
| Starter / Pro / Enterprise | API 返回 403 错误，可购买额外用量恢复服务 |
| Custom | 按合同约定处理 |

### 购买额外用量

付费套餐用户可以在 Dashboard 中购买额外用量，价格按照套餐对应的超额费率计算：

| 套餐 | 额外用量价格 |
|:--|:--|
| Starter | $12.9 / 1M Units |
| Pro | $9.9 / 1M Units |
| Enterprise | $6.9 / 1M Units |

<Warning>
额度用尽后服务会立即中断（返回 403 错误）。建议设置 80% 用量预警，提前购买额外用量避免业务受影响。
</Warning>

## 付款方式

### 支持的付款方式

| 方式 | 说明 | 适用套餐 |
|:--|:--|:--|

## 常见问题

<AccordionGroup>
  <Accordion title="如何选择合适的套餐？">
    建议先使用 Free 套餐评估实际用量，然后根据月均消耗选择：
    - &lt; 30K Units：Free
    - 30K - 10M Units：Starter
    - 10M - 50M Units：Pro
    - &gt; 50M Units：Enterprise
  </Accordion>
  <Accordion title="可以随时升级/降级套餐吗？">
    可以。升级立即生效，按比例补差价。降级在下个账单周期生效。
  </Accordion>
  <Accordion title="未使用的额度可以累积吗？">
    不可以。每月额度在月底清零，不累积到下月。
  </Accordion>
  <Accordion title="如何获取发票？">
    在 Dashboard → Billing → Invoices 页面下载历史发票。企业客户可联系销售获取定制发票。
  </Accordion>
</AccordionGroup>
