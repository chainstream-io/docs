## 版本策略

### API 版本

- **主版本** (v1, v2)：包含破坏性变更
- **次版本** (v1.1, v1.2)：新功能，向后兼容
- **补丁** (v1.1.1)：Bug 修复

### SDK 版本

SDK 遵循语义化版本 (SemVer)：
- **Major**：破坏性变更
- **Minor**：新功能
- **Patch**：Bug 修复

### 废弃策略

- 废弃的端点至少保留 6 个月
- 废弃前会在响应头中添加警告
- 重大变更会提前 30 天通知
