本术语表收集了 ChainStream 文档中常用的专业术语和缩写，帮助开发者快速理解 Web3 数据和区块链领域的概念。

## 区块链术语

基础区块链概念和通用术语。

### A

<Accordion title="Address（地址）">
区块链上的唯一标识符，用于发送和接收资产。以太坊地址以 `0x` 开头，长度为 42 个字符。

示例：`0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045`
</Accordion>

<Accordion title="ABI (Application Binary Interface)">
应用二进制接口。智能合约的接口定义，描述了合约的函数和数据结构，用于与合约交互。
</Accordion>

### B

<Accordion title="Block（区块）">
区块链中存储交易数据的基本单位。每个区块包含一组交易、时间戳和前一区块的哈希值。

| 字段 | 说明 |
|------|------|
| Block Number | 区块高度 |
| Block Hash | 区块哈希 |
| Timestamp | 出块时间 |
| Transactions | 包含的交易 |
</Accordion>

<Accordion title="Block Confirmation（区块确认）">
交易被打包进区块后，后续产生的区块数量。确认数越多，交易越不可逆。

| 链 | 建议确认数 |
|----|-----------|
| Ethereum | 12-32 |
| BSC | 15 |
| Solana | 32 |
</Accordion>

<Accordion title="Bridge（跨链桥）">
允许资产和数据在不同区块链之间转移的协议。常见跨链桥包括 LayerZero、Wormhole 等。
</Accordion>

### C

<Accordion title="Chain（链）">
区块链网络。ChainStream 支持多条链，包括 Ethereum、BSC、Polygon、Arbitrum、Solana 等。

链标识示例：
- `ethereum` - 以太坊主网
- `bsc` - BNB Chain
- `arbitrum` - Arbitrum One
- `solana` - Solana
</Accordion>

<Accordion title="Contract Address（合约地址）">
智能合约部署在区块链上的唯一地址。与普通地址（EOA）不同，合约地址由代码控制。
</Accordion>

### E

<Accordion title="EOA (Externally Owned Account)">
外部拥有账户。由私钥控制的普通钱包地址，与智能合约地址相对。
</Accordion>

<Accordion title="ERC-20">
以太坊代币标准。定义了代币的基本功能接口，如 `transfer`、`approve`、`balanceOf` 等。
</Accordion>

<Accordion title="ERC-721">
以太坊 NFT 标准。定义了非同质化代币的接口规范。
</Accordion>

<Accordion title="EVM (Ethereum Virtual Machine)">
以太坊虚拟机。执行智能合约代码的运行环境。Polygon、Arbitrum、Base 等都是 EVM 兼容链。
</Accordion>

### G

<Accordion title="Gas">
执行区块链交易所需支付的手续费单位。Gas 费 = Gas Used × Gas Price。

| 术语 | 说明 |
|------|------|
| Gas Limit | 交易最大 Gas 消耗 |
| Gas Used | 实际消耗的 Gas |
| Gas Price | 每单位 Gas 的价格 |
| Gwei | Gas Price 单位，1 Gwei = 10⁻⁹ ETH |
</Accordion>

### H

<Accordion title="Hash（哈希）">
将任意长度数据转换为固定长度字符串的加密函数。区块链中常用于交易哈希、区块哈希等。

示例：`0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef`
</Accordion>

### L

<Accordion title="Layer 1 (L1)">
第一层区块链。独立运行的主链，如 Ethereum、Solana、Bitcoin。
</Accordion>

<Accordion title="Layer 2 (L2)">
第二层扩展方案。构建在 L1 之上的扩容解决方案，如 Arbitrum、Optimism、Base、zkSync。
</Accordion>

### M

<Accordion title="Mempool（内存池）">
待处理交易的暂存区。交易在被打包进区块前，会在 mempool 中等待。
</Accordion>

### N

<Accordion title="NFT (Non-Fungible Token)">
非同质化代币。具有唯一性的数字资产，常用于数字艺术、游戏道具等。
</Accordion>

<Accordion title="Node（节点）">
运行区块链软件的计算机，参与网络验证和数据存储。
</Accordion>

<Accordion title="Nonce">
交易序号。每个地址发送交易时的递增计数器，防止重放攻击。
</Accordion>

### R

<Accordion title="RPC (Remote Procedure Call)">
远程过程调用。与区块链节点交互的协议，用于发送交易、查询数据等。
</Accordion>

### S

<Accordion title="Smart Contract（智能合约）">
在区块链上自动执行的程序代码。一旦部署，代码逻辑不可更改（除非设计了升级机制）。
</Accordion>

### T

<Accordion title="Token（代币）">
区块链上的数字资产。包括原生代币（如 ETH）和合约代币（如 USDC）。
</Accordion>

<Accordion title="Transaction（交易）">
区块链上的状态变更操作，如转账、合约调用等。

| 字段 | 说明 |
|------|------|
| TxHash | 交易哈希，唯一标识 |
| From | 发送方地址 |
| To | 接收方地址 |
| Value | 转账金额 |
| Input Data | 合约调用数据 |
</Accordion>

<Accordion title="Transaction Hash / TxHash（交易哈希）">
交易的唯一标识符。可用于在区块浏览器上查询交易详情。
</Accordion>

### W

<Accordion title="Wallet（钱包）">
存储和管理加密资产的工具。可以是软件（如 MetaMask）或硬件（如 Ledger）。
</Accordion>

## DeFi 术语

去中心化金融相关概念。

### A

<Accordion title="AMM (Automated Market Maker)">
自动做市商。DEX 中用于提供流动性和定价的算法机制，如 Uniswap 的恒定乘积公式 `x * y = k`。
</Accordion>

<Accordion title="APR (Annual Percentage Rate)">
年化利率。不考虑复利的年化收益百分比。
</Accordion>

<Accordion title="APY (Annual Percentage Yield)">
年化收益率。考虑复利后的年化收益百分比。

计算公式：`APY = (1 + APR/n)^n - 1`，其中 n 为复利次数。
</Accordion>

<Accordion title="ATH (All-Time High)">
历史最高价。代币或资产曾达到的最高价格。
</Accordion>

<Accordion title="ATL (All-Time Low)">
历史最低价。代币或资产曾达到的最低价格。
</Accordion>

### C

<Accordion title="CEX (Centralized Exchange)">
中心化交易所。由中心化机构运营的加密货币交易平台。

示例：Binance、Coinbase、OKX、Bybit
</Accordion>

<Accordion title="Collateral（抵押品）">
借贷协议中用于担保借款的资产。抵押率通常需要超过 100%。
</Accordion>

### D

<Accordion title="DEX (Decentralized Exchange)">
去中心化交易所。无需中心化机构运营的交易平台，通过智能合约自动执行交易。

示例：Uniswap、SushiSwap、Raydium、Jupiter
</Accordion>

<Accordion title="dApp (Decentralized Application)">
去中心化应用。运行在区块链上的应用程序。
</Accordion>

<Accordion title="DeFi (Decentralized Finance)">
去中心化金融。基于区块链的金融服务和应用，包括交易、借贷、衍生品等。
</Accordion>

### F

<Accordion title="Farming / Yield Farming（流动性挖矿）">
通过向 DeFi 协议提供流动性来获取代币奖励的行为。
</Accordion>

<Accordion title="FDV (Fully Diluted Valuation)">
完全稀释估值。假设所有代币都已发行的市场总价值。

计算公式：`FDV = 代币价格 × 总供应量`
</Accordion>

<Accordion title="Flash Loan（闪电贷）">
无需抵押的即时借贷，必须在同一区块/交易内还款。常用于套利、清算等场景。
</Accordion>

### H

<Accordion title="Health Factor（健康因子）">
借贷协议中衡量头寸安全性的指标。当健康因子 &lt; 1 时，头寸可能被清算。
</Accordion>

### I

<Accordion title="Impermanent Loss（无常损失）">
流动性提供者因价格波动而相对于简单持有资产产生的损失。

| 价格变化 | 无常损失 |
|---------|---------|
| ±25% | -0.6% |
| ±50% | -2.0% |
| ±100% | -5.7% |
</Accordion>

### L

<Accordion title="Liquidation（清算）">
当借贷头寸的抵押率低于阈值时，协议自动出售抵押品偿还债务的过程。
</Accordion>

<Accordion title="Liquidity（流动性）">
市场中可供交易的资产数量。高流动性意味着大额交易不会显著影响价格。
</Accordion>

<Accordion title="Liquidity Pool（流动性池）">
DEX 中存放交易对资产的智能合约，为交易提供流动性。
</Accordion>

<Accordion title="LP (Liquidity Provider)">
流动性提供者。向 DEX 池子提供资产的用户，获取交易手续费作为回报。
</Accordion>

<Accordion title="LP Token">
流动性提供者代币。代表用户在流动性池中份额的凭证，可用于赎回资产或质押挖矿。
</Accordion>

### M

<Accordion title="Market Cap（市值）">
代币的市场总价值。

计算公式：`Market Cap = 代币价格 × 流通供应量`
</Accordion>

<Accordion title="MEV (Maximal Extractable Value)">
最大可提取价值。矿工或验证者通过交易排序、插入或审查获取的额外价值。

常见 MEV 类型：
- Sandwich Attack（三明治攻击）
- Front-running（抢跑）
- Back-running（尾随）
- Liquidation（清算）
</Accordion>

### O

<Accordion title="Oracle（预言机）">
为智能合约提供链外数据的服务，如 Chainlink、Pyth。常用于获取价格数据。
</Accordion>

### S

<Accordion title="Slippage（滑点）">
交易预期价格与实际成交价格的差异。在流动性较低的池子中，滑点可能较大。
</Accordion>

<Accordion title="Staking（质押）">
将代币锁定在协议中以获取奖励或参与治理。
</Accordion>

<Accordion title="Swap（交换）">
在 DEX 上进行的代币兑换操作。
</Accordion>

### T

<Accordion title="TVL (Total Value Locked)">
总锁仓价值。锁定在 DeFi 协议中的资产总价值，是衡量协议规模的重要指标。
</Accordion>

### W

<Accordion title="Whale（巨鲸）">
持有大量加密资产的地址或投资者。通常指持有某代币 1% 以上供应量的地址。
</Accordion>

### Y

<Accordion title="Yield（收益）">
投资 DeFi 协议获得的回报，包括交易费分成、代币奖励、利息等。
</Accordion>

## ChainStream 专有术语

ChainStream 产品和服务相关概念。

### A

<Accordion title="Access Token（访问令牌）">
用于验证 API 请求的 JWT（JSON Web Token）令牌。

**获取流程**：
1. 在 Dashboard 创建 App，获得 Client ID 和 Client Secret
2. 使用 Client ID 和 Client Secret 调用认证服务生成 Access Token
3. 在 API 请求头中使用 `Authorization: Bearer <Access Token>`

**认证域名**：`dex.asia.auth.chainstream.io`

**Scope（权限范围）**：
| Scope | 说明 |
|-------|------|
| `webhook.read` | Webhook API 读取权限 |
| `webhook.write` | Webhook API 写入权限 |
| `kyt.read` | KYT API 读取权限 |
| `kyt.write` | KYT API 写入权限 |

安全提示：
- 不要在客户端代码中暴露 Client Secret
- 不要将凭据提交到版本控制
- 使用环境变量存储敏感信息
- Access Token 有过期时间，需定期刷新

参见 [认证文档](/cn/api-reference/authentication/authenticate)
</Accordion>

### C

<Accordion title="CU (Compute Unit)">
计算单元。ChainStream 用于衡量 API 请求复杂度的计量单位，不同端点消耗不同 CU。
</Accordion>

### K

<Accordion title="KYA (Know Your Address)">
了解你的地址。ChainStream 提供的地址风险评估服务，分析地址的历史行为和关联风险。
</Accordion>

<Accordion title="KYT (Know Your Transaction)">
了解你的交易。ChainStream 提供的交易风控服务，评估交易的资金来源和风险等级。

风险等级：

| 等级 | 说明 |
|------|------|
| Low | 低风险，可自动处理 |
| Medium | 中等风险，建议人工审核 |
| High | 高风险，建议拦截 |
| Severe | 严重风险，必须拦截 |
</Accordion>

### R

<Accordion title="Rate Limit（速率限制）">
API 对请求频率的限制。ChainStream 根据套餐提供不同的速率限制。

响应头：
- `X-RateLimit-Limit` - 限制总量
- `X-RateLimit-Remaining` - 剩余量
- `X-RateLimit-Reset` - 重置时间
</Accordion>

### S

<Accordion title="Smart Money">
聪明钱。ChainStream 识别的在市场中持续获得超额收益的地址。

类型：
- Top Traders（顶级交易者）
- Early Investors（早期投资者）
- Institutional Wallets（机构钱包）
- KOL Wallets（KOL 钱包）

参见 [Smart Money 方法论](/cn/guides/data-concepts/smart-money-methodology)
</Accordion>

<Accordion title="Smart Score">
智能评分。ChainStream 对钱包地址的综合评分，反映其交易能力和历史表现。

| 分数区间 | 等级 |
|---------|------|
| 90-100 | 顶级 Smart Money |
| 75-89 | 优质交易者 |
| 60-74 | 良好交易者 |
| &lt; 60 | 普通交易者 |
</Accordion>

### W

<Accordion title="Webhook">
网络钩子。事件触发时向指定 URL 发送通知的机制。

**支持的事件类型**：
| 事件 | 说明 |
|------|------|
| `sol.token.created` | Solana 链上新代币创建 |
| `sol.token.migrated` | Solana 链上代币毕业（从 Bonding Curve 迁移到 DEX） |

<Note>
更多事件类型正在开发中。
</Note>

**管理端点**：
- `POST /v1/webhook/endpoint` - 创建 Webhook
- `GET /v1/webhook/endpoint` - 列出 Webhook
- `DELETE /v1/webhook/endpoint/{id}` - 删除 Webhook

参见 [Webhook 基础](/cn/playbooks/frameworks/webhook-fundamentals)
</Accordion>

<Accordion title="WebSocket">
一种网络协议，支持实时双向通信。ChainStream 通过 WebSocket 提供实时数据流。

**端点**：`wss://realtime-dex.chainstream.io/connection/websocket`

**连接方式**：
```
wss://realtime-dex.chainstream.io/connection/websocket?token=YOUR_ACCESS_TOKEN
```

**支持的订阅类型**：
- 代币 K 线（`dex-token-candles`）
- 代币统计（`dex-token-stats`）
- 代币持有者（`token-holders`）
- 新代币（`dex-new-token`）
- 钱包余额（`wallet-balance`）
- 钱包交易（`wallet-trade`）

<Note>
使用 SDK 时无需手动处理认证，SDK 已内置支持。
</Note>

参见 [WebSocket API](/cn/api-reference/endpoint/websocket/introduction)
</Accordion>

## AI 术语

AI 和 MCP 相关概念。

### A

<Accordion title="Agent（代理）">
能够自主执行任务的 AI 系统。AI Agent 可以使用工具、访问外部数据、做出决策。
</Accordion>

### C

<Accordion title="Context（上下文）">
AI 模型在生成回答时参考的信息，包括对话历史、系统提示、外部数据等。
</Accordion>

<Accordion title="Context Window（上下文窗口）">
AI 模型一次能处理的最大 Token 数量。更大的上下文窗口允许处理更多信息。
</Accordion>

### L

<Accordion title="LLM (Large Language Model)">
大语言模型。基于大量文本数据训练的 AI 模型，如 Claude、GPT 等。
</Accordion>

### M

<Accordion title="MCP (Model Context Protocol)">
模型上下文协议。Anthropic 开发的开放协议，允许 AI 模型与外部数据源和工具交互。

ChainStream MCP Server 实现了该协议，让 AI 能够直接查询链上数据。

参见 [MCP 介绍](/cn/guides/ai-infrastructure/mcp-server/introduction)
</Accordion>

<Accordion title="MCP Server">
MCP 服务端。实现 MCP 协议、提供工具和数据的服务。ChainStream MCP Server 提供链上数据查询能力。
</Accordion>

<Accordion title="MCP Client">
MCP 客户端。使用 MCP 协议调用工具的应用，如 Claude Desktop、Cursor。
</Accordion>

### P

<Accordion title="Prompt（提示词）">
给 AI 模型的输入文本，引导模型生成期望的输出。
</Accordion>

### T

<Accordion title="TEE (Trusted Execution Environment)">
可信执行环境。提供硬件级别安全保护的计算环境，确保代码和数据的机密性和完整性。
</Accordion>

<Accordion title="Token（词元）">
AI 模型处理文本的基本单位。一个 Token 约等于 4 个英文字符或 1-2 个中文字符。

<Note>
此处 Token 指 AI 领域的词元，与区块链代币是不同概念。
</Note>
</Accordion>

<Accordion title="Tool（工具）">
MCP 中定义的可被 AI 调用的功能。ChainStream 提供的工具包括 `get_token_balance`、`analyze_wallet` 等。
</Accordion>

### X

<Accordion title="x402">
基于 HTTP 402 状态码的支付协议规范，用于实现 AI Agent 的自动小额支付。
</Accordion>
