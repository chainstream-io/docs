## Install

```bash
npm install @chainstream-io/sdk          # TypeScript
pip install chainstream                  # Python
go get github.com/chainstream-io/chainstream-go
cargo add chainstream                    # Rust
```

## First call ‚Äî TypeScript

```ts
import { ChainStream } from "@chainstream-io/sdk";

const cs = new ChainStream({ apiKey: process.env.CHAINSTREAM_API_KEY! });

const results = await cs.token.search({
  chain: "solana",
  keyword: "USDC",
  limit: 5,
});

for (const t of results) {
  console.log(t.symbol, t.priceUsd);
}
```

## Common patterns

- **REST** ‚Äî `cs.token.*`, `cs.wallet.*`, `cs.trade.*`, `cs.ranking.*`, `cs.dex.*` ...
- **WebSocket** ‚Äî `cs.stream.subscribeTokenTrade({ chain, tokenAddress, callback })` (and parallel methods for candles, stats, holders, wallets, rankings, pools)
- **GraphQL** ‚Äî `cs.gql(query, variables)` returns a fully typed result via codegen
- **CLI bridge** ‚Äî `cs.cli("market", "trending", { chain: "sol", duration: "24h" })` runs the same logic as `chainstream market trending --chain sol --duration 24h`

## When to use

- Integrating into an application ‚Äî almost always the right default
- When you want types, retries, error mapping and pagination handled for you
- When you want to swap REST ‚Üî WebSocket for the same resource without rewriting

When **not** to: quick one-offs where `curl` + REST is faster (or CLI).
