## What You'll Learn

After completing this tutorial, you will be able to:

- Authenticate with ChainStream using an API Key or OAuth token
- Call the API to get token metadata
- Use the CLI and SDK for the same query
- Understand the API response structure

<Note>
**Estimated time**: 5 minutes
</Note>

---

## Prerequisites

- A registered ChainStream account ([Register here](https://www.chainstream.io/dashboard))

---

## Step 1: Get Your API Key

1. Log in to [ChainStream Dashboard](https://www.chainstream.io/dashboard)
2. Go to **Applications**
3. Click **Create New App**
4. Copy your **API Key** (starts with `cs_live_...`)

<Tip>
The API Key is the fastest way to authenticate. For OAuth 2.0 JWT flow, see [Authentication](/en/guides/getting-started/authentication).
</Tip>

---

## Step 2: Make Your First Request

Query the SOL token metadata on Solana. Choose your preferred method:

<Tabs>
  <Tab title="cURL">
```bash
curl "https://api.chainstream.io/v2/token/sol/So11111111111111111111111111111111111111112" \
  -H "X-API-KEY: your_api_key"
```
  </Tab>
  <Tab title="CLI">
```bash
# One-liner with npx (no install needed)
npx @chainstream-io/cli token info --chain sol \
  --address So11111111111111111111111111111111111111112
```

The CLI will prompt you to authenticate on first run. Or set your key upfront:

```bash
npx @chainstream-io/cli config set --key apiKey --value your_api_key
```
  </Tab>
  <Tab title="SDK (TypeScript)">
```typescript
import { ChainStreamClient } from "@chainstream-io/sdk";

const cs = new ChainStreamClient({ apiKey: "your_api_key" });

const token = await cs.token.getToken(
  "So11111111111111111111111111111111111111112",
  "solana"
);
console.log(token.name, token.symbol, token.decimals);
```
  </Tab>
  <Tab title="MCP (AI Agent)">
Connect your AI agent to ChainStream data — no code needed:

```bash
CHAINSTREAM_API_KEY=your_api_key npx @chainstream-io/mcp
```

Then ask your AI: *"What is the SOL token on Solana?"*

See [MCP Server Setup](/en/guides/ai-infrastructure/mcp-server/setup-guide) for Claude/Cursor configuration.
  </Tab>
  <Tab title="OAuth 2.0 (JWT)">
For apps using OAuth 2.0 Client Credentials flow:

```bash
# Step 1: Get access token
curl -X POST "https://dex.asia.auth.chainstream.io/oauth/token" \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET",
    "audience": "https://api.dex.chainstream.io",
    "grant_type": "client_credentials"
  }'

# Step 2: Use the token
curl "https://api.chainstream.io/v2/token/sol/So11111111111111111111111111111111111111112" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```
  </Tab>
</Tabs>

<Tip>
Replace `YOUR_ACCESS_TOKEN` with the token you generated in Step 2.
</Tip>

### Response Example

```json
{
  "chain": "sol",
  "decimals": 9,
  "name": "Wrapped SOL",
  "symbol": "SOL",
  "address": "So11111111111111111111111111111111111111112",
  "imageUrl": "https://raw.githubusercontent.com/solana-labs/token-list/main/assets/mainnet/So11111111111111111111111111111111111111112/logo.png",
  "tokenCreatedAt": 1710417600000
}
```

---

## Step 4: Understand the Response

| Field | Type | Description |
|:--|:--|:--|
| `chain` | string | Blockchain network identifier |
| `decimals` | integer | Token decimal places |
| `name` | string | Token name |
| `symbol` | string | Token symbol |
| `address` | string | Token contract address |
| `imageUrl` | string | Token logo URL |
| `tokenCreatedAt` | integer | Token creation timestamp (milliseconds) |

<Note>
For complete field descriptions, see the [Token Metadata API Reference](/en/api-reference/endpoint/token/v1/token-chain-tokenaddress-metadata-get).
</Note>

## Common Issues

<AccordionGroup>
  <Accordion title="401 Unauthorized Error">
    Please check:
    - Is the Access Token correct and not expired?
    - Is the Header format `Authorization: Bearer YOUR_ACCESS_TOKEN`?
    - Try generating a new token using your Client ID and Client Secret
  </Accordion>
  <Accordion title="429 Too Many Requests Error">
    You've reached the request rate limit. The free tier is limited to 10 requests/second. You can upgrade your plan to increase the limit.
    
    See [API Security](/en/guides/data-concepts/api-security) for details.
  </Accordion>
  <Accordion title="How to query tokens on other chains?">
    Change the chain parameter in the URL, for example:
    
    `GET /v1/token/eth/{address}/metadata` for Ethereum
    
    `GET /v1/token/bsc/{address}/metadata` for BSC
    
    See [Realtime Streaming](/en/guides/data-concepts/realtime-streaming) for the full list.
  </Accordion>
</AccordionGroup>
