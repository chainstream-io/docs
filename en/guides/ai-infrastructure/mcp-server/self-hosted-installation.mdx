## Package

The ChainStream MCP server is distributed as an npm package:

```
@chainstream-io/mcp
```

## Installation

<Tabs>
  <Tab title="Global Install">
    Install the package globally to make the `chainstream-mcp` binary available system-wide:

    ```bash
    npm install -g @chainstream-io/mcp
    ```

    After installation, run the server directly:

    ```bash
    chainstream-mcp
    ```
  </Tab>
  <Tab title="npx (No Install)">
    Run the MCP server on-demand without a permanent installation:

    ```bash
    npx @chainstream-io/mcp
    ```

    This downloads and executes the latest version each time. Ideal for quick testing.
  </Tab>
</Tabs>

## Binary

The package exposes a single binary:

```
chainstream-mcp
```

This binary starts the MCP server using stdio transport by default — the standard mode for connecting to Claude Desktop, Cursor, and other MCP clients. See [Stdio Transport](/en/guides/ai-infrastructure/mcp-server/stdio-transport) for client-specific configuration.

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `CHAINSTREAM_API_KEY` | Yes | — | Your ChainStream API key |
| `CHAINSTREAM_API_URL` | No | `https://api.chainstream.io` | ChainStream API base URL |

Set the API key as an environment variable or pass `--api-key` on the command line:

```bash
export CHAINSTREAM_API_KEY=your-key
chainstream-mcp

# Or pass directly
chainstream-mcp --api-key your-key
```

<Warning>
  Keep your API key secret. Never commit it to version control.
</Warning>

## Verify Installation

After installing, verify the package is available:

```bash
npx @chainstream-io/mcp --help
```

You should see the available options:

```
Options:
  --transport  Transport type: stdio for local CLI   [choices: "stdio"] [default: "stdio"]
  --api-key    ChainStream API key                   [string]
  --help       Show help                             [boolean]
  --version    Show version number                   [boolean]
```
