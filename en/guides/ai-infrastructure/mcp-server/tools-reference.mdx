The ChainStream MCP server exposes **17 tools** for token analytics, wallet profiling, DEX trading, market trends, and blockchain operations. The same 17 tools are available whether you use the **cloud endpoint** (`https://mcp.chainstream.io/mcp`) or the **npm package** (`@chainstream-io/mcp`).

---

## Tools (17)

### Token Tools (5)

#### `tokens_search`

Search tokens by name, symbol, or contract address across all supported chains. Returns top matches with name, symbol, chain, price, 24h change, and market cap. Use this as the first step when a user mentions a token by name.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `query` | string | Yes | ŌĆö | Search query: token name, symbol, or contract address |
| `chain` | string | No | ŌĆö | Filter by chain (omit to search all chains) |
| `limit` | number | No | `10` | Maximum results (1ŌĆō50) |

#### `tokens_analyze`

Comprehensive token analysis combining price, volume, holder distribution, security assessment, and liquidity depth. This is the primary tool for evaluating any token ŌĆö use it whenever someone asks "tell me about token X" or "is this token safe?". For price charts, use `tokens_price_history` instead.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `address` | string | Yes | ŌĆö | Token contract address |
| `sections` | array | No | `["overview","metrics","holders","security"]` | Data sections to include. Values: `overview`, `metrics`, `holders`, `security`, `liquidity` |

<Info>
  This tool aggregates multiple REST API calls (token detail, stats, market data, top holders, security) into a single response.
</Info>

#### `tokens_price_history`

Get OHLCV candlestick data for charting and technical analysis. Returns up to 100 candles with open, high, low, close, volume. Use `1h` or `4h` resolution for intraday analysis, `1d` for longer trends.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `address` | string | Yes | ŌĆö | Token contract address |
| `resolution` | enum | No | `"1h"` | Candle resolution: `1m`, `5m`, `15m`, `1h`, `4h`, `1d` |
| `limit` | number | No | `50` | Number of candles (1ŌĆō100) |

#### `tokens_discover`

Discover tokens by on-chain metrics. Use this to find tokens matching specific criteria like "high volume tokens on Solana" or "tokens with biggest price increase".

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `sort_by` | enum | No | `"volume"` | Sort metric: `volume`, `price_change`, `trades`, `market_cap` |
| `time_frame` | enum | No | `"24h"` | Time window: `1h`, `4h`, `24h` |
| `min_volume_usd` | number | No | ŌĆö | Minimum volume in USD |
| `limit` | number | No | `20` | Maximum results (1ŌĆō50) |

#### `tokens_compare`

Compare up to 5 tokens side by side. Returns standardized metrics for each: price, volume, market cap, holder count, 24h change. Use when the user asks "which token is better?" or needs to compare options.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `tokens` | array | Yes | ŌĆö | Array of `{ chain, address }` objects (2ŌĆō5 items) |

### Wallet Tools (2)

#### `wallets_profile`

Complete wallet profile: current token holdings with values, realized/unrealized PnL, and total net worth. Use when someone asks "what does this wallet hold?" or "how profitable is this wallet?". For recent transactions, use `wallets_activity` instead.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `address` | string | Yes | ŌĆö | Wallet address |
| `include` | array | No | `["holdings","pnl"]` | Data sections: `holdings` (token balances), `pnl` (profit/loss), `net_worth` (total value) |

<Info>
  Equivalent to `wallet_get_balance` + `wallet_get_pnl` + `wallet_get_net_worth` on the hosted endpoint.
</Info>

#### `wallets_activity`

Recent wallet activity: token transfers and transaction history. Use when investigating "what has this wallet been doing recently?" or checking for suspicious activity. For portfolio and PnL data, use `wallets_profile` instead.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `address` | string | Yes | ŌĆö | Wallet address |
| `limit` | number | No | `20` | Number of transfers (1ŌĆō50) |

### Market Tools (1)

#### `market_trending`

Get trending tokens by category. `hot` = most actively traded, `new` = recently created tokens, `migrated` = tokens that moved from launchpad to DEX, `graduating` = tokens approaching migration threshold, `stocks` = stock-related meme tokens. Default chain is solana.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | No | `"solana"` | Chain to query |
| `category` | enum | No | `"hot"` | Trending category: `hot`, `new`, `migrated`, `graduating`, `stocks` |
| `limit` | number | No | `20` | Maximum results (1ŌĆō50) |

### DEX Tools (3)

#### `dex_quote`

Get a swap quote showing expected output amount, price impact, and fees. Read-only ŌĆö no funds are moved. Always call this before `dex_swap` to show the user what they will receive and confirm before executing.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `dex` | string | Yes | ŌĆö | DEX protocol (jupiter, raydium, pumpfun, etc.) |
| `inputMint` | string | Yes | ŌĆö | Input token address |
| `outputMint` | string | Yes | ŌĆö | Output token address |
| `amount` | string | Yes | ŌĆö | Input amount in smallest unit |
| `slippage` | number | Yes | ŌĆö | Slippage tolerance in percent (0ŌĆō100) |

#### `dex_swap`

Execute a token swap on a DEX. **HIGH RISK** ŌĆö this creates a real transaction that moves funds. Always call `dex_quote` first and present the quote to the user. The user must explicitly confirm before this tool is called.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `dex` | string | Yes | ŌĆö | DEX protocol |
| `userAddress` | string | Yes | ŌĆö | User wallet address |
| `inputMint` | string | Yes | ŌĆö | Input token address |
| `outputMint` | string | Yes | ŌĆö | Output token address |
| `amount` | string | Yes | ŌĆö | Input amount in smallest unit |
| `slippage` | number | Yes | ŌĆö | Slippage tolerance in percent (0ŌĆō100) |
| `swapMode` | enum | No | `"ExactIn"` | Swap mode: `ExactIn`, `ExactOut` |
| `poolAddress` | string | No | ŌĆö | Specific pool address |
| `priorityFee` | string | No | ŌĆö | Priority fee for Solana |

<Warning>
  Executes a real on-chain transaction. MCP clients should require explicit user confirmation.
</Warning>

#### `dex_create_token`

Create a new token on a launchpad DEX (PumpFun, Raydium, Moonshot, etc.). **HIGH RISK** ŌĆö this deploys a real token contract. Requires user confirmation.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `dex` | enum | Yes | ŌĆö | Launchpad DEX: `raydium`, `pumpfun`, `moonshot`, `candy`, `launchpad` |
| `userAddress` | string | Yes | ŌĆö | Creator wallet address |
| `name` | string | Yes | ŌĆö | Token name (max 32 chars) |
| `symbol` | string | Yes | ŌĆö | Token symbol (max 10 chars) |
| `uri` | string | No | ŌĆö | Token metadata URI |
| `image` | string | No | ŌĆö | Token image URL |
| `priorityFee` | string | No | ŌĆö | Priority fee for Solana |

<Warning>
  Creates a real on-chain token. Require explicit user confirmation.
</Warning>

### Trading Tools (2)

#### `trading_backtest`

Backtest a trading strategy against historical data. Returns PnL curve, Sharpe ratio, max drawdown, and trade details. Currently in development ŌĆö results will be delivered via progress notifications when available.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `token` | string | Yes | ŌĆö | Token address |
| `strategy` | string | Yes | ŌĆö | Strategy description or code |
| `startTime` | number | Yes | ŌĆö | Start timestamp (unix seconds) |
| `endTime` | number | Yes | ŌĆö | End timestamp (unix seconds) |
| `initialCapital` | number | No | `10000` | Starting capital in USD |

<Note>
  This tool is in development and returns a placeholder response with the submitted parameters.
</Note>

#### `trading_execute`

Execute a trade on a DEX based on a trading signal. **HIGH RISK** ŌĆö this moves real funds. Requires explicit user confirmation. Consider using `dex_swap` directly for more control over swap parameters.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `action` | enum | Yes | ŌĆö | Trade action: `buy`, `sell` |
| `token` | string | Yes | ŌĆö | Token address |
| `amount` | string | Yes | ŌĆö | Trade amount |
| `maxSlippage` | number | No | `3.0` | Max slippage percent |
| `userAddress` | string | Yes | ŌĆö | User wallet address |

<Warning>
  Executes a real on-chain transaction. MCP clients should require explicit user confirmation.
</Warning>

<Note>
  This tool is in development and returns a placeholder response.
</Note>

### Infrastructure Tools (3)

#### `webhooks_manage`

Manage webhook endpoints for real-time event notifications. Use `list` to see existing webhooks, `create` to set up new event subscriptions, `delete` to remove an endpoint.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `action` | enum | Yes | ŌĆö | Action: `list`, `create`, `delete` |
| `endpoint_id` | string | No | ŌĆö | Required for `delete` action |
| `url` | string | No | ŌĆö | Webhook URL (required for `create`) |
| `events` | array | No | ŌĆö | Event types to subscribe (required for `create`) |

#### `blockchain_info`

Get information about supported blockchains and available DEX protocols. Use when the agent needs to know what chains or DEXes are available.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | No | ŌĆö | Specific chain to query (omit for all chains) |

#### `transaction_send`

Broadcast a signed transaction to the blockchain. **HIGH RISK** ŌĆö this submits a real transaction that cannot be reversed. The transaction must already be signed by the user's wallet.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `chain` | string | Yes | ŌĆö | Blockchain identifier |
| `signedTransaction` | string | Yes | ŌĆö | Signed transaction data (base64 or hex) |

<Warning>
  Irreversible on-chain transaction. Ensure proper signing and review.
</Warning>
