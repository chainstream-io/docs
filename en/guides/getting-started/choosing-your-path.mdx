## Quick Selection

Choose your path based on your primary needs:

| I want to... | Recommended Path |
|:--|:--|
| Get token prices and trading data | [Data User](#data-user) |
| Detect transaction risks and analyze addresses | [Compliance User](#compliance-user) |
| Let AI query on-chain data | [AI Builder](#ai-builder) |

---

## Data User

### Use Cases

Developers who need to obtain on-chain data for display, analysis, or trading decisions.

**Typical scenarios:**
- Display real-time token prices in dApps
- Build market dashboards or data analytics tools
- Monitor position changes of specific wallets
- Develop trading bots to get market data

**Typical users:**
- DeFi application developers
- Market data display platforms
- On-chain data analysts
- Trading bot developers

### Recommended Path

<Steps>
  <Step title="Understand Data Models">
    Read [Data Concepts](/en/guides/data-concepts/realtime-streaming) to understand ChainStream's data structure
  </Step>
  <Step title="Call Token API">
    View [Token API Reference](/en/api-reference/endpoint/token/v1/token-chain-tokenaddress-get) to get price and token information
  </Step>
  <Step title="Hands-on: Price Alert">
    Follow the [Price Alert Bot Tutorial](/en/playbooks/tutorials/build-price-alert-bot) to build a complete application
  </Step>
</Steps>

### Core Capabilities

You will use the following capabilities:
- Real-time price data (WebSocket push)
- Multi-chain token information queries
- Trading and position data
- Smart Money wallet tracking

<Note>
**Estimated time**: Completing the recommended path takes approximately 1-2 hours
</Note>

## Compliance User

### Use Cases

Teams that need to assess transaction risks or analyze address security.

**Typical scenarios:**
- Detect if user deposits come from high-risk addresses
- Analyze wallet address historical behavior
- Generate transaction risk assessment reports
- Integrate risk control flows into business systems

**Typical users:**
- Exchange compliance teams
- Wallet security teams
- Risk control system developers

### Recommended Path

<Steps>
  <Step title="Understand KYT/KYA">
    Read [Security Compliance Overview](/en/guides/security-compliance/overview) to understand risk control capabilities
  </Step>
  <Step title="Call Risk Assessment API">
    View [KYT API Reference](/en/api-reference/endpoint/kyt/v1/kyt-transfer-post) to understand transaction risk assessment interfaces
  </Step>
  <Step title="Hands-on: Deposit Risk Control">
    Follow the [Deposit Risk Control Integration Tutorial](/en/playbooks/tutorials/integrate-deposit-check) to implement the complete flow
  </Step>
</Steps>

### Core Capabilities

You will use the following capabilities:
- KYT transaction risk assessment (returns risk scores and labels)
- KYA address profiling (address type, historical behavior)
- Risk label system (configurable risk rules)

<Note>
**Estimated time**: Completing the recommended path takes approximately 2-3 hours
</Note>

## AI Builder

### Use Cases

Developers who want AI Agents to query and understand on-chain data.

**Typical scenarios:**
- Let Claude/GPT answer on-chain data questions
- Build AI assistants that understand crypto markets
- Integrate on-chain data queries into AI workflows

**Typical users:**
- AI Agent developers
- Automated trading system developers
- LLM application developers
- MCP integration developers

### Recommended Path

<Steps>
  <Step title="Understand AI Capabilities">
    Read [AI Infrastructure Overview](/en/guides/ai-infrastructure/overview) to understand the MCP protocol
  </Step>
  <Step title="Configure MCP Server">
    Follow the [MCP Setup Guide](/en/guides/ai-infrastructure/mcp-server/setup-guide) to complete integration
  </Step>
  <Step title="Hands-on: AI Assistant">
    Follow the [AI Trading Assistant Tutorial](/en/playbooks/tutorials/ai-agent-with-mcp) to build a complete application
  </Step>
</Steps>

### Core Capabilities

You will use the following capabilities:
- MCP Server integration (supports Claude Desktop and other clients)
- AI-friendly toolset (structured output)
- Natural language queries (e.g., "Query ETH price in the last 24 hours")

<Warning>
**Note**: The current AI execution layer only supports read operations and does not support automatic execution of write operations like trades.
</Warning>

<Note>
**Estimated time**: Completing the recommended path takes approximately 1-2 hours
</Note>
