## Quick Selection

Choose your path based on your primary needs:

| I want to... | Recommended Path |
|:--|:--|
| Get token prices and trading data | [Data User](#data-user) |
| Run flexible analytical queries on on-chain data | [GraphQL Analyst](#graphql-analyst) |

## Data User

### Use Cases

Developers who need to obtain on-chain data for display, analysis, or trading decisions.

**Typical scenarios:**
- Display real-time token prices in dApps
- Build market dashboards or data analytics tools
- Monitor position changes of specific wallets
- Develop trading bots to get market data

**Typical users:**
- DeFi application developers
- Market data display platforms
- On-chain data analysts
- Trading bot developers

### Recommended Path

<Steps>
  <Step title="Understand Data Models">
    Read [Data Concepts](/en/guides/data-concepts/realtime-streaming) to understand ChainStream's data structure
  </Step>
  <Step title="Call Token API">
    View [Token API Reference](/en/api-reference/endpoint/token/v1/token-chain-tokenaddress-get) to get price and token information
  </Step>
  <Step title="Hands-on: Price Alert">
    Follow the [Price Alert Bot Tutorial](/en/playbooks/tutorials/build-price-alert-bot) to build a complete application
  </Step>
</Steps>

### Core Capabilities

You will use the following capabilities:
- Real-time price data (WebSocket push)
- Multi-chain token information queries
- Trading and position data
- Smart Money wallet tracking

<Note>
**Estimated time**: Completing the recommended path takes approximately 1-2 hours
</Note>

## GraphQL Analyst

### Use Cases

Developers and analysts who need flexible, declarative queries on on-chain OLAP data with aggregation capabilities.

**Typical scenarios:**
- Analyze DEX trading patterns across multiple chains
- Build custom dashboards with aggregated on-chain metrics
- Query token holder distributions and wallet PnL
- Perform ad-hoc analytical queries on trades, transfers, and pools

**Typical users:**
- On-chain data analysts
- Quantitative researchers
- Dashboard and analytics tool builders
- DeFi protocol teams

### Recommended Path

<Steps>
  <Step title="Understand GraphQL API">
    Read the [GraphQL Overview](/en/graphql/getting-started/overview) to understand the OLAP analytical API
  </Step>
  <Step title="Run Your First Query">
    Follow the [First Query Guide](/en/graphql/getting-started/first-query) using the GraphQL IDE
  </Step>
  <Step title="Explore Data Cubes">
    Learn about the [12 Data Cubes](/en/graphql/schema/cubes) covering trades, transfers, balances, and more
  </Step>
  <Step title="Try Query Examples">
    Browse [DEX Trades Examples](/en/graphql/examples/dex-trades) and other query examples
  </Step>
</Steps>

### Core Capabilities

You will use the following capabilities:
- Single GraphQL endpoint for all on-chain data queries
- 12 Data Cubes spanning DEX trades, transfers, balances, pools, OHLC, and token metrics
- Built-in aggregation (count, sum, avg, min, max, uniq)
- Flexible filtering, ordering, and pagination
- Browser-based GraphQL IDE with templates and code export

<Note>
**Estimated time**: Completing the recommended path takes approximately 1-2 hours
</Note>

## Compliance User

### Use Cases

Teams that need to assess transaction risks or analyze address security.

**Typical scenarios:**
- Detect if user deposits come from high-risk addresses
- Analyze wallet address historical behavior
- Generate transaction risk assessment reports
- Integrate risk control flows into business systems

**Typical users:**
- Exchange compliance teams
- Wallet security teams
- Risk control system developers

### Recommended Path

<Steps>
  <Step title="Understand KYT/KYA">
    Read [Security Compliance Overview](/en/guides/security-compliance/overview) to understand risk control capabilities
  </Step>
  <Step title="Call Risk Assessment API">
    View [KYT API Reference](/en/api-reference/endpoint/kyt/v1/kyt-transfer-post) to understand transaction risk assessment interfaces
  </Step>
  <Step title="Hands-on: Deposit Risk Control">
    Follow the [Deposit Risk Control Integration Tutorial](/en/playbooks/tutorials/integrate-deposit-check) to implement the complete flow
  </Step>
</Steps>

### Core Capabilities

You will use the following capabilities:
- KYT transaction risk assessment (returns risk scores and labels)
- KYA address profiling (address type, historical behavior)
- Risk label system (configurable risk rules)

<Note>
**Estimated time**: Completing the recommended path takes approximately 2-3 hours
</Note>

## AI Builder

### Use Cases

Developers who want AI Agents to query and understand on-chain data.

**Typical scenarios:**
- Let Claude/GPT answer on-chain data questions
- Build AI assistants that understand crypto markets
- Integrate on-chain data queries into AI workflows

**Typical users:**
- AI Agent developers
- Automated trading system developers
- LLM application developers
- MCP integration developers

### Recommended Path

<Steps>
  <Step title="Understand AI Capabilities">
    Read [AI Infrastructure Overview](/en/guides/ai-infrastructure/overview) to understand the MCP protocol
  </Step>
  <Step title="Configure MCP Server">
    Follow the [MCP Setup Guide](/en/guides/ai-infrastructure/mcp-server/setup-guide) to complete integration
  </Step>
  <Step title="Hands-on: AI Assistant">
    Follow the [AI Trading Assistant Tutorial](/en/playbooks/tutorials/ai-agent-with-mcp) to build a complete application
  </Step>
</Steps>

### Core Capabilities

You will use the following capabilities:
- MCP Server integration (supports Claude Desktop and other clients)
- AI-friendly toolset (structured output)
- Natural language queries (e.g., "Query ETH price in the last 24 hours")

<Note>
**Estimated time**: Completing the recommended path takes approximately 1-2 hours
</Note>

## AI Agent Developer

### Use Cases

Developers building AI agents that need both data access and DeFi execution via CLI or Agent Skills.

**Typical scenarios:**
- Build AI agents that query on-chain data and execute trades
- Automate token research and portfolio management
- Integrate ChainStream into CI/CD pipelines
- Use Agent Skills in Cursor, Claude Code, or Codex

**Typical users:**
- AI Agent developers
- DevOps engineers automating on-chain workflows
- Cursor / Claude Code power users

### Recommended Path

<Steps>
  <Step title="Install CLI">
    Read [CLI Overview](/en/guides/cli/overview) and run `npx @chainstream-io/cli login`
  </Step>
  <Step title="Install Agent Skills">
    Follow the [Skills Installation Guide](/en/guides/ai-infrastructure/agent-skills/installation) for your platform
  </Step>
  <Step title="Explore Data Queries">
    Learn [chainstream-data skill](/en/guides/ai-infrastructure/agent-skills/chainstream-data) for token research and wallet analysis
  </Step>
  <Step title="Execute DeFi (Optional)">
    Learn [chainstream-defi skill](/en/guides/ai-infrastructure/agent-skills/chainstream-defi) for swap and token creation
  </Step>
</Steps>

### Core Capabilities

You will use the following capabilities:
- CLI commands for token, market, wallet, and DeFi operations
- Agent Skills for structured AI workflows
- x402 auto-payment for seamless subscription management
- MCP Server integration for AI chat assistants

<Note>
**Estimated time**: Completing the recommended path takes approximately 30-60 minutes
</Note>
