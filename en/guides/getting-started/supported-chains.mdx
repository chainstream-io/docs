## Network Types

### EVM (Ethereum Virtual Machine)

Chains: Ethereum, BSC, Polygon, Arbitrum, Base, Avalanche, Optimism

- Use standard `{chain}` parameter: `eth`, `bsc`, `polygon`, `arbitrum`, `base`, `avax`, `optimism`
- Compatible with ERC-20 token standard
- Address format: `0x` prefixed, 42-character hex string

### SVM (Solana Virtual Machine)

Chains: Solana

- Use `sol` as chain parameter
- SPL token standard
- Address format: Base58 encoded, 32-44 characters

### TVM (TRON Virtual Machine)

Chains: TRON

- Use `tron` as chain parameter
- TRC-20 token standard
- Address format: Base58Check encoded, starts with `T`

## Kafka Streams by Chain

Kafka Streams provides the lowest-latency data pipeline. Currently supported on:

| Chain | Topics Available | Latency |
|:--|:--|:--|
| Solana | dex.trades, tokens, balances, dex.pools, transfers, candlesticks | < 500ms |
| Ethereum | dex.trades, tokens, balances, dex.pools, transfers, candlesticks | < 2s |
| BSC | dex.trades, tokens, balances, dex.pools, transfers, candlesticks | < 2s |
| TRON | dex.trades, tokens, balances, transfers | < 2s |

All chains also support derived topics: `token-supplies`, `token-prices`, `token-holdings`, `token-market-caps`, `trade-stats`.

<Tip>
Kafka Streams requires separate credentials. Contact [support@chainstream.io](mailto:support@chainstream.io) to apply for access.
</Tip>

## API Base URL

All chains use the same base URL. Specify the chain via the `{chain}` path parameter:

```
https://api.chainstream.io/v2/{resource}/{chain}/...
```

**Example:**

```bash
# Solana token metadata
GET https://api.chainstream.io/v2/token/sol/{tokenAddress}/metadata

# Ethereum token price
GET https://api.chainstream.io/v2/token/eth/{tokenAddress}/price

# BSC wallet balance
GET https://api.chainstream.io/v2/wallet/bsc/{walletAddress}/tokens/balance
```
