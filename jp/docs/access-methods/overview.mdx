ChainStream は同じ [データプロダクト](/jp/docs/data-products/overview) のカタログを、複数のインターフェースから提供します。

## 比較

| インターフェース | 向いている用途 | 遅延 | 認証 |
|:--|:--|:--|:--|
| REST | 特定値の参照、スナップショット | 〜100ms | API Key / OAuth / x402 |
| GraphQL | 柔軟な分析クエリ | 〜200ms | API Key / OAuth |
| WebSocket | リアルタイムストリーム | 数百 ms | API Key |
| Kafka Streams | 低遅延・高信頼のサーバ側消費 | 数十 ms | mTLS / SASL |
| SDK (TS/Py/Go/Rust) | 型付きアプリ統合 | REST に準ずる | API Key |
| CLI | Shell / スクリプト / Agent | REST に準ずる | 設定済みキー |
| MCP Server | AI Agent からの接続 | REST に準ずる | ホスト / セルフホスト |

## どれを選ぶか

<AccordionGroup>
  <Accordion title="Web / モバイルアプリ">
    REST または SDK を選びましょう。ストリーミング UI があれば WebSocket を追加。
  </Accordion>
  <Accordion title="データパイプライン / ウェアハウス">
    Kafka Streams。バックフィルは REST を併用。
  </Accordion>
  <Accordion title="取引ボット">
    価格と取引は WebSocket、執行は REST の DeFi エンドポイント。
  </Accordion>
  <Accordion title="AI Agent">
    [MCP Server](/jp/docs/ai-agents/mcp-server/introduction) または SDK。
  </Accordion>
</AccordionGroup>
