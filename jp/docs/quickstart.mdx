## 学べること

このチュートリアルを完了すると、以下ができるようになります：

- API Key または OAuth トークンで ChainStream に認証する
- API を呼び出してトークンメタデータを取得する
- CLI と SDK で同じクエリを実行する
- API レスポンスの構造を理解する

<Note>
**推定所要時間**: 5 分
</Note>

---

## 前提条件

- ChainStream アカウントの登録（[こちらから登録](https://www.chainstream.io/dashboard)）

---

## ステップ 1: API Key の取得

1. [ChainStream Dashboard](https://www.chainstream.io/dashboard) にログイン
2. **Applications** に移動
3. **Create New App** をクリック
4. **API Key**（`cs_live_...` で始まるもの）をコピー

<Tip>
API Key は最も簡単な認証方法です。OAuth 2.0 JWT フローについては、[認証](/jp/guides/getting-started/authentication)を参照してください。
</Tip>

---

## ステップ 2: 最初のリクエスト

Solana 上の SOL トークンメタデータをクエリします。お好みの方法を選択してください：

<Tabs>
  <Tab title="cURL">
```bash
curl "https://api.chainstream.io/v2/token/sol/So11111111111111111111111111111111111111112" \
  -H "X-API-KEY: your_api_key"
```
  </Tab>
  <Tab title="CLI">
```bash
# npx でワンライナー実行（インストール不要）
npx @chainstream-io/cli token info --chain sol \
  --address So11111111111111111111111111111111111111112
```

CLI は初回実行時に認証を求めます。事前にキーを設定することもできます：

```bash
npx @chainstream-io/cli config set --key apiKey --value your_api_key
```
  </Tab>
  <Tab title="SDK (TypeScript)">
```typescript
import { ChainStreamClient } from "@chainstream-io/sdk";

const cs = new ChainStreamClient({ apiKey: "your_api_key" });

const token = await cs.token.getToken(
  "So11111111111111111111111111111111111111112",
  "solana"
);
console.log(token.name, token.symbol, token.decimals);
```
  </Tab>
  <Tab title="MCP (AI Agent)">
AI エージェントを ChainStream データに接続します — コード不要：

```bash
CHAINSTREAM_API_KEY=your_api_key npx @chainstream-io/mcp
```

その後、AI に尋ねてみましょう：*「Solana の SOL トークンは何ですか？」*

Claude/Cursor の設定については、[MCP Server セットアップ](/jp/guides/ai-infrastructure/mcp-server/setup-guide)を参照してください。
  </Tab>
  <Tab title="OAuth 2.0 (JWT)">
OAuth 2.0 Client Credentials フローを使用するアプリ向け：

```bash
# ステップ 1: アクセストークンの取得
curl -X POST "https://dex.asia.auth.chainstream.io/oauth/token" \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET",
    "audience": "https://api.dex.chainstream.io",
    "grant_type": "client_credentials"
  }'

# ステップ 2: トークンを使用
curl "https://api.chainstream.io/v2/token/sol/So11111111111111111111111111111111111111112" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```
  </Tab>
</Tabs>

<Tip>
`YOUR_ACCESS_TOKEN` をステップ 2 で生成したトークンに置き換えてください。
</Tip>

### レスポンス例

```json
{
  "chain": "sol",
  "decimals": 9,
  "name": "Wrapped SOL",
  "symbol": "SOL",
  "address": "So11111111111111111111111111111111111111112",
  "imageUrl": "https://raw.githubusercontent.com/solana-labs/token-list/main/assets/mainnet/So11111111111111111111111111111111111111112/logo.png",
  "tokenCreatedAt": 1710417600000
}
```

---

## ステップ 4: レスポンスの理解

| フィールド | 型 | 説明 |
|:--|:--|:--|
| `chain` | string | ブロックチェーンネットワーク識別子 |
| `decimals` | integer | トークンの小数桁数 |
| `name` | string | トークン名 |
| `symbol` | string | トークンシンボル |
| `address` | string | トークンのコントラクトアドレス |
| `imageUrl` | string | トークンロゴの URL |
| `tokenCreatedAt` | integer | トークン作成タイムスタンプ（ミリ秒） |

<Note>
完全なフィールド説明については、[Token Metadata API リファレンス](/jp/api-reference/endpoint/token/v1/token-chain-tokenaddress-metadata-get)を参照してください。
</Note>

## よくある問題

<AccordionGroup>
  <Accordion title="401 Unauthorized エラー">
    以下を確認してください：
    - Access Token は正しく、期限切れではありませんか？
    - ヘッダー形式は `Authorization: Bearer YOUR_ACCESS_TOKEN` ですか？
    - Client ID と Client Secret を使用して新しいトークンを生成してみてください
  </Accordion>
  <Accordion title="429 Too Many Requests エラー">
    リクエストのレート制限に達しました。フリープランは 10 リクエスト/秒に制限されています。プランをアップグレードすることで制限を引き上げられます。
    
    詳細は [API セキュリティ](/jp/guides/data-concepts/api-security)を参照してください。
  </Accordion>
  <Accordion title="他のチェーンのトークンをクエリするには？">
    URL のチェーンパラメータを変更します：
    
    `GET /v1/token/eth/{address}/metadata` ← Ethereum の場合
    
    `GET /v1/token/bsc/{address}/metadata` ← BSC の場合
    
    完全なリストは[リアルタイムストリーミング](/jp/guides/data-concepts/realtime-streaming)を参照してください。
  </Accordion>
</AccordionGroup>
