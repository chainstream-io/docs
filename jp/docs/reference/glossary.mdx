## ブロックチェーン用語

基本的なブロックチェーンの概念と一般用語。

### A

<Accordion title="Address">
ブロックチェーン上で資産の送受信に使用される一意の識別子。Ethereumアドレスは `0x` で始まり、42文字の長さです。

例：`0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045`
</Accordion>

<Accordion title="ABI (Application Binary Interface)">
スマートコントラクトのインターフェース定義で、コントラクトとやり取りするための関数やデータ構造を記述します。
</Accordion>

### B

<Accordion title="Block">
ブロックチェーンにおけるトランザクションデータを格納する基本単位。各ブロックにはトランザクションのセット、タイムスタンプ、前のブロックのハッシュが含まれます。

| フィールド | 説明 |
|-------|-------------|
| Block Number | ブロックの高さ |
| Block Hash | ブロックハッシュ |
| Timestamp | ブロック時刻 |
| Transactions | 含まれるトランザクション |
</Accordion>

<Accordion title="Block Confirmation">
トランザクションがブロックに含まれた後に生成された後続ブロックの数。確認数が多いほど、トランザクションの不可逆性が高くなります。

| チェーン | 推奨確認数 |
|-------|--------------------------|
| Ethereum | 12-32 |
| BSC | 15 |
| Solana | 32 |
</Accordion>

<Accordion title="Bridge">
異なるブロックチェーン間で資産やデータの転送を可能にするプロトコル。代表的なブリッジにはLayerZero、Wormholeなどがあります。
</Accordion>

### C

<Accordion title="Chain">
ブロックチェーンネットワーク。ChainStreamはEthereum、BSC、Polygon、Arbitrum、Solanaなど複数のチェーンをサポートしています。

チェーン識別子の例：
- `ethereum` - Ethereumメインネット
- `bsc` - BNB Chain
- `arbitrum` - Arbitrum One
- `solana` - Solana
</Accordion>

<Accordion title="Contract Address">
スマートコントラクトがブロックチェーン上にデプロイされた一意のアドレス。通常のアドレス（EOA）とは異なり、コントラクトアドレスはコードによって制御されます。
</Accordion>

### E

<Accordion title="EOA (Externally Owned Account)">
秘密鍵によって制御されるアカウント。スマートコントラクトアドレスとは対照的です。
</Accordion>

<Accordion title="ERC-20">
Ethereumのトークン標準。`transfer`、`approve`、`balanceOf` などの基本的なトークン関数インターフェースを定義します。
</Accordion>

<Accordion title="ERC-721">
EthereumのNFT標準。非代替性トークンのインターフェース仕様を定義します。
</Accordion>

<Accordion title="EVM (Ethereum Virtual Machine)">
スマートコントラクトコードを実行するためのランタイム環境。Polygon、Arbitrum、BaseはすべてEVM互換チェーンです。
</Accordion>

### G

<Accordion title="Gas">
ブロックチェーントランザクションの実行に必要な手数料の単位。Gas Fee = Gas Used × Gas Price。

| 用語 | 説明 |
|------|-------------|
| Gas Limit | トランザクションの最大ガス消費量 |
| Gas Used | 実際に消費されたガス |
| Gas Price | ガス1単位あたりの価格 |
| Gwei | Gas Priceの単位、1 Gwei = 10⁻⁹ ETH |
</Accordion>

### H

<Accordion title="Hash">
任意の長さのデータを固定長の文字列に変換する暗号化関数。ブロックチェーンではトランザクションハッシュやブロックハッシュによく使用されます。

例：`0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef`
</Accordion>

### L

<Accordion title="Layer 1 (L1)">
第1層ブロックチェーン。Ethereum、Solana、Bitcoinなどの独立したメインチェーン。
</Accordion>

<Accordion title="Layer 2 (L2)">
第2層スケーリングソリューション。Arbitrum、Optimism、Base、zkSyncなど、L1上に構築されたスケーリングソリューション。
</Accordion>

### M

<Accordion title="Mempool">
保留中のトランザクションの待機エリア。トランザクションはブロックに含まれる前にmempoolで待機します。
</Accordion>

### N

<Accordion title="NFT (Non-Fungible Token)">
一意のデジタル資産。デジタルアート、ゲームアイテムなどに広く使用されています。
</Accordion>

<Accordion title="Node">
ブロックチェーンソフトウェアを実行するコンピュータで、ネットワークの検証とデータストレージに参加します。
</Accordion>

<Accordion title="Nonce">
トランザクションのシーケンス番号。各アドレスがトランザクションを送信する際のインクリメンタルカウンターで、リプレイ攻撃を防止します。
</Accordion>

### R

<Accordion title="RPC (Remote Procedure Call)">
ブロックチェーンノードとやり取りするためのプロトコル。トランザクションの送信やデータの照会などに使用されます。
</Accordion>

### S

<Accordion title="Smart Contract">
ブロックチェーン上で自動的に実行されるプログラムコード。一度デプロイされると、コードロジックは変更できません（アップグレードメカニズムが設計されている場合を除く）。
</Accordion>

### T

<Accordion title="Token">
ブロックチェーン上のデジタル資産。ネイティブトークン（ETHなど）とコントラクトトークン（USDCなど）を含みます。
</Accordion>

<Accordion title="Transaction">
送金、コントラクト呼び出しなど、ブロックチェーン上の状態変更操作。

| フィールド | 説明 |
|-------|-------------|
| TxHash | トランザクションハッシュ、一意の識別子 |
| From | 送信者アドレス |
| To | 受信者アドレス |
| Value | 送金額 |
| Input Data | コントラクト呼び出しデータ |
</Accordion>

<Accordion title="Transaction Hash / TxHash">
トランザクションの一意の識別子。ブロックエクスプローラーでトランザクションの詳細を照会するために使用できます。
</Accordion>

### W

<Accordion title="Wallet">
暗号資産を保管・管理するためのツール。ソフトウェア（MetaMaskなど）またはハードウェア（Ledgerなど）があります。
</Accordion>

## DeFi用語

分散型金融に関連する概念。

### A

<Accordion title="AMM (Automated Market Maker)">
DEXで流動性提供と価格設定を行うアルゴリズムメカニズム。Uniswapの定積公式 `x * y = k` などがあります。
</Accordion>

<Accordion title="APR (Annual Percentage Rate)">
複利を考慮しない年間利率。
</Accordion>

<Accordion title="APY (Annual Percentage Yield)">
複利を考慮した年間利回り。

計算式：`APY = (1 + APR/n)^n - 1`、nは複利計算の回数。
</Accordion>

<Accordion title="ATH (All-Time High)">
トークンや資産がこれまでに到達した最高価格。
</Accordion>

<Accordion title="ATL (All-Time Low)">
トークンや資産がこれまでに到達した最低価格。
</Accordion>

### C

<Accordion title="CEX (Centralized Exchange)">
中央集権型機関が運営する暗号通貨取引プラットフォーム。

例：Binance、Coinbase、OKX、Bybit
</Accordion>

<Accordion title="Collateral">
レンディングプロトコルでローンを担保するために使用される資産。担保率は通常100%を超える必要があります。
</Accordion>

### D

<Accordion title="DEX (Decentralized Exchange)">
中央集権型機関を必要とせず、スマートコントラクトによって自動的に取引を実行する取引プラットフォーム。

例：Uniswap、SushiSwap、Raydium、Jupiter
</Accordion>

<Accordion title="dApp (Decentralized Application)">
ブロックチェーン上で動作するアプリケーション。
</Accordion>

<Accordion title="DeFi (Decentralized Finance)">
ブロックチェーンベースの金融サービスおよびアプリケーション。取引、レンディング、デリバティブなどを含みます。
</Accordion>

### F

<Accordion title="Farming / Yield Farming">
DeFiプロトコルに流動性を提供してトークン報酬を得る行為。
</Accordion>

<Accordion title="FDV (Fully Diluted Valuation)">
すべてのトークンが発行されたと仮定した場合の市場価値の合計。

計算式：`FDV = Token Price × Total Supply`
</Accordion>

<Accordion title="Flash Loan">
同一ブロック/トランザクション内で返済する必要がある無担保の即時ローン。アービトラージや清算などによく使用されます。
</Accordion>

### H

<Accordion title="Health Factor">
レンディングプロトコルにおけるポジションの安全性を測定する指標。Health Factorが1未満になるとポジションが清算される可能性があります。
</Accordion>

### I

<Accordion title="Impermanent Loss">
流動性提供者が資産を単純に保有する場合と比較して、価格変動によって被る損失。

| 価格変動 | インパーマネントロス |
|--------------|------------------|
| ±25% | -0.6% |
| ±50% | -2.0% |
| ±100% | -5.7% |
</Accordion>

### L

<Accordion title="Liquidation">
レンディングポジションの担保率が閾値を下回った際に、プロトコルが自動的に担保を売却して債務を返済するプロセス。
</Accordion>

<Accordion title="Liquidity">
市場で取引可能な資産の量。流動性が高いほど、大口取引でも価格への影響が少なくなります。
</Accordion>

<Accordion title="Liquidity Pool">
DEXにおいて取引ペア資産を格納するスマートコントラクト。取引に流動性を提供します。
</Accordion>

<Accordion title="LP (Liquidity Provider)">
DEXプールに資産を提供し、取引手数料を報酬として得るユーザー。
</Accordion>

<Accordion title="LP Token">
流動性プール内のユーザーのシェアを表すトークン。資産の引き出しやステーキングマイニングに使用できます。
</Accordion>

### M

<Accordion title="Market Cap">
トークンの市場価値の合計。

計算式：`Market Cap = Token Price × Circulating Supply`
</Accordion>

<Accordion title="MEV (Maximal Extractable Value)">
マイナーやバリデーターがトランザクションの順序変更、挿入、検閲を通じて抽出する追加的な価値。

一般的なMEVの種類：
- サンドイッチ攻撃
- フロントランニング
- バックランニング
- 清算
</Accordion>

### O

<Accordion title="Oracle">
Chainlink、Pythなど、オフチェーンデータをスマートコントラクトに提供するサービス。主に価格データに使用されます。
</Accordion>

### S

<Accordion title="Slippage">
期待される取引価格と実際の約定価格の差。流動性の低いプールではより大きくなる可能性があります。
</Accordion>

<Accordion title="Staking">
報酬を得たりガバナンスに参加するために、プロトコルにトークンをロックすること。
</Accordion>

<Accordion title="Swap">
DEXにおけるトークン交換操作。
</Accordion>

### T

<Accordion title="TVL (Total Value Locked)">
DeFiプロトコルにロックされた資産の総額。プロトコルの規模を測る重要な指標です。
</Accordion>

### W

<Accordion title="Whale">
大量の暗号資産を保有するアドレスや投資家。通常、トークン供給量の1%以上を保有するアドレスを指します。
</Accordion>

### Y

<Accordion title="Yield">
DeFiプロトコルへの投資から得られるリターン。取引手数料の分配、トークン報酬、利息などを含みます。
</Accordion>

## ChainStream用語

ChainStreamの製品およびサービスに関連する概念。

### A

<Accordion title="Access Token">
APIリクエストの認証に使用されるJWT（JSON Web Token）。

**取得方法**：
1. ダッシュボードでアプリを作成し、Client IDとClient Secretを取得
2. Client IDとClient Secretを使用して認証サービスを呼び出し、Access Tokenを生成
3. APIリクエストヘッダーに `Authorization: Bearer <Access Token>` を使用

**認証ドメイン**：`dex.asia.auth.chainstream.io`

**スコープ**：
| スコープ | 説明 |
|-------|-------------|
| `webhook.read` | Webhook APIの読み取り権限 |
| `webhook.write` | Webhook APIの書き込み権限 |
| `kyt.read` | KYT APIの読み取り権限 |
| `kyt.write` | KYT APIの書き込み権限 |

セキュリティに関するヒント：
- Client Secretをクライアントサイドのコードに公開しないでください
- 認証情報をバージョン管理にコミットしないでください
- 機密情報は環境変数に保存してください
- Access Tokenには有効期限があるため、定期的に更新してください

[認証ドキュメント](/jp/api-reference/authentication/authenticate)を参照
</Accordion>

### C

<Accordion title="CU (Compute Unit)">
ChainStreamにおけるAPIリクエストの複雑さを測定する単位。エンドポイントによって消費されるCUが異なります。
</Accordion>

### K

<Accordion title="KYA (Know Your Address)">
ChainStreamのアドレスリスク評価サービス。アドレスの過去の行動と関連するリスクを分析します。
</Accordion>

<Accordion title="KYT (Know Your Transaction)">
ChainStreamのトランザクションリスク管理サービス。トランザクションの資金源とリスクレベルを評価します。

リスクレベル：

| レベル | 説明 |
|-------|-------------|
| Low | 低リスク、自動処理可能 |
| Medium | 中リスク、手動レビュー推奨 |
| High | 高リスク、ブロック推奨 |
| Severe | 重大リスク、ブロック必須 |
</Accordion>

### R

<Accordion title="Rate Limit">
APIリクエストの頻度制限。ChainStreamはサブスクリプションティアに基づいて異なるレート制限を提供します。

レスポンスヘッダー：
- `X-RateLimit-Limit` - 制限総数
- `X-RateLimit-Remaining` - 残りの回数
- `X-RateLimit-Reset` - リセット時間
</Accordion>

### S

<Accordion title="Smart Money">
ChainStreamが特定した、市場平均を上回るリターンを継続的に達成しているアドレス。

種類：
- Top Traders
- Early Investors
- Institutional Wallets
- KOL Wallets

[Smart Money メソドロジー](/jp/docs/data-products/smart-money)を参照
</Accordion>

<Accordion title="Smart Score">
ChainStreamによるウォレットアドレスの総合スコアで、取引能力と過去のパフォーマンスを反映します。

| スコア範囲 | レベル |
|-------------|-------|
| 90-100 | トップ Smart Money |
| 75-89 | 優良トレーダー |
| 60-74 | 良好なトレーダー |
| &lt; 60 | 平均的なトレーダー |
</Accordion>

### W

<Accordion title="Webhook">
イベントがトリガーされた際に指定されたURLに通知を送信するメカニズム。

**サポートされるイベントタイプ**：
| イベント | 説明 |
|-------|-------------|
| `sol.token.created` | Solana上で新しいトークンが作成 |
| `sol.token.migrated` | Solana上でトークンが卒業（Bonding CurveからDEXへ移行） |

<Note>
さらに多くのイベントタイプが開発中です。
</Note>

**管理エンドポイント**：
- `POST /v1/webhook/endpoint` - Webhookの作成
- `GET /v1/webhook/endpoint` - Webhookの一覧取得
- `DELETE /v1/webhook/endpoint/{id}` - Webhookの削除

[Webhook基礎](/jp/docs/recipes/webhook-fundamentals)を参照
</Accordion>

<Accordion title="WebSocket">
リアルタイムの双方向通信をサポートするネットワークプロトコル。ChainStreamはWebSocketを介してリアルタイムデータストリームを提供します。

**エンドポイント**：`wss://realtime-dex.chainstream.io/connection/websocket`

**接続形式**：
```
wss://realtime-dex.chainstream.io/connection/websocket?token=YOUR_ACCESS_TOKEN
```

**サポートされるサブスクリプションタイプ**：
- トークンローソク足（`dex-token-candles`）
- トークン統計（`dex-token-stats`）
- トークンホルダー（`token-holders`）
- 新規トークン（`dex-new-token`）
- ウォレット残高（`wallet-balance`）
- ウォレット取引（`wallet-trade`）

<Note>
SDKを使用する場合、認証は自動的に処理されます。
</Note>

[WebSocket API](/jp/api-reference/endpoint/websocket/introduction)を参照
</Accordion>

## AI用語

AIおよびMCPに関連する概念。

### A

<Accordion title="Agent">
タスクを自律的に実行できるAIシステム。AI Agentはツールの使用、外部データへのアクセス、意思決定が可能です。
</Accordion>

<Accordion title="Agent Skills">
AIコーディングアシスタントに特定の機能の使い方を教える構造化された命令パッケージ（`SKILL.md`ファイル）。ChainStreamは2つのスキルを提供しています：`chainstream-data`（読み取り専用データクエリ）と`chainstream-defi`（DeFi実行）。スキルにはデシジョンツリー、ワークフロー、安全規則、エラーリカバリーが含まれており、AI Agentが自律的に動作するために必要なすべてが揃っています。

[Agent Skills入門](/jp/docs/ai-agents/agent-skills/introduction)を参照
</Accordion>

### C

<Accordion title="Context">
会話履歴、システムプロンプト、外部データなど、AIモデルが応答生成時に参照する情報。
</Accordion>

<Accordion title="Context Window">
AIモデルが一度に処理できるトークンの最大数。コンテキストウィンドウが大きいほど、より多くの情報を処理できます。
</Accordion>

### L

<Accordion title="LLM (Large Language Model)">
Claude、GPTなど、大量のテキストデータで学習されたAIモデル。
</Accordion>

### M

<Accordion title="MCP (Model Context Protocol)">
Anthropicが開発したオープンプロトコルで、AIモデルが外部データソースやツールとやり取りすることを可能にします。

ChainStream MCP Serverはこのプロトコルを実装しており、AIがオンチェーンデータを直接照会できます。

[MCP入門](/jp/docs/ai-agents/mcp-server/introduction)を参照
</Accordion>

<Accordion title="MCP Server">
MCPプロトコルを実装し、ツールとデータを提供するサーバー。ChainStream MCP Serverはオンチェーンデータの照会機能を提供します。
</Accordion>

<Accordion title="MCP Client">
Claude Desktop、CursorなどのMCPプロトコルを使用してツールを呼び出すアプリケーション。
</Accordion>

<Accordion title="MPP (Micropayment Protocol)">
Tempoネットワーク（チェーンID 4217）でのUSDC.e支払いに使用される決済プロトコル。Tempo上での支払いを好むAI Agent向けのx402の代替手段です。ガスはUSDC.eで直接支払われるため、ETHは不要です。

[x402 Payment](/jp/docs/platform/billing-payments/x402-payments)を参照
</Accordion>

### P

<Accordion title="Prompt">
AIモデルに与える入力テキストで、モデルが期待される出力を生成するよう誘導します。
</Accordion>

### T

<Accordion title="SKILL.md">
Agent Skillsの標準ファイル形式。YAMLフロントマター（名前、説明）と、デシジョンツリー、ワークフロー、安全規則、リファレンスを含む構造化されたMarkdownで構成されます。Cursor、Claude Code、Codexなどのコーディングアシスタントで使用されます。
</Accordion>

<Accordion title="TEE (Trusted Execution Environment)">
コードとデータの機密性と完全性を保証する、ハードウェアレベルのセキュリティ保護を提供するコンピューティング環境。ChainStream CLIはウォレットの鍵管理にTEEを使用しており、秘密鍵がセキュアエンクレーブの外に出ることはありません。

[CLI認証](/jp/docs/platform/authentication/api-keys-oauth)を参照
</Accordion>

<Accordion title="Token (AI)">
AIモデルがテキストを処理する際の基本単位。1トークンは約4英字または1〜2日本語文字に相当します。

<Note>
ここでのTokenはAIトークンを指しており、ブロックチェーンのトークンとは異なります。
</Note>
</Accordion>

<Accordion title="Tool">
MCPで定義され、AIが呼び出せる関数。ChainStreamが提供するツールには `get_token_balance`、`analyze_wallet` などがあります。
</Accordion>

### X

<Accordion title="x402">
[Coinbase](https://github.com/coinbase/x402)によるHTTP 402（Payment Required）上に構築されたオープン決済プロトコル。AI AgentがBaseネットワーク上でUSDCステーブルコインを使用してAPI呼び出しごとに支払うことを可能にします。ChainStreamはx402を認証スキームとしてサポートしており、APIキーを持たないAgentがFacilitatorサービスを通じて検証される署名済みオンチェーン承認（EIP-3009またはPermit2）を介して各ツール呼び出しに対して支払うことができます。アカウント登録は不要です。
</Accordion>
