## パッケージ

ChainStream MCPサーバーはnpmパッケージとして配布されています：

```
@chainstream-io/mcp
```

## インストール

<Tabs>
  <Tab title="グローバルインストール">
    パッケージをグローバルにインストールして、`chainstream-mcp` バイナリをシステム全体で利用可能にします：

    ```bash
    npm install -g @chainstream-io/mcp
    ```

    インストール後、サーバーを直接実行できます：

    ```bash
    chainstream-mcp
    ```
  </Tab>
  <Tab title="npx（インストール不要）">
    恒久的なインストールなしでMCPサーバーをオンデマンドで実行します：

    ```bash
    npx @chainstream-io/mcp
    ```

    毎回最新バージョンをダウンロードして実行します。クイックテストに最適です。
  </Tab>
</Tabs>

## バイナリ

パッケージは単一のバイナリを公開します：

```
chainstream-mcp
```

このバイナリはデフォルトでstdioトランスポートを使用してMCPサーバーを起動します — これはClaude Desktop、Cursor、その他のMCPクライアントへの接続の標準モードです。クライアント固有の設定については[Stdioトランスポート](/jp/guides/ai-infrastructure/mcp-server/stdio-transport)を参照してください。

## 環境変数

| 変数 | 必須 | デフォルト | 説明 |
|------|------|---------|------|
| `CHAINSTREAM_API_KEY` | はい | — | ChainStream APIキー |
| `CHAINSTREAM_API_URL` | いいえ | `https://api.chainstream.io` | ChainStream APIベースURL |

APIキーを環境変数として設定するか、コマンドラインで `--api-key` を渡します：

```bash
export CHAINSTREAM_API_KEY=your-key
chainstream-mcp

# Or pass directly
chainstream-mcp --api-key your-key
```

<Warning>
  APIキーは秘密にしてください。バージョン管理にコミットしないでください。
</Warning>

## インストールの確認

インストール後、パッケージが利用可能か確認します：

```bash
npx @chainstream-io/mcp --help
```

利用可能なオプションが表示されるはずです：

```
Options:
  --transport  Transport type: stdio for local CLI   [choices: "stdio"] [default: "stdio"]
  --api-key    ChainStream API key                   [string]
  --help       Show help                             [boolean]
  --version    Show version number                   [boolean]
```
