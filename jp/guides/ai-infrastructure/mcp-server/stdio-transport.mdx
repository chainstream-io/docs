## 概要

Stdioは、ChainStream MCPサーバーの**デフォルトトランスポート**です。標準入出力を介して通信し、ローカルIDEやデスクトップ統合の標準的な選択肢です。

```bash
chainstream-mcp
```

またはグローバルインストールなしで：

```bash
npx @chainstream-io/mcp
```

追加のフラグは不要です — `--transport` を指定しない場合、stdioが自動的に使用されます。

## Claude Desktop

Claude DesktopのMCP設定ファイルに以下を追加してください：

<Tabs>
  <Tab title="macOS">
    `~/Library/Application Support/Claude/claude_desktop_config.json` を編集：

    ```json
    {
      "mcpServers": {
        "chainstream": {
          "command": "npx",
          "args": ["@chainstream-io/mcp"],
          "env": {
            "CHAINSTREAM_API_KEY": "your-key"
          }
        }
      }
    }
    ```
  </Tab>
  <Tab title="Windows">
    `%APPDATA%\Claude\claude_desktop_config.json` を編集：

    ```json
    {
      "mcpServers": {
        "chainstream": {
          "command": "npx",
          "args": ["@chainstream-io/mcp"],
          "env": {
            "CHAINSTREAM_API_KEY": "your-key"
          }
        }
      }
    }
    ```
  </Tab>
</Tabs>

パッケージをグローバルにインストールしている場合は、バイナリを直接使用できます：

```json
{
  "mcpServers": {
    "chainstream": {
      "command": "chainstream-mcp",
      "env": {
        "CHAINSTREAM_API_KEY": "your-key"
      }
    }
  }
}
```

設定を保存した後、Claude Desktopを再起動してください。ChainStreamのツールがツールリストに自動的に表示されます。

## Claude Code

CLIを使用してClaude CodeにMCPサーバーを追加します：

```bash
claude mcp add chainstream -- npx @chainstream-io/mcp
```

Claude Codeを起動する前に、APIキーを環境変数として設定してください：

```bash
export CHAINSTREAM_API_KEY=your-key
```

または、プロジェクトの `.mcp.json` にサーバー設定を追加します：

```json
{
  "mcpServers": {
    "chainstream": {
      "command": "npx",
      "args": ["@chainstream-io/mcp"],
      "env": {
        "CHAINSTREAM_API_KEY": "your-key"
      }
    }
  }
}
```

## Cursor

Cursor Settingsを開き、**Features > MCP Servers**に移動します。**Add new MCP server**をクリックし、以下を入力します：

| フィールド | 値 |
|-----------|---|
| Name | `chainstream` |
| Type | `stdio` |
| Command | `npx @chainstream-io/mcp` |

次に、Cursorの環境設定でAPIキーを設定します：

```
CHAINSTREAM_API_KEY=your-key
```

または、プロジェクトレベルの `.cursor/mcp.json` に追加します：

```json
{
  "mcpServers": {
    "chainstream": {
      "command": "npx",
      "args": ["@chainstream-io/mcp"],
      "env": {
        "CHAINSTREAM_API_KEY": "your-key"
      }
    }
  }
}
```

<Tip>
  CursorはClaude Desktopと同じMCP設定フォーマットを使用します。Claude Desktopの設定が既にある場合は、同じ `mcpServers` ブロックを再利用できます。
</Tip>

## トラブルシューティング

<AccordionGroup>
  <Accordion title="ツールリストにサーバーが表示されない">
    `env` ブロックで `CHAINSTREAM_API_KEY` が正しく設定されていることを確認してください。キーが不足しているか無効な場合、ツール呼び出しはAPIエラーで失敗します。ターミナルでコマンドを手動実行してテストしてください：

    ```bash
    CHAINSTREAM_API_KEY=your-key npx @chainstream-io/mcp
    ```
  </Accordion>
  <Accordion title="npxの起動が遅い">
    `npx` の初回実行時はパッケージのダウンロードに数秒かかることがあります。`npm install -g @chainstream-io/mcp` でグローバルにインストールすると起動が速くなります。その場合、コマンドには `chainstream-mcp` を使用してください。
  </Accordion>
  <Accordion title="macOSでの権限エラー">
    グローバルnpmインストールで権限の問題が発生した場合は、`npx` を使用するか、npmをユーザーレベルのプレフィックスに設定してください：

    ```bash
    mkdir -p ~/.npm-global
    npm config set prefix '~/.npm-global'
    export PATH=~/.npm-global/bin:$PATH
    ```
  </Accordion>
</AccordionGroup>
