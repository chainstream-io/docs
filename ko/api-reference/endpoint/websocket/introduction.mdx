## 개요

WebSocket 엔드포인트는 실시간 데이터 스트리밍 및 ChainStream 거래소에서의 HTTP 요청 전송 대안으로 사용할 수 있습니다. 네트워크별 WebSocket URL은 다음과 같습니다:

* 메인넷: `wss://realtime-dex.chainstream.io/connection/websocket`

## 인증

WebSocket 연결은 URL의 `token` 쿼리 매개변수를 통한 인증이 필요합니다:

```
wss://realtime-dex.chainstream.io/connection/websocket?token=YOUR_ACCESS_TOKEN
```

<Note>
**Go SDK** 또는 **JavaScript SDK**를 사용하는 경우, 인증은 내장되어 자동으로 처리됩니다. 네이티브 WebSocket 라이브러리 또는 기타 서드파티 라이브러리를 사용할 때만 수동으로 token 매개변수를 추가해야 합니다.
</Note>

## 연결

WebSocket API에 연결하려면, 원하는 네트워크에 따른 해당 URL로 WebSocket 연결을 설정하고 URL에 token 매개변수를 포함해야 합니다. 연결이 설정되면, 구독 메시지를 전송하여 실시간 데이터 업데이트를 수신할 수 있습니다.

### 커맨드 라인 예제

`wscat`을 사용하여 연결을 테스트하는 경우:

```bash
$ wscat -c "wss://realtime-dex.chainstream.io/connection/websocket?token=YOUR_ACCESS_TOKEN"
Connected (press CTRL+C to quit)
> { "method": "subscribe", "subscription": { "type": "trades", "coin": "SOL" } }
< {"channel":"subscriptionResponse","data":{"method":"subscribe","subscription":{"type":"trades","coin":"SOL"}}}
```

<Warning>
유효한 `token` 매개변수를 제공하지 않으면 연결이 거부되거나 데이터를 수신할 수 없게 됩니다.
</Warning>

<Note>
이 문서는 메시지 타입 정의에 TypeScript를 사용합니다. Python 사용자의 경우, Python SDK에서 동일한 타입을 찾을 수 있습니다.
</Note>
