ChainStream은 동일한 [데이터 제품](/ko/docs/data-products/overview) 카탈로그를 여러 인터페이스로 제공합니다.

## 비교

| 인터페이스 | 적합한 용도 | 지연 | 인증 |
|:--|:--|:--|:--|
| REST | 특정 값 조회, 스냅샷 | ~100ms | API Key / OAuth / x402 |
| GraphQL | 유연한 분석 쿼리 | ~200ms | API Key / OAuth |
| WebSocket | 실시간 스트림 | 수백 ms | API Key |
| Kafka Streams | 저지연/고신뢰 서버 소비 | 수십 ms | mTLS / SASL |
| SDK (TS/Py/Go/Rust) | 타입 지정된 앱 통합 | REST 기준 | API Key |
| CLI | Shell/스크립트/Agent | REST 기준 | 구성된 Key |
| MCP Server | AI Agent 연결 | REST 기준 | 호스팅/자체 호스팅 |

## 어떤 것을 선택할까

<AccordionGroup>
  <Accordion title="웹/모바일 앱">
    REST 또는 SDK. 스트리밍 UI가 필요하면 WebSocket 추가.
  </Accordion>
  <Accordion title="데이터 파이프라인/웨어하우스">
    Kafka Streams. 백필은 REST 사용.
  </Accordion>
  <Accordion title="트레이딩 봇">
    가격과 체결은 WebSocket, 실행은 REST DeFi 엔드포인트.
  </Accordion>
  <Accordion title="AI Agent">
    [MCP Server](/ko/docs/ai-agents/mcp-server/introduction) 또는 SDK.
  </Accordion>
</AccordionGroup>
