ChainStream MCP Server는 API Reference에 정의된 모든 인터페이스를 지원합니다. AI가 이러한 도구를 직접 호출하여 온체인 데이터를 가져올 수 있습니다.

<Note>
MCP 도구의 파라미터와 반환값은 REST API와 동일합니다. 상세 필드 설명은 [API Reference](/ko/api-reference/authentication/authenticate)를 참조하세요.
</Note>

---

## 도구 개요

  <Tab title="트레이딩 실행">

  </Tab>
  <Tab title="인프라">

  <Tab title="실시간 구독">

    |----------|------|

    | Wallet Balance | 잔액 변동 알림 |
    | Trade Activities | 실시간 거래 스트림 |
    | Pool | 풀 상태 변동 |

## Token 도구

### token_search

토큰을 검색합니다.

```
PEPE 관련 토큰 검색해 줘
```

### token_get

토큰 상세 정보를 조회합니다.

```
토큰 So11111111111111111111111111111111111111112의 상세 정보를 보여 줘
```

### token_get_metadata

토큰 메타데이터(이름, 심볼, 로고 등)를 조회합니다.

```
이 토큰의 로고와 소셜 미디어 링크를 가져와 줘
```

### token_get_stats

토큰 통계(거래량, 매수/매도 횟수 등)를 조회합니다.

```
이 토큰의 지난 24시간 거래 통계를 보여 줘
```

### token_get_price

현재 토큰 가격을 조회합니다.

```
BONK의 현재 가격은 얼마야?
```

### token_get_prices

토큰 가격 이력을 조회합니다.

```
지난 1주일간 ETH 가격 추이를 보여 줘
```

### token_get_candles

토큰 캔들스틱 데이터를 조회합니다.

```
SOL의 최근 1시간봉 100개를 가져와 줘
```

### token_get_holders

토큰 홀더 정보를 조회합니다.

```
이 토큰의 홀더가 몇 명이야?
```

### token_get_top_holders

상위 토큰 홀더 목록을 조회합니다.

```
이 토큰의 상위 10명 홀더를 보여 줘
```

### token_get_pools

토큰의 유동성 풀을 조회합니다.

```
이 토큰의 유동성 풀이 어떤 DEX에 있어?
```

### token_get_security

토큰 보안 검사 결과를 조회합니다.

```
이 토큰이 안전한지 확인해 줘
```

### token_get_creation

토큰 생성 정보를 조회합니다.

```
이 토큰은 언제 생성됐어? 누가 만들었어?
```

### token_get_mint_and_burn

토큰 발행 및 소각 이력을 조회합니다.

```
이 토큰의 발행 및 소각 이력을 보여 줘
```

### token_list

토큰 목록을 조회합니다 (페이지네이션 및 필터 지원).

```
Solana에서 최근 생성된 토큰 목록을 보여 줘
```

### token_get_traders_tag

토큰의 트레이더 태그 분포를 조회합니다.

```
이 토큰을 거래하는 트레이더 유형을 분석해 줘
```

### token_get_liquidity_snapshots

토큰 유동성 스냅샷을 조회합니다.

```
이 토큰의 지난 1주일간 유동성 변화를 보여 줘
```

### token_dev_get_tokens

개발자가 생성한 모든 토큰을 조회합니다.

```
이 개발자가 만든 다른 토큰은 뭐가 있어?
```

## WebSocket 구독

MCP 서비스는 다음 데이터에 대한 WebSocket 실시간 구독도 지원합니다:

### 구독 유형

| 유형 | 설명 | 예시 프롬프트 |
|------|------|-------------|
| Candles | 실시간 가격 캔들 | "이 토큰의 1분봉을 구독해 줘" |
| Token Stats | 실시간 거래 통계 | "이 토큰의 거래 데이터를 실시간으로 모니터링해 줘" |
| New Tokens | 신규 토큰 알림 | "Solana에서 새 토큰이 생기면 알려 줘" |
| Token Metadata | 신규 토큰 메타데이터 | "새 토큰의 상세 정보를 가져와 줘" |
| Wallet Balance | 잔액 변동 | "이 지갑의 잔액 변동을 모니터링해 줘" |
| Wallet Trades | 거래 알림 | "이 지갑이 거래하면 알려 줘" |
| Trade Activities | 실시간 거래 스트림 | "이 토큰의 실시간 거래를 보여 줘" |
| Pool | 풀 상태 | "이 풀의 유동성 변화를 모니터링해 줘" |
| Rankings | 인기 토큰 업데이트 | "인기 토큰 랭킹 실시간 업데이트" |

<Note>
WebSocket 구독 파라미터 및 응답 형식에 대한 자세한 내용은 [WebSocket API 문서](/ko/api-reference/endpoint/websocket/api)를 참조하세요.
</Note>

## 프롬프트 모범 사례

### 체인 지정하기

```
✅ 좋은 예: Ethereum에서 0x...의 잔액을 조회해 줘
❌ 나쁜 예: 0x...의 잔액을 조회해 줘
```

### 전체 주소 제공하기

```
✅ 좋은 예: 0x1234567890abcdef1234567890abcdef12345678을 분석해 줘
❌ 나쁜 예: 0x1234...5678을 분석해 줘
```

### 시간 범위 지정하기

```
✅ 좋은 예: 지난 7일간의 거래 기록을 보여 줘
❌ 나쁜 예: 최근 거래를 보여 줘
```

### 복잡한 작업 분해하기

복잡한 작업은 간단한 질문으로 나누세요:

```
✅ 좋은 예:
1. 먼저 이 주소의 보유 내역을 조회
2. 그다음 거래 이력을 분석
3. 마지막으로 투자 스타일을 판단

❌ 나쁜 예:
이 주소에 대해 전부 다 알려 줘
```

### 후속 질문에 컨텍스트 활용하기

```
사용자: 0x...의 보유 내역을 조회해 줘
AI: [보유 목록 반환]
사용자: 이 중에서 어떤 토큰이 수익률이 제일 높아?
사용자: PEPE를 보유하고 있는 이유를 분석해 줘
```
