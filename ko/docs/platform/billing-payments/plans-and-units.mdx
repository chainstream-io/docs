## 개요

ChainStream은 **Unit 기반 빌링 모델**을 사용합니다. API마다 다른 양의 Unit을 소비하며, 비즈니스 요구에 맞는 플랜을 선택할 수 있습니다.

## Unit 계산

### Unit이란

Unit은 ChainStream의 빌링 단위입니다. API마다 다른 양의 Unit을 소비하며, 더 복잡한 API일수록 더 많이 소비합니다.

### Data API 소비량

Unit 비용은 연산 복잡도에 따라 계층화됩니다. 단순 조회는 저렴하고, 집계가 많은 엔드포인트와 온체인 실행 엔드포인트는 더 비쌉니다.

## WebSocket 실시간 구독 빌링

WebSocket 실시간 데이터 구독은 별도 과금되며, 데이터 바이트 기준으로 과금됩니다:

| 항목 | 요금 | 설명 |
|:--|:--|:--|
| WebSocket 데이터 | **0.005 Unit/byte** | 푸시 데이터 바이트 기준 과금 |
| WebSocket 연결 | 무료 | 연결 수립 시 과금 없음 |
| 하트비트 메시지 | 무료 | 하트비트 패킷 과금 없음 |

### WebSocket 사용량 추정 예시

```
시나리오: 10개 토큰의 실시간 캔들 모니터링 (1분 해상도)

계산:
- 캔들 메시지 약 200바이트
- 각 토큰당 분당 약 1개 캔들 메시지
- 10개 토큰 × 1메시지/분 × 60분 × 24시간 × 30일 = 432,000 메시지
- 총 데이터: 432,000 × 200 = 86,400,000 바이트
- 월간: 86,400,000 × 0.005 = 432,000 Units

시나리오: 50개 토큰의 통계 모니터링 (실시간, 약 1회/초 업데이트)

계산:
- 토큰 통계 메시지 약 500바이트
- 50개 토큰 × 1메시지/초 × 86,400초 × 30일 = 129,600,000 메시지
- 총 데이터: 129,600,000 × 500 = 64,800,000,000 바이트 (~64.8 GB)
- 월간: 64,800,000,000 × 0.005 = 324,000,000 Units (~324M)
```

<Note>
배치 API는 일반적으로 할인이 적용됩니다. 정확한 소비량은 API 레퍼런스를 참고하세요. WebSocket 메시지 빈도는 시장 활동에 따라 다르며, 변동성이 높은 시기에는 메시지 양이 증가합니다.
</Note>

## KYT/KYA 서비스 빌링

<Warning>
KYT/KYA API는 **플랜 Unit을 소비하지 않습니다**. 별도의 KYT 계정 잔액(USD)에서 차감됩니다. 대시보드 → KYT 서비스에서 충전하세요.
</Warning>

| API | 비용 | 엔드포인트 |
|:--|:--|:--|
| Register Transfer | $0.25/호출 | `POST /v2/kyt/transfer` |
| Register Withdrawal | $0.25/호출 | `POST /v2/kyt/withdrawal` |
| Register Address | $1.25/호출 | `POST /v2/kyt/address` |
| Query Risk / Alerts / Exposures | 포함 | `GET /v2/kyt/addresses/*/risk`, `GET /v2/kyt/transfers/*/summary` 등 |

### 대시보드에서 사용량 확인

<Steps>

  </Step>
  <Step title="사용량 확인">
    Usage 페이지에서 현재 월 사용량, 잔여 쿼터, 히스토리 트렌드 확인
  </Step>
</Steps>

### 쿼터 알림 설정

사용량이 임계값에 도달하면 알림을 받도록 쿼터 알림을 설정할 수 있습니다:

| 알림 수준 | 트리거 조건 | 알림 방법 |
|:--|:--|:--|
| 알림 | 50% 사용 | 이메일 |
| 경고 | 80% 사용 | 이메일 + 대시보드 알림 |
| 위험 | 100% 사용 | 이메일 + 대시보드 알림 |

<Tip>
예상치 못한 서비스 중단을 방지하려면 최소 80% 알림을 활성화하는 것을 추천합니다.
</Tip>

## 쿼터 소진 처리

### 쿼터가 소진되면 어떻게 되나요

월간 쿼터가 소진되면 API는 403 오류를 반환하며 계속 호출할 수 없습니다. 유료 플랜 사용자는 수동으로 추가 쿼터를 구매하여 서비스를 복원할 수 있습니다.

| 플랜 | 처리 방법 |
|:--|:--|
| Free | API가 403 오류 반환, 다음 달 쿼터 리셋 대기 |
| Starter / Pro / Enterprise | API가 403 오류 반환, 추가 쿼터 구매로 서비스 복원 가능 |
| Custom | 계약 조건에 따름 |

### 추가 쿼터 구매

유료 플랜 사용자는 대시보드에서 추가 쿼터를 구매할 수 있으며, 플랜의 초과 요금에 따라 과금됩니다:

| 플랜 | 추가 쿼터 가격 |
|:--|:--|
| Starter | $12.9 / 1M Units |
| Pro | $9.9 / 1M Units |
| Enterprise | $6.9 / 1M Units |

<Warning>
쿼터 소진 시 서비스가 즉시 중단됩니다 (403 오류 반환). 80% 사용량 알림을 설정하고 사전에 추가 쿼터를 구매하여 비즈니스 영향을 방지하는 것을 추천합니다.
</Warning>

## AI 에이전트 결제 (x402 / MPP)

대시보드 가입 없이 프로그래밍 방식의 접근이 필요한 AI 에이전트를 위해, ChainStream은 두 가지 온체인 결제 프로토콜을 지원합니다:

### x402 (Base / Solana USDC)

[x402 프로토콜](https://www.x402.org/)은 HTTP 네이티브 마이크로결제를 가능하게 합니다. API 호출이 402를 반환하면, 에이전트의 지갑이 USDC 전송에 서명하고 재시도합니다 — 계정 등록이 필요 없습니다.

| 속성 | 값 |
|----------|-------|
| 네트워크 | Base (chain ID 8453), Solana |
| 통화 | USDC |
| 표준 | EIP-3009 (Base), 네이티브 전송 (Solana) |
| 가스 수수료 | **무료** — ChainStream의 facilitator가 모든 가스 비용을 부담 |

<Tip>
USDC로 구독 가격만 지불하면 됩니다. ChainStream은 자체 x402 facilitator를 운영하여 온체인 트랜잭션을 제출하고 대신 가스 수수료를 지불합니다 — 에이전트 지갑에 ETH나 SOL이 필요 없습니다.
</Tip>

### MPP (Tempo USDC.e)

MPP는 Tempo 네트워크를 사용하여 결제합니다. 가스는 USDC.e로 직접 지불되며 — ETH가 필요 없습니다.

| 속성 | 값 |
|----------|-------|
| 네트워크 | Tempo (chain ID 4217) |
| 통화 | USDC.e |

### 동작 원리

1. 에이전트가 ChainStream API 호출
2. 서버가 402 반환 (활성 구독 없음)
3. 에이전트가 `/x402/pricing`에서 사용 가능한 플랜 조회
4. 에이전트가 x402 또는 MPP를 통해 플랜 구매
5. 서버가 **API Key**를 반환 — 에이전트가 향후 모든 호출에 사용

## FAQ

<AccordionGroup>
  <Accordion title="올바른 플랜은 어떻게 선택하나요?">
    실제 사용량을 평가하기 위해 Free 플랜부터 시작하고, 월간 소비량에 따라 선택하는 것을 추천합니다:
    - &lt; 30K Units: Free
    - 30K - 10M Units: Starter
    - 10M - 50M Units: Pro
    - &gt; 50M Units: Enterprise
  </Accordion>
  <Accordion title="언제든 업그레이드/다운그레이드할 수 있나요?">
    예. 업그레이드는 즉시 적용되며 비례 과금됩니다. 다운그레이드는 다음 빌링 주기에 적용됩니다.
  </Accordion>
  <Accordion title="미사용 쿼터는 이월되나요?">
    아니오. 월간 쿼터는 매월 말에 리셋되며 이월되지 않습니다.
  </Accordion>
  <Accordion title="인보이스는 어떻게 받나요?">
    대시보드 → Billing → Invoices에서 과거 인보이스를 다운로드하세요. Enterprise 고객은 영업팀에 커스텀 인보이스를 요청할 수 있습니다.
  </Accordion>
</AccordionGroup>
