## 블록체인 용어

기본적인 블록체인 개념 및 일반 용어입니다.

### A

<Accordion title="Address">
블록체인에서 자산을 송수신하는 데 사용되는 고유 식별자입니다. Ethereum 주소는 `0x`로 시작하며 42자입니다.

예시: `0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045`
</Accordion>

<Accordion title="ABI (Application Binary Interface)">
스마트 컨트랙트의 인터페이스 정의로, 컨트랙트와 상호작용하기 위한 함수 및 데이터 구조를 설명합니다.
</Accordion>

### B

<Accordion title="Block">
블록체인에서 트랜잭션 데이터를 저장하는 기본 단위입니다. 각 블록에는 트랜잭션 집합, 타임스탬프, 이전 블록의 해시가 포함됩니다.

| 필드 | 설명 |
|------|------|
| Block Number | 블록 높이 |
| Block Hash | 블록 해시 |
| Timestamp | 블록 시간 |
| Transactions | 포함된 트랜잭션 |
</Accordion>

<Accordion title="Block Confirmation">
트랜잭션이 블록에 포함된 후 생성된 후속 블록의 수입니다. 확인 수가 많을수록 트랜잭션이 더 불가역적입니다.

| 체인 | 권장 확인 수 |
|------|-------------|
| Ethereum | 12-32 |
| BSC | 15 |
| Solana | 32 |
</Accordion>

<Accordion title="Bridge">
서로 다른 블록체인 간에 자산 및 데이터를 전송할 수 있게 하는 프로토콜입니다. 일반적인 브릿지에는 LayerZero, Wormhole 등이 있습니다.
</Accordion>

### C

<Accordion title="Chain">
블록체인 네트워크입니다. ChainStream은 Ethereum, BSC, Polygon, Arbitrum, Solana 등 다수의 체인을 지원합니다.

체인 식별자 예시:
- `ethereum` - Ethereum 메인넷
- `bsc` - BNB Chain
- `arbitrum` - Arbitrum One
- `solana` - Solana
</Accordion>

<Accordion title="Contract Address">
스마트 컨트랙트가 블록체인에 배포된 고유 주소입니다. 일반 주소(EOA)와 달리 컨트랙트 주소는 코드에 의해 제어됩니다.
</Accordion>

### E

<Accordion title="EOA (Externally Owned Account)">
프라이빗 키로 제어되는 계정으로, 스마트 컨트랙트 주소와 구별됩니다.
</Accordion>

<Accordion title="ERC-20">
Ethereum 토큰 표준입니다. `transfer`, `approve`, `balanceOf` 등의 기본 토큰 함수 인터페이스를 정의합니다.
</Accordion>

<Accordion title="ERC-721">
Ethereum NFT 표준입니다. 대체 불가능한 토큰의 인터페이스 사양을 정의합니다.
</Accordion>

<Accordion title="EVM (Ethereum Virtual Machine)">
스마트 컨트랙트 코드를 실행하는 런타임 환경입니다. Polygon, Arbitrum, Base는 모두 EVM 호환 체인입니다.
</Accordion>

### G

<Accordion title="Gas">
블록체인 트랜잭션을 실행하는 데 필요한 수수료 단위입니다. Gas Fee = Gas Used × Gas Price.

| 용어 | 설명 |
|------|------|
| Gas Limit | 트랜잭션의 최대 가스 소비량 |
| Gas Used | 실제 소비된 가스 |
| Gas Price | 가스 단위당 가격 |
| Gwei | Gas Price 단위, 1 Gwei = 10⁻⁹ ETH |
</Accordion>

### H

<Accordion title="Hash">
임의 길이의 데이터를 고정 길이 문자열로 변환하는 암호화 함수입니다. 블록체인에서는 트랜잭션 해시, 블록 해시 등에 일반적으로 사용됩니다.

예시: `0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef`
</Accordion>

### L

<Accordion title="Layer 1 (L1)">
1계층 블록체인입니다. Ethereum, Solana, Bitcoin과 같은 독립적인 메인 체인입니다.
</Accordion>

<Accordion title="Layer 2 (L2)">
2계층 스케일링 솔루션입니다. L1 위에 구축된 확장 솔루션으로, Arbitrum, Optimism, Base, zkSync 등이 있습니다.
</Accordion>

### M

<Accordion title="Mempool">
대기 중인 트랜잭션의 스테이징 영역입니다. 트랜잭션은 블록에 포함되기 전에 멤풀에서 대기합니다.
</Accordion>

### N

<Accordion title="NFT (Non-Fungible Token)">
고유한 디지털 자산으로, 디지털 아트, 게임 아이템 등에 일반적으로 사용됩니다.
</Accordion>

<Accordion title="Node">
블록체인 소프트웨어를 실행하는 컴퓨터로, 네트워크 검증 및 데이터 저장에 참여합니다.
</Accordion>

<Accordion title="Nonce">
트랜잭션 시퀀스 번호입니다. 각 주소에서 트랜잭션을 보낼 때 증가하는 카운터로, 리플레이 공격을 방지합니다.
</Accordion>

### R

<Accordion title="RPC (Remote Procedure Call)">
블록체인 노드와 상호작용하기 위한 프로토콜로, 트랜잭션 전송, 데이터 조회 등에 사용됩니다.
</Accordion>

### S

<Accordion title="Smart Contract">
블록체인에서 자동으로 실행되는 프로그램 코드입니다. 배포된 후에는 코드 로직을 변경할 수 없습니다(업그레이드 메커니즘이 설계되어 있지 않는 한).
</Accordion>

### T

<Accordion title="Token">
블록체인의 디지털 자산입니다. 네이티브 토큰(ETH 등)과 컨트랙트 토큰(USDC 등)이 있습니다.
</Accordion>

<Accordion title="Transaction">
이체, 컨트랙트 호출 등 블록체인의 상태 변경 작업입니다.

| 필드 | 설명 |
|------|------|
| TxHash | 트랜잭션 해시, 고유 식별자 |
| From | 발신자 주소 |
| To | 수신자 주소 |
| Value | 이체 금액 |
| Input Data | 컨트랙트 호출 데이터 |
</Accordion>

<Accordion title="Transaction Hash / TxHash">
트랜잭션의 고유 식별자입니다. 블록 익스플로러에서 트랜잭션 상세 정보를 조회하는 데 사용할 수 있습니다.
</Accordion>

### W

<Accordion title="Wallet">
암호화 자산을 저장하고 관리하는 도구입니다. 소프트웨어(MetaMask 등) 또는 하드웨어(Ledger 등) 형태가 있습니다.
</Accordion>

## DeFi 용어

탈중앙화 금융 관련 개념입니다.

### A

<Accordion title="AMM (Automated Market Maker)">
DEX에서 유동성 공급 및 가격 책정을 위한 알고리즘 메커니즘으로, Uniswap의 상수곱 공식 `x * y = k`가 대표적입니다.
</Accordion>

<Accordion title="APR (Annual Percentage Rate)">
복리를 고려하지 않은 연간 이자율입니다.
</Accordion>

<Accordion title="APY (Annual Percentage Yield)">
복리를 고려한 연간 수익률입니다.

공식: `APY = (1 + APR/n)^n - 1`, 여기서 n은 복리 기간의 수입니다.
</Accordion>

<Accordion title="ATH (All-Time High)">
토큰이나 자산이 기록한 역대 최고 가격입니다.
</Accordion>

<Accordion title="ATL (All-Time Low)">
토큰이나 자산이 기록한 역대 최저 가격입니다.
</Accordion>

### C

<Accordion title="CEX (Centralized Exchange)">
중앙화 기관이 운영하는 암호화폐 거래 플랫폼입니다.

예시: Binance, Coinbase, OKX, Bybit
</Accordion>

<Accordion title="Collateral">
대출 프로토콜에서 대출을 담보하는 데 사용되는 자산입니다. 담보 비율은 일반적으로 100%를 초과해야 합니다.
</Accordion>

### D

<Accordion title="DEX (Decentralized Exchange)">
중앙화 기관이 필요 없이 스마트 컨트랙트를 통해 자동으로 거래를 실행하는 거래 플랫폼입니다.

예시: Uniswap, SushiSwap, Raydium, Jupiter
</Accordion>

<Accordion title="dApp (Decentralized Application)">
블록체인에서 실행되는 애플리케이션입니다.
</Accordion>

<Accordion title="DeFi (Decentralized Finance)">
거래, 대출, 파생상품 등을 포함하는 블록체인 기반 금융 서비스 및 애플리케이션입니다.
</Accordion>

### F

<Accordion title="Farming / Yield Farming">
DeFi 프로토콜에 유동성을 제공하여 토큰 보상을 획득하는 행위입니다.
</Accordion>

<Accordion title="FDV (Fully Diluted Valuation)">
모든 토큰이 발행되었다고 가정한 총 시장 가치입니다.

공식: `FDV = Token Price × Total Supply`
</Accordion>

<Accordion title="Flash Loan">
동일한 블록/트랜잭션 내에서 상환해야 하는 무담보 즉시 대출입니다. 주로 차익거래, 청산 등에 사용됩니다.
</Accordion>

### H

<Accordion title="Health Factor">
대출 프로토콜에서 포지션 안전도를 측정하는 지표입니다. Health Factor가 1 미만이면 포지션이 청산될 수 있습니다.
</Accordion>

### I

<Accordion title="Impermanent Loss">
가격 변동으로 인해 유동성 공급자가 단순 보유 대비 발생하는 손실입니다.

| 가격 변동 | 비영구적 손실 |
|-----------|--------------|
| ±25% | -0.6% |
| ±50% | -2.0% |
| ±100% | -5.7% |
</Accordion>

### L

<Accordion title="Liquidation">
대출 포지션의 담보 비율이 임계값 이하로 떨어지면 프로토콜이 자동으로 담보를 매각하여 부채를 상환하는 프로세스입니다.
</Accordion>

<Accordion title="Liquidity">
시장에서 거래 가능한 자산의 양입니다. 유동성이 높으면 대규모 거래도 가격에 큰 영향을 미치지 않습니다.
</Accordion>

<Accordion title="Liquidity Pool">
DEX에서 거래 쌍 자산을 저장하는 스마트 컨트랙트로, 거래를 위한 유동성을 제공합니다.
</Accordion>

<Accordion title="LP (Liquidity Provider)">
DEX 풀에 자산을 제공하고 거래 수수료를 보상으로 받는 사용자입니다.
</Accordion>

<Accordion title="LP Token">
유동성 풀에서 사용자의 지분을 나타내는 토큰으로, 자산 인출이나 마이닝 스테이킹에 사용할 수 있습니다.
</Accordion>

### M

<Accordion title="Market Cap">
토큰의 총 시장 가치입니다.

공식: `Market Cap = Token Price × Circulating Supply`
</Accordion>

<Accordion title="MEV (Maximal Extractable Value)">
채굴자 또는 검증자가 트랜잭션 순서 조정, 삽입, 검열을 통해 추출하는 추가 가치입니다.

일반적인 MEV 유형:
- Sandwich Attack
- Front-running
- Back-running
- Liquidation
</Accordion>

### O

<Accordion title="Oracle">
Chainlink, Pyth와 같이 오프체인 데이터를 스마트 컨트랙트에 제공하는 서비스입니다. 주로 가격 데이터에 사용됩니다.
</Accordion>

### S

<Accordion title="Slippage">
예상 거래 가격과 실제 체결 가격 간의 차이입니다. 유동성이 낮은 풀에서 더 크게 발생할 수 있습니다.
</Accordion>

<Accordion title="Staking">
보상 획득이나 거버넌스 참여를 위해 프로토콜에 토큰을 잠그는 행위입니다.
</Accordion>

<Accordion title="Swap">
DEX에서의 토큰 교환 작업입니다.
</Accordion>

### T

<Accordion title="TVL (Total Value Locked)">
DeFi 프로토콜에 잠긴 자산의 총 가치로, 프로토콜 규모를 측정하는 중요한 지표입니다.
</Accordion>

### W

<Accordion title="Whale">
대량의 암호화 자산을 보유한 주소 또는 투자자입니다. 일반적으로 토큰 공급량의 1% 이상을 보유한 주소를 지칭합니다.
</Accordion>

### Y

<Accordion title="Yield">
DeFi 프로토콜 투자로 얻는 수익으로, 거래 수수료 배분, 토큰 보상, 이자 등이 포함됩니다.
</Accordion>

## ChainStream 용어

ChainStream 제품 및 서비스 관련 개념입니다.

### A

<Accordion title="Access Token">
API 요청을 인증하는 데 사용되는 JWT(JSON Web Token)입니다.

**획득 방법**:
1. 대시보드에서 앱을 생성하여 Client ID와 Client Secret을 받습니다
2. Client ID와 Client Secret을 사용하여 인증 서비스를 호출해 Access Token을 생성합니다
3. API 요청 헤더에 `Authorization: Bearer <Access Token>`을 사용합니다

**인증 도메인**: `dex.asia.auth.chainstream.io`

**스코프**:
| 스코프 | 설명 |
|--------|------|
| `webhook.read` | Webhook API 읽기 권한 |
| `webhook.write` | Webhook API 쓰기 권한 |
| `kyt.read` | KYT API 읽기 권한 |
| `kyt.write` | KYT API 쓰기 권한 |

보안 팁:
- 클라이언트 측 코드에 Client Secret을 노출하지 마세요
- 인증 정보를 버전 관리에 커밋하지 마세요
- 민감한 정보는 환경 변수에 저장하세요
- Access Token은 만료되므로 주기적으로 갱신하세요

[인증 문서](/ko/api-reference/authentication/authenticate) 참조
</Accordion>

### C

<Accordion title="CU (Compute Unit)">
ChainStream에서 API 요청 복잡도를 측정하는 단위입니다. 엔드포인트마다 소비하는 CU가 다릅니다.
</Accordion>

### K

<Accordion title="KYA (Know Your Address)">
ChainStream의 주소 위험 평가 서비스로, 주소의 과거 행동 및 연관 위험을 분석합니다.
</Accordion>

<Accordion title="KYT (Know Your Transaction)">
ChainStream의 트랜잭션 리스크 관리 서비스로, 트랜잭션 자금 출처 및 위험 수준을 평가합니다.

위험 수준:

| 수준 | 설명 |
|------|------|
| Low | 저위험, 자동 처리 가능 |
| Medium | 중위험, 수동 검토 권장 |
| High | 고위험, 차단 권장 |
| Severe | 심각한 위험, 반드시 차단 |
</Accordion>

### R

<Accordion title="Rate Limit">
API 요청 빈도 제한입니다. ChainStream은 구독 등급에 따라 다른 속도 제한을 제공합니다.

응답 헤더:
- `X-RateLimit-Limit` - 총 제한
- `X-RateLimit-Remaining` - 남은 횟수
- `X-RateLimit-Reset` - 리셋 시간
</Accordion>

### S

<Accordion title="Smart Money">
ChainStream이 식별한, 지속적으로 시장 평균 이상의 수익을 달성하는 주소입니다.

유형:
- Top Traders
- Early Investors
- Institutional Wallets
- KOL Wallets

[Smart Money 방법론](/ko/docs/data-products/smart-money) 참조
</Accordion>

<Accordion title="Smart Score">
ChainStream의 지갑 주소 종합 점수로, 거래 능력 및 과거 성과를 반영합니다.

| 점수 범위 | 수준 |
|-----------|------|
| 90-100 | Top Smart Money |
| 75-89 | Quality Trader |
| 60-74 | Good Trader |
| &lt; 60 | Average Trader |
</Accordion>

### W

<Accordion title="Webhook">
이벤트가 트리거되면 지정된 URL로 알림을 보내는 메커니즘입니다.

**지원 이벤트 유형**:
| 이벤트 | 설명 |
|--------|------|
| `sol.token.created` | Solana에서 새 토큰 생성 |
| `sol.token.migrated` | Solana에서 토큰 졸업(Bonding Curve에서 DEX로 마이그레이션) |

<Note>
더 많은 이벤트 유형이 개발 중입니다.
</Note>

**관리 엔드포인트**:
- `POST /v1/webhook/endpoint` - Webhook 생성
- `GET /v1/webhook/endpoint` - Webhook 목록
- `DELETE /v1/webhook/endpoint/{id}` - Webhook 삭제

[Webhook 기초](/ko/docs/recipes/webhook-fundamentals) 참조
</Accordion>

<Accordion title="WebSocket">
실시간 양방향 통신을 지원하는 네트워크 프로토콜입니다. ChainStream은 WebSocket을 통해 실시간 데이터 스트림을 제공합니다.

**엔드포인트**: `wss://realtime-dex.chainstream.io/connection/websocket`

**연결 형식**:
```
wss://realtime-dex.chainstream.io/connection/websocket?token=YOUR_ACCESS_TOKEN
```

**지원 구독 유형**:
- Token candles (`dex-token-candles`)
- Token stats (`dex-token-stats`)
- Token holders (`token-holders`)
- New tokens (`dex-new-token`)
- Wallet balance (`wallet-balance`)
- Wallet trades (`wallet-trade`)

<Note>
SDK 사용 시 인증은 자동으로 처리됩니다.
</Note>

[WebSocket API](/ko/api-reference/endpoint/websocket/introduction) 참조
</Accordion>

## AI 용어

AI 및 MCP 관련 개념입니다.

### A

<Accordion title="Agent">
자율적으로 작업을 수행할 수 있는 AI 시스템입니다. AI Agent는 도구를 사용하고, 외부 데이터에 접근하고, 의사결정을 내릴 수 있습니다.
</Accordion>

<Accordion title="Agent Skills">
AI 코딩 어시스턴트에게 특정 기능 사용 방법을 알려주는 구조화된 명령어 패키지(`SKILL.md` 파일)입니다. ChainStream은 두 가지 스킬을 제공합니다: `chainstream-data`(읽기 전용 데이터 조회)와 `chainstream-defi`(DeFi 실행). 스킬에는 의사결정 트리, 워크플로, 안전 규칙, 오류 복구 등 AI 에이전트가 자율적으로 운영하는 데 필요한 모든 것이 포함됩니다.

[Agent Skills 소개](/ko/docs/ai-agents/agent-skills/introduction) 참조
</Accordion>

### C

<Accordion title="Context">
대화 이력, 시스템 프롬프트, 외부 데이터 등 AI 모델이 응답을 생성할 때 참조하는 정보입니다.
</Accordion>

<Accordion title="Context Window">
AI 모델이 한 번에 처리할 수 있는 최대 토큰 수입니다. 컨텍스트 윈도우가 클수록 더 많은 정보를 처리할 수 있습니다.
</Accordion>

### L

<Accordion title="LLM (Large Language Model)">
Claude, GPT 등 대량의 텍스트 데이터로 학습된 AI 모델입니다.
</Accordion>

### M

<Accordion title="MCP (Model Context Protocol)">
Anthropic이 개발한 오픈 프로토콜로, AI 모델이 외부 데이터 소스 및 도구와 상호작용할 수 있게 합니다.

ChainStream MCP Server는 이 프로토콜을 구현하여 AI가 온체인 데이터를 직접 조회할 수 있게 합니다.

[MCP 소개](/ko/docs/ai-agents/mcp-server/introduction) 참조
</Accordion>

<Accordion title="MCP Server">
MCP 프로토콜을 구현한 서버로, 도구와 데이터를 제공합니다. ChainStream MCP Server는 온체인 데이터 조회 기능을 제공합니다.
</Accordion>

<Accordion title="MCP Client">
Claude Desktop, Cursor 등 MCP 프로토콜을 사용하여 도구를 호출하는 애플리케이션입니다.
</Accordion>

<Accordion title="MPP (Micropayment Protocol)">
Tempo 네트워크(체인 ID 4217)를 사용하여 USDC.e로 결제하는 결제 프로토콜입니다. Tempo에서 결제를 선호하는 AI 에이전트를 위한 x402의 대안입니다. 가스는 USDC.e로 직접 지불되며 — ETH가 필요 없습니다.

[x402 결제](/ko/docs/platform/billing-payments/x402-payments) 참조
</Accordion>

### P

<Accordion title="Prompt">
AI 모델에 주어지는 입력 텍스트로, 모델이 예상된 출력을 생성하도록 안내합니다.
</Accordion>

### T

<Accordion title="SKILL.md">
Agent Skills의 표준 파일 형식입니다. YAML 프론트매터(이름, 설명)와 의사결정 트리, 워크플로, 안전 규칙, 참조를 포함하는 구조화된 마크다운으로 구성됩니다. Cursor, Claude Code, Codex 등의 AI 코딩 어시스턴트가 사용합니다.
</Accordion>

<Accordion title="TEE (Trusted Execution Environment)">
코드와 데이터의 기밀성 및 무결성을 보장하는 하드웨어 수준의 보안 보호를 제공하는 컴퓨팅 환경입니다. ChainStream CLI는 지갑 키 관리에 TEE를 사용합니다 — 프라이빗 키는 보안 인클레이브를 벗어나지 않습니다.

[CLI 인증](/ko/docs/platform/authentication/api-keys-oauth) 참조
</Accordion>

<Accordion title="Token (AI)">
AI 모델이 텍스트를 처리하는 기본 단위입니다. 하나의 토큰은 약 4개의 영어 문자 또는 1-2개의 한국어 문자에 해당합니다.

<Note>
이 Token은 AI 토큰을 의미하며, 블록체인 토큰과는 다릅니다.
</Note>
</Accordion>

<Accordion title="Tool">
MCP에서 정의된 함수로, AI가 호출할 수 있습니다. ChainStream이 제공하는 도구에는 `get_token_balance`, `analyze_wallet` 등이 있습니다.
</Accordion>

### X

<Accordion title="x402">
[Coinbase](https://github.com/coinbase/x402)가 HTTP 402(Payment Required) 기반으로 구축한 오픈 결제 프로토콜입니다. AI Agent가 Base 네트워크에서 USDC 스테이블코인을 사용하여 API 호출당 결제할 수 있게 합니다. ChainStream은 x402를 인증 방식으로 지원합니다 — API 키가 없는 에이전트도 서명된 온체인 인가(EIP-3009 또는 Permit2)를 통해 각 도구 호출에 대해 결제할 수 있으며, Facilitator 서비스를 통해 검증됩니다. 계정 등록이 필요 없습니다.
</Accordion>
