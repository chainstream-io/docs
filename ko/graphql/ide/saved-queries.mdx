## 개요

GraphQL IDE를 사용하면 나중에 빠르게 접근할 수 있도록 쿼리를 브라우저에 저장할 수 있습니다. 저장된 쿼리는 브라우저 세션 간에 유지되며 사이드바의 **Saved** 탭에 표시되어 외부 도구 없이도 자주 사용하는 쿼리의 개인 라이브러리를 만들 수 있습니다.

---

## 쿼리 저장하기

<Steps>
  <Step title="쿼리 작성 또는 불러오기">
    에디터에 쿼리를 입력하거나 [템플릿](/ko/graphql/ide/query-templates)에서 불러옵니다.
  </Step>
  <Step title="Save Query 클릭">
    툴바에서 **Save Query** 버튼을 클릭합니다. 두 개의 필드가 있는 대화 상자가 나타납니다:
    - **Title** (필수) — 쿼리에 대한 설명적인 이름
    - **Tags** (선택) — 정리를 위한 쉼표로 구분된 라벨 (예: `solana, dex, monitoring`)
  </Step>
  <Step title="확인">
    **Save**를 클릭합니다. 쿼리가 로컬에 저장되고 사이드바 **Saved** 탭에 즉시 나타납니다.
  </Step>
</Steps>

<Tip>
체인과 Cube 이름이 포함된 설명적인 제목을 사용하세요. 예: "Solana — Latest Raydium Trades" 또는 "ETH — USDC Top Holders". 컬렉션이 늘어날수록 쿼리를 쉽게 찾을 수 있습니다.
</Tip>

---

## 저장된 쿼리 불러오기

1. 사이드바를 열기 (**Ctrl/Cmd+B**)
2. **Saved** 탭으로 전환
3. 저장된 쿼리를 클릭하여 에디터에 로드

쿼리 텍스트, 변수, 헤더가 모두 쿼리가 저장된 시점의 상태로 복원됩니다.

---

## 저장된 쿼리 삭제

저장된 쿼리를 삭제하려면:

1. 사이드바를 열고 **Saved** 탭으로 전환
2. 삭제할 쿼리 위에 마우스를 올리면 **delete** 아이콘이 나타남
3. 삭제 아이콘을 클릭하여 쿼리 제거

<Note>
삭제는 영구적입니다. 실행 취소가 없습니다 — 삭제된 쿼리는 복구할 수 없습니다.
</Note>

---

## 저장소 세부 정보

| 속성 | 세부 사항 |
|:--|:--|
| **저장 메커니즘** | 브라우저 `localStorage` |
| **지속성** | 페이지 새로고침 및 브라우저 재시작 후에도 유지 |
| **범위** | 브라우저별, 오리진별 (동일 도메인) |
| **동기화** | 기기 또는 브라우저 간 동기화되지 않음 |
| **삭제** | 브라우저 데이터 또는 `localStorage` 삭제 시 모든 저장된 쿼리 제거 |

<Info>
저장된 쿼리는 브라우저의 `localStorage`에 완전히 저장됩니다. 서버로 전송되지 **않으며** 다른 기기에서 접근할 수 없습니다. 쿼리를 공유해야 하는 경우 [코드 내보내기](/ko/graphql/ide/code-export) 기능을 사용하여 이동 가능한 스니펫을 생성하세요.
</Info>

## 태그로 정리하기

태그는 컬렉션이 늘어남에 따라 저장된 쿼리를 분류하고 필터링하는 데 도움이 됩니다. 권장하는 태깅 전략:

<AccordionGroup>
  <Accordion title="체인별">
    `solana`, `ethereum`, `bsc` 같은 체인별 태그를 사용하여 특정 네트워크의 쿼리를 빠르게 필터링합니다.
  </Accordion>
  <Accordion title="Cube별">
    Cube 이름으로 쿼리에 태그를 지정합니다: `dextrades`, `transfers`, `ohlc`, `holders` 등.
  </Accordion>
  <Accordion title="사용 사례별">
    `monitoring`, `analysis`, `debug`, `dashboard` 같은 기능적 태그로 쿼리의 목적을 나타냅니다.
  </Accordion>
  <Accordion title="토큰 또는 프로젝트별">
    특정 프로젝트 이름이나 토큰 심볼로 태그합니다: `usdc`, `bonk`, `raydium`, `uniswap`.
  </Accordion>
</AccordionGroup>

<Tip>
쿼리당 태그 수에 제한이 없습니다. 여러 태깅 차원을 결합하세요 — 예: `solana, dextrades, raydium, monitoring` — 최대한의 검색성을 위해.
</Tip>

## 모범 사례

- **반복적으로 저장** — 쿼리를 다듬으면서 작동하는 버전을 저장하세요. 필요한 경우 "OHLC v1", "OHLC v2"처럼 버전이 포함된 제목을 사용하세요.
- **정기적으로 정리** — 더 이상 사용하지 않는 쿼리를 삭제하여 사이드바를 관리하기 쉽게 유지하세요.
- **삭제 전 내보내기** — 브라우저 데이터를 삭제할 계획이라면 [코드 내보내기](/ko/graphql/ide/code-export)를 통해 중요한 쿼리를 먼저 코드 스니펫으로 내보내세요.
- **관심사별 하나의 쿼리** — 너무 많은 작업을 수행하는 거대한 쿼리를 저장하지 마세요. 작고 집중된 쿼리가 재사용하고 조합하기 더 쉽습니다.
