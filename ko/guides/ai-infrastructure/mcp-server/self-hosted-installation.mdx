## 패키지

ChainStream MCP 서버는 npm 패키지로 배포됩니다:

```
@chainstream-io/mcp
```

## 설치

<Tabs>
  <Tab title="글로벌 설치">
    패키지를 글로벌로 설치하여 `chainstream-mcp` 바이너리를 시스템 전체에서 사용할 수 있도록 합니다:

    ```bash
    npm install -g @chainstream-io/mcp
    ```

    설치 후 서버를 직접 실행합니다:

    ```bash
    chainstream-mcp
    ```
  </Tab>
  <Tab title="npx (설치 불필요)">
    영구 설치 없이 MCP 서버를 바로 실행합니다:

    ```bash
    npx @chainstream-io/mcp
    ```

    매번 최신 버전을 다운로드하여 실행합니다. 빠른 테스트에 적합합니다.
  </Tab>
</Tabs>

## 바이너리

이 패키지는 하나의 바이너리를 제공합니다:

```
chainstream-mcp
```

이 바이너리는 기본적으로 stdio 전송 방식으로 MCP 서버를 시작합니다 — Claude Desktop, Cursor 및 기타 MCP 클라이언트에 연결하기 위한 표준 방식입니다. 클라이언트별 설정은 [Stdio 전송](/ko/guides/ai-infrastructure/mcp-server/stdio-transport)을 참조하세요.

## 환경 변수

| 변수 | 필수 | 기본값 | 설명 |
|------|------|--------|------|
| `CHAINSTREAM_API_KEY` | 예 | — | ChainStream API 키 |
| `CHAINSTREAM_API_URL` | 아니오 | `https://api.chainstream.io` | ChainStream API 기본 URL |

API 키를 환경 변수로 설정하거나 커맨드라인에서 `--api-key`를 전달합니다:

```bash
export CHAINSTREAM_API_KEY=your-key
chainstream-mcp

# 또는 직접 전달
chainstream-mcp --api-key your-key
```

<Warning>
  API 키를 안전하게 보관하세요. 버전 관리 시스템에 커밋하지 마세요.
</Warning>

## 설치 확인

설치 후 패키지가 사용 가능한지 확인합니다:

```bash
npx @chainstream-io/mcp --help
```

사용 가능한 옵션이 표시됩니다:

```
Options:
  --transport  Transport type: stdio for local CLI   [choices: "stdio"] [default: "stdio"]
  --api-key    ChainStream API key                   [string]
  --help       Show help                             [boolean]
  --version    Show version number                   [boolean]
```
