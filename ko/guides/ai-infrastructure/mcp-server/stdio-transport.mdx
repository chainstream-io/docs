## 개요

Stdio는 ChainStream MCP 서버의 **기본 전송 방식**입니다. 표준 입출력으로 통신하며, 로컬 IDE 및 데스크톱 통합에 가장 적합한 방식입니다.

```bash
chainstream-mcp
```

또는 글로벌 설치 없이:

```bash
npx @chainstream-io/mcp
```

`--transport`를 지정하지 않으면 자동으로 stdio가 사용되므로, 추가 플래그가 필요 없습니다.

## Claude Desktop

Claude Desktop MCP 설정 파일에 다음을 추가합니다:

<Tabs>
  <Tab title="macOS">
    `~/Library/Application Support/Claude/claude_desktop_config.json`을 편집합니다:

    ```json
    {
      "mcpServers": {
        "chainstream": {
          "command": "npx",
          "args": ["@chainstream-io/mcp"],
          "env": {
            "CHAINSTREAM_API_KEY": "your-key"
          }
        }
      }
    }
    ```
  </Tab>
  <Tab title="Windows">
    `%APPDATA%\Claude\claude_desktop_config.json`을 편집합니다:

    ```json
    {
      "mcpServers": {
        "chainstream": {
          "command": "npx",
          "args": ["@chainstream-io/mcp"],
          "env": {
            "CHAINSTREAM_API_KEY": "your-key"
          }
        }
      }
    }
    ```
  </Tab>
</Tabs>

패키지를 글로벌로 설치한 경우, 바이너리를 직접 사용할 수 있습니다:

```json
{
  "mcpServers": {
    "chainstream": {
      "command": "chainstream-mcp",
      "env": {
        "CHAINSTREAM_API_KEY": "your-key"
      }
    }
  }
}
```

설정을 저장한 후 Claude Desktop을 재시작하세요. ChainStream 도구가 도구 목록에 자동으로 나타납니다.

## Claude Code

CLI를 사용하여 Claude Code에 MCP 서버를 추가합니다:

```bash
claude mcp add chainstream -- npx @chainstream-io/mcp
```

Claude Code를 실행하기 전에 API 키를 환경 변수로 설정합니다:

```bash
export CHAINSTREAM_API_KEY=your-key
```

또는 프로젝트의 `.mcp.json`에 서버 설정을 추가합니다:

```json
{
  "mcpServers": {
    "chainstream": {
      "command": "npx",
      "args": ["@chainstream-io/mcp"],
      "env": {
        "CHAINSTREAM_API_KEY": "your-key"
      }
    }
  }
}
```

## Cursor

Cursor Settings를 열고 **Features > MCP Servers**로 이동합니다. **Add new MCP server**를 클릭하고 다음을 입력합니다:

| 필드 | 값 |
|------|-----|
| Name | `chainstream` |
| Type | `stdio` |
| Command | `npx @chainstream-io/mcp` |

그다음 Cursor의 환경 설정에서 API 키를 설정합니다:

```
CHAINSTREAM_API_KEY=your-key
```

또는 프로젝트 레벨의 `.cursor/mcp.json`에 추가합니다:

```json
{
  "mcpServers": {
    "chainstream": {
      "command": "npx",
      "args": ["@chainstream-io/mcp"],
      "env": {
        "CHAINSTREAM_API_KEY": "your-key"
      }
    }
  }
}
```

<Tip>
  Cursor는 Claude Desktop과 동일한 MCP 설정 형식을 사용합니다. Claude Desktop 설정이 이미 있다면, 같은 `mcpServers` 블록을 재사용할 수 있습니다.
</Tip>

## 문제 해결

<AccordionGroup>
  <Accordion title="도구 목록에 서버가 나타나지 않음">
    `env` 블록에 `CHAINSTREAM_API_KEY`가 올바르게 설정되어 있는지 확인하세요. 키가 누락되거나 유효하지 않으면 도구 호출 시 API 오류가 발생합니다. 터미널에서 직접 명령을 실행하여 테스트하세요:

    ```bash
    CHAINSTREAM_API_KEY=your-key npx @chainstream-io/mcp
    ```
  </Accordion>
  <Accordion title="npx 시작이 느림">
    `npx`의 첫 실행 시 패키지를 다운로드하므로 몇 초가 걸릴 수 있습니다. 빠른 시작을 위해 `npm install -g @chainstream-io/mcp`로 글로벌 설치한 후, 명령으로 `chainstream-mcp`를 사용하세요.
  </Accordion>
  <Accordion title="macOS 권한 오류">
    글로벌 npm 설치 시 권한 문제가 발생하면, `npx`를 사용하거나 npm이 사용자 레벨 prefix를 사용하도록 설정하세요:

    ```bash
    mkdir -p ~/.npm-global
    npm config set prefix '~/.npm-global'
    export PATH=~/.npm-global/bin:$PATH
    ```
  </Accordion>
</AccordionGroup>
