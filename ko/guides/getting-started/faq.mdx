## 빌링 관련

<AccordionGroup>
  <Accordion title="Free 플랜의 제한은 무엇인가요?">
    Free 플랜 제한:

    - **쿼터:** 30K Units/월
    - **요청 속도:** 10 요청/초
    - **데이터 지연:** 1-2초 지연이 있을 수 있음
    - **SLA:** 보장 없음
    - **초과:** 쿼터 소진 시 403 오류 반환, 다음 달 리셋

    개발 테스트와 POC에 적합하며, 프로덕션에는 추천하지 않습니다.
  </Accordion>

  <Accordion title="현재 사용량은 어떻게 확인하나요?">
    1. [대시보드](https://www.chainstream.io/dashboard)에 로그인
    2. **Usage** 페이지에서 현재 월 사용량, 잔여 쿼터, 히스토리 트렌드 확인
  </Accordion>

  <Accordion title="어떤 결제 수단을 지원하나요?">
    | 수단 | 지원 플랜 |
    |:--|:--|
    | 신용카드 (Visa, MasterCard, AMEX) | 모든 플랜 |
    | 암호화폐 (USDT, USDC) | Starter 이상 |
    | 은행 이체 | Enterprise / Custom |

    암호화폐 결제는 ERC-20 및 TRC-20 네트워크를 지원합니다.
  </Accordion>

  <Accordion title="언제든 업그레이드 또는 다운그레이드할 수 있나요?">
    - **업그레이드:** 즉시 적용, 비례 과금
    - **다운그레이드:** 다음 빌링 주기에 적용

    대시보드 → Billing → Subscription에서 관리하세요.
  </Accordion>

  <Accordion title="미사용 쿼터는 이월되나요?">
    아니오. 월간 쿼터는 매월 말에 리셋되며 이월되지 않습니다. 실제 사용량에 맞는 플랜을 선택하세요.
  </Accordion>
</AccordionGroup>

## 기술 문제

<AccordionGroup>
  <Accordion title="429 또는 403 오류가 발생하면 어떻게 하나요?">
    - **429 오류**: 요청 속도 초과
    - **403 오류**: 쿼터 소진

    **문제 해결:**

    1. **429 - 속도 제한 확인**
       - Free 플랜: 10 요청/분
       - 유료 플랜: [API 보안](/ko/guides/data-concepts/api-security) 참고

    2. **403 - 쿼터 소진 확인**
       - 대시보드 → Usage에서 잔여 쿼터 확인
       - 유료 플랜은 추가 쿼터를 구매하여 서비스 복원 가능

    **해결 방법:**
    - 429: 요청 스로틀링 또는 지수 백오프 재시도 구현
    - 403: 추가 쿼터 구매 또는 플랜 업그레이드
    - 폴링 대신 WebSocket 사용으로 요청 수 감소

```javascript
// 오류 처리 예시
async function handleApiError(error) {
  if (error.status === 429) {
    // 속도 제한, 대기 후 재시도
    await sleep(1000);
    return retry();
  } else if (error.status === 403) {
    // 쿼터 소진, 추가 쿼터 구매 필요
    console.error('Quota exhausted, please purchase additional quota in Dashboard');
  }
}
```
  </Accordion>

  <Accordion title="WebSocket 연결 끊김은 어떻게 처리하나요?">
    자동 재연결 메커니즘을 구현하세요:

```javascript
class ChainStreamWebSocket {
  constructor(baseUrl, accessToken) {
    this.baseUrl = baseUrl;
    this.accessToken = accessToken;
    this.reconnectDelay = 1000;
    this.maxReconnectDelay = 30000;
    this.connect();
  }

  connect() {
    // URL 파라미터를 통해 토큰 전달
    const url = `${this.baseUrl}?token=${this.accessToken}`;
    this.ws = new WebSocket(url);
    this.ws.onopen = () => {
      console.log('Connected');
      this.reconnectDelay = 1000; // 지연 초기화
    };
    this.ws.onclose = () => {
      console.log('Disconnected, reconnecting...');
      setTimeout(() => this.connect(), this.reconnectDelay);
      // 지수 백오프
      this.reconnectDelay = Math.min(
        this.reconnectDelay * 2,
        this.maxReconnectDelay
      );
    };
    this.ws.onerror = (error) => {
      console.error('WebSocket error:', error);
    };
  }
}
```

    **재연결 팁:**
    - 지수 백오프 사용 (1s → 2s → 4s → ... → 30s)
    - 최대 재연결 지연 설정 (예: 30초)
    - 재연결 성공 후 데이터 재구독
  </Accordion>

  <Accordion title="API 응답 형식은 어떻게 되나요?">

## KYT/KYA 관련

<AccordionGroup>
  <Accordion title="KYT와 KYA의 차이점은 무엇인가요?">
    | 기능 | KYA (주소 검증) | KYT 보고서 |
    |:--|:--|:--|
    | 목적 | 주소 리스크 및 프로필 분석 확인 | 거래 수준 리스크 보고서 |
    | 입력 | 지갑 주소 | 트랜잭션 해시 또는 주소 |
    | 출력 | 리스크 수준 + 주소 유형 + 리스크 노출 | 거래 관련 리스크 분석 |
    | 대표 사용 사례 | 사용자 등록/입금 전 주소 스크리닝 | 특정 거래의 컴플라이언스 확인 |

    자세한 내용은 [보안 컴플라이언스 문서](/ko/guides/security-compliance/overview)를 참고하세요.
  </Accordion>

  <Accordion title="주소 검증은 어떤 정보를 반환하나요?">
    주소 검증이 반환하는 필드:

    | 필드 | 설명 | 예시 값 |
    |:--|:--|:--|
    | Risk | 리스크 수준 | Low, Medium, High, Severe |
    | Status | 검증 상태 | COMPLETE, PENDING |
    | Address Type | 주소 유형 | PRIVATE_WALLET, EXCHANGE, CONTRACT 등 |
    | Risk Exposures | 리스크 노출 상세 | 리스크 라벨 및 관련 금액 |
  </Accordion>

  <Accordion title="리스크 수준은 어떻게 해석하나요?">
    주소 검증이 반환하는 리스크 수준:

    | 리스크 수준 | 의미 | 권장 조치 |
    |:--|:--|:--|
    | Low | 낮은 리스크, 명확한 의심 연관 없음 | 정상 처리 |
    | Medium | 중간 리스크, 일부 의심 연관 있음 | 수동 검토 추천 |
    | High | 높은 리스크, 상당한 의심 연관 있음 | 거부 또는 강화 검토 추천 |
    | Severe | 심각한 리스크, 제재/불법 엔티티와 직접 연관 | 거부 강력 추천 |

    **참고:** 구체적인 처리 정책은 비즈니스 컴플라이언스 요구사항에 따라야 합니다. 위는 참고 제안일 뿐입니다.
  </Accordion>

  <Accordion title="Risk Exposures에는 어떤 리스크 라벨이 있나요?">
    Risk Exposures는 다양한 리스크 엔티티와의 주소 연관을 보여줍니다. 일반적인 라벨:

    **고위험 라벨 (Severe):**

    | 라벨 | 설명 |
    |:--|:--|
    | sanctioned entity | 제재 엔티티와 연관 |
    | sanctioned jurisdiction | 제재 관할권과 연관 |
    | terrorist financing | 테러 자금 조달과 연관 |

    **중립/저위험 라벨:**

    | 라벨 | 설명 |
    |:--|:--|
    | bridge | 크로스체인 브릿지를 통해 자금 이전 |
    | decentralized exchange | DEX를 통해 거래 |
    | atm | 암호화폐 ATM을 통해 거래 |

    **기타 리스크 라벨:**

    | 라벨 | 설명 |
    |:--|:--|
    | mixer | 믹싱 서비스를 통해 |
    | gambling | 도박 플랫폼과 연관 |
    | darknet | 다크넷 마켓과 연관 |

    각 라벨에 표시되는 내용:
    - **direct / indirect**: 직접 또는 간접 연관
    - **amount**: 관련 자금 금액
    - **percentage**: 총 거래량 대비 비율
  </Accordion>

  <Accordion title="direct와 indirect의 차이점은 무엇인가요?">
    | 유형 | 의미 | 리스크 수준 |
    |:--|:--|:--|
    | direct | 주소가 리스크 엔티티와 직접 상호작용 | 더 높음 |
    | indirect | 중간 주소를 통해 간접적으로 연관 | 상대적으로 낮음 |

    **예시:**
    - `sanctioned entity + direct`: 주소가 제재 주소로 직접 전송
    - `sanctioned entity + indirect`: 이 주소의 연관 주소가 이전에 제재 주소와 상호작용

    Severe 수준 라벨 (제재, 테러 자금 조달)과 관련된 간접 연관도 신중한 처리가 필요합니다.
  </Accordion>

  <Accordion title="어떤 주소 유형이 있나요?">
    | 주소 유형 | 설명 |
    |:--|:--|
    | PRIVATE_WALLET | 개인 지갑 주소 |
    | EXCHANGE | 중앙화 거래소 주소 |
    | CONTRACT | 스마트 컨트랙트 주소 |
    | MINING_POOL | 마이닝 풀 주소 |
    | MERCHANT | 가맹점 주소 |
    | PAYMENT_PROCESSOR | 결제 프로세서 |

    주소 유형은 자금 흐름과 비즈니스 컨텍스트를 이해하는 데 도움이 됩니다.
  </Accordion>

## AI/MCP 관련

<AccordionGroup>
  <Accordion title="MCP란 무엇인가요?">
    MCP (Model Context Protocol)는 Anthropic이 제안한 프로토콜로, AI 모델이 외부 도구를 호출할 수 있게 합니다.

    ChainStream은 Claude와 다른 AI가 온체인 데이터를 직접 조회할 수 있는 MCP Server를 제공합니다.

    자세한 내용은 [MCP Server 문서](/ko/guides/ai-infrastructure/mcp-server/introduction)를 참고하세요.
  </Accordion>

  <Accordion title="Claude Desktop에서 ChainStream을 어떻게 사용하나요?">
    1. ChainStream MCP Server 설치
    2. Claude Desktop의 MCP 설정 구성
    3. Claude Desktop 재시작
    4. 대화 시작 - Claude가 자동으로 ChainStream을 호출하여 데이터 조회

    상세 설정은 [Claude 통합 가이드](/ko/guides/ai-infrastructure/mcp-server/setup-guide)를 참고하세요.
  </Accordion>

  <Accordion title="AI가 거래를 실행할 수 있나요?">
    **현재는 지원하지 않습니다.** ChainStream MCP Server는 읽기 작업만 제공합니다:

    - 토큰 가격 및 정보 조회
    - 지갑 보유량 및 거래 내역 조회
    - KYT/KYA 리스크 평가 실행

    보안상의 이유로 자동 거래 실행, 전송 또는 기타 쓰기 작업은 지원하지 않습니다.
  </Accordion>

  <Accordion title="MCP 호출은 어떻게 과금되나요?">
    MCP 호출은 직접 API 호출과 동일하게 과금되며, 실제 호출된 API 유형에 따라 Unit이 소비됩니다.
  </Accordion>
</AccordionGroup>
