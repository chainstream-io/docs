**DEX Pools** 資料產品給你 ChainStream 索引的每個流動性池的狀態——經典 Uniswap v2 恆積池、v3/v4 集中流動性池、Solana AMM 等——欄位已規整。

## 包含什麼

| 欄位組 | 舉例 |
|:--|:--|
| 身份 | `chainId`, `poolAddress`, `dex`, `pairType`（v2/v3/v4/CLMM/StableSwap） |
| 代幣 | `token0`, `token1`, `decimals0`, `decimals1`, `reserve0`, `reserve1` |
| 經濟 | `tvlUsd`, `volume24hUsd`, `feeTierBps`, `apy7d` |
| 狀態 | `priceToken0InToken1`, `sqrtPriceX96`, `tick`, `liquidity` |
| 來源 | `createdAtBlock`, `factoryAddress`, `initCodeHash` |

## 覆蓋與新鮮度

- 池子建立按區塊檢測；狀態欄位在每次 swap / add / remove 之後更新（通常秒級）。
- 歷史 TVL 與成交量按 1h、1d 聚合。

## 常見用例

- 構建按 TVL / 成交量篩選的 all-pools 看板
- 抓新池 / 打新告警
- swap 前檢查具體池子深度
- 驅動需要實時儲備的路由層

## 計費

單池查詢 **1 read unit**；搜尋與列表按 `limit` 計費。
