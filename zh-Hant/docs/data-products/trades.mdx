**Trades** 資料產品是 ChainStream 索引的每一筆 DEX swap 的規整版本——你可以按代幣、錢包、池子、路由器查詢，不用關心事件來自哪個 AMM。

## 包含什麼

| 欄位組 | 舉例 |
|:--|:--|
| 身份 | `chainId`, `txHash`, `logIndex`, `blockNumber`, `timestamp` |
| 雙方 | `makerAddress`, `takerAddress`, `router`, `dex` |
| 經濟 | `tokenIn`, `tokenOut`, `amountIn`, `amountOut`, `priceUsd`, `sizeUsd` |
| 上下文 | `poolAddress`, `feeTier`, `slippageBps`, `gasUsedUsd` |
| 分類 | `side` (buy/sell)、`type` (open/close/add/remove)、`isFirstBuy` |

## 覆蓋與新鮮度

- **鏈**：[支援的鏈](/zh-Hant/docs/supported-chains) 裡全部。
- **DEX**：Uniswap v2/v3/v4、Raydium、Meteora、PancakeSwap、Aerodrome、Jupiter 路由等——詳見 [分鏈支援表](/zh-Hant/api-reference/kafka-topics/evm)。
- **延遲**：區塊級——Kafka 在確認後 1–2s 投遞，REST 5–10s。

## 常見用例

- 渲染某代幣的實時成交 tape
- 計算錢包的已實現 PnL
- 在首筆買入 / 大單上觸發告警
- 為交易機器人提供 maker/taker 上下文
- 自定義週期聚合成 OHLC（或直接用 [Candles](/zh-Hant/docs/data-products/candles-ohlc)）

## 計費

REST / GraphQL 按 `limit` 分頁計費；WebSocket 與 Kafka 按投遞條數，流量越大階梯折扣越多。
