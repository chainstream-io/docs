**Wallets** 資料產品把任何一個鏈上地址變成可查詢的檔案——當前持倉、歷史 PnL、活躍度、首筆交易時間——覆蓋所有支援鏈，你不用自己拼多個 RPC。

## 包含什麼

| 欄位組 | 舉例 |
|:--|:--|
| 身份 | `chainId`, `address`, `ens`/`sns`, `tags`, `firstSeenAt` |
| 持倉 | `holdings[]`：`token`, `balance`, `valueUsd`, `costBasisUsd`, `unrealizedPnlUsd` |
| 淨值 | `netWorthUsd`、小時 / 日粒度的 `netWorthHistory[]` |
| 活躍度 | `txCount24h`, `tradeCount30d`, `avgTradeSizeUsd`, `activeChains[]` |
| 分類 | `walletType` (EOA / 合約 / 交易所 / smart-money)、`riskScore` |

## 覆蓋與新鮮度

- **跨鏈**：預設按所有支援鏈聚合餘額與 PnL；需要時按鏈過濾。
- **新鮮度**：持倉在命中的轉賬 / 交易之後秒級重新整理；淨值歷史按小時快照。

## 常見用例

- 零售使用者的資產總覽
- 大戶 / smart money 跟單 feed
- "這個錢包是誰？"的輕量級預覽
- 活動歸因（空投、獎勵）
- 作為合規初篩輸入（配合 [KYA](/zh-Hant/docs/compliance/kya-concepts)）

## 計費

檔案與持倉端點 **1 read unit**；時間序列 PnL 與歷史按返回範圍計費。
