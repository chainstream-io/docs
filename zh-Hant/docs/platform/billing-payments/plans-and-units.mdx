## 概述

ChainStream 採用 **Unit 計費模型**。不同 API 消耗不同數量的 Unit，你可以選擇適合業務需求的套餐。

## Unit 計算

### 什麼是 Unit

Unit 是 ChainStream 的計費單位。不同 API 消耗不同數量的 Unit — 越複雜的 API 消耗越多。

### Data API 消耗

Unit 成本按計算複雜度分層。簡單查詢成本較低；聚合密集型和鏈上執行端點成本較高。

### WebSocket 用量估算示例

```
场景：监控 10 个代币的实时 K 线（1 分钟分辨率）

计算：
- 每条 K 线消息约 200 字节
- 每个代币每分钟约 1 条 K 线消息
- 10 代币 × 1 条/分 × 60 分 × 24 小时 × 30 天 = 432,000 条消息
- 总数据量：432,000 × 200 = 86,400,000 字节
- 月费：86,400,000 × 0.005 = 432,000 Units

场景：监控 50 个代币的统计数据（实时，约每秒 1 次更新）

计算：
- 代币统计消息约 500 字节
- 50 代币 × 1 条/秒 × 86,400 秒 × 30 天 = 129,600,000 条消息
- 总数据量：129,600,000 × 500 = 64,800,000,000 字节（约 64.8 GB）
- 月费：64,800,000,000 × 0.005 = 324,000,000 Units（约 324M）
```

<Note>
批次 API 通常有折扣。具體消耗請參考 API 參考文件。WebSocket 訊息頻率取決於市場活躍度；高波動時期訊息量會增加。
</Note>

| 告警級別 | 觸發條件 | 通知方式 |
|:--|:--|:--|

| 警告 | 使用 80% | 郵件 + Dashboard 告警 |
| 嚴重 | 使用 100% | 郵件 + Dashboard 告警 |

<Tip>
建議至少開啟 80% 告警，避免意外的服務中斷。
</Tip>

## 配額耗盡處理

### 配額用完會怎樣

當月配額耗盡時，API 將返回 403 錯誤，無法繼續呼叫。付費套餐使用者可手動購買額外配額以恢復服務。

| 套餐 | 處理方式 |
|:--|:--|
| Free | API 返回 403 錯誤，等待下月配額重置 |
| Starter / Pro / Enterprise | API 返回 403 錯誤，可購買額外配額恢復服務 |
| Custom | 按合同條款 |

### 購買額外配額

付費套餐使用者可在 Dashboard 購買額外配額，按套餐的超量費率定價：

| 套餐 | 額外配額價格 |
|:--|:--|
| Starter | $12.9 / 百萬 Units |
| Pro | $9.9 / 百萬 Units |
| Enterprise | $6.9 / 百萬 Units |

<Warning>
配額耗盡時服務立即中斷（返回 403 錯誤）。建議設定 80% 用量告警並提前購買額外配額，避免影響業務。
</Warning>

## AI Agent 支付（x402 / MPP）

對於需要程式設計訪問而無需 Dashboard 註冊的 AI Agent，ChainStream 支援兩種鏈上支付協議：

### x402（Base / Solana USDC）

[x402 協議](https://www.x402.org/) 實現 HTTP 原生微支付。當 API 呼叫返回 402 時，Agent 的錢包簽名 USDC 轉賬並重試 — 無需註冊賬號。

| 屬性 | 值 |
|------|-----|
| 網路 | Base（鏈 ID 8453）、Solana |
| 貨幣 | USDC |
| 標準 | EIP-3009（Base）、原生轉賬（Solana） |
| Gas 費 | **免費** — ChainStream 的 facilitator 代付所有 gas 費用 |

<Tip>
你只需支付 USDC 訂閱費用。ChainStream 運營自有的 x402 facilitator，負責提交鏈上交易並代付 gas 費 — Agent 錢包無需持有 ETH 或 SOL 來支付 gas。
</Tip>

### MPP（Tempo USDC.e）

MPP 使用 Tempo 網路進行支付。Gas 直接用 USDC.e 支付 — 無需 ETH。

| 屬性 | 值 |
|------|-----|
| 網路 | Tempo（鏈 ID 4217） |
| 貨幣 | USDC.e |

### 工作原理

1. Agent 呼叫任意 ChainStream API
2. 伺服器返回 402（無有效訂閱）
3. Agent 從 `/x402/pricing` 獲取可用套餐
4. Agent 透過 x402 或 MPP 購買套餐
5. 伺服器返回 **API Key** — Agent 儲存用於後續所有呼叫
