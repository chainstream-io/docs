本頁涵蓋下列用於錢包與持倉者分析的 Cube：

- **BalanceUpdates**（DWD）— 錢包逐事件的餘額變動
- **TokenHolders**（DWS）— 代幣目前的持倉快照
- **WalletTokenPnL**（DWS）— 錢包各代幣的損益

<Note>
以下範例皆使用 `network: sol`（Solana）。若為其他支援鏈，請改為 `eth`、`bsc` 或 `polygon`。
</Note>

---

## 如何取得錢包的代幣餘額？

取得錢包目前的代幣餘額。此查詢回傳該錢包持有各代幣的最新一筆餘額更新事件。

```graphql
query {
  Solana {
    BalanceUpdates(
      limit: {count: 50}
      ownerAddress: {is: "WALLET_ADDRESS"}
    ) {
      BalanceUpdate {
        Currency { MintAddress }
        Account { Owner }
        PostBalance
        PostBalanceInUSD
      }
      Block { Time }
    }
  }
}
```

<Tip>
[在 GraphQL IDE 開啟](https://ide.chainstream.io) — 貼上上方查詢即可互動執行，並享有自動完成與 schema 探索。
</Tip>

<Tip>
請將 `WALLET_ADDRESS` 換成要檢視的錢包。`ownerAddress` 選擇器會依錢包擁有者篩選餘額更新事件。
</Tip>

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `BalanceUpdate.Currency.MintAddress` | 代幣地址 |
    | `BalanceUpdate.Account.Owner` | 錢包擁有者地址 |
    | `BalanceUpdate.PostBalance` | 更新後的代幣餘額 |
    | `BalanceUpdate.PostBalanceInUSD` | 餘額的美元價值 |
    | `Block.Time` | 餘額更新的時間戳 |
  </Accordion>
  <Accordion title="自訂建議">
    - **提高 limit**：設 `count: 200` 以取得更多代幣（持倉較多的錢包）
    - **依價值篩選**：加上 `where: {BalanceUpdate: {PostBalanceInUSD: {gt: 1}}}` 排除碎額餘額
  </Accordion>
</AccordionGroup>

## 如何取得餘額變動歷史？

追蹤某錢包對特定代幣的餘額如何隨時間變化。

```graphql
query {
  Solana {
    BalanceUpdates(
      limit: {count: 20}
      ownerAddress: {is: "WALLET_ADDRESS"}
      tokenAddress: {is: "TOKEN_ADDRESS"}
      orderBy: {descending: Block_Time}
    ) {
      BalanceUpdate {
        PreBalance
        PostBalance
        PostBalanceInUSD
      }
      Block { Time }
      Transaction { Hash }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `BalanceUpdate.PreBalance` | 事件前的餘額 |
    | `BalanceUpdate.PostBalance` | 事件後的餘額 |
    | `BalanceUpdate.PostBalanceInUSD` | 更新後餘額的美元價值 |
    | `Block.Time` | 餘額變動時間 |
    | `Transaction.Hash` | 造成變動的交易 |
  </Accordion>
  <Accordion title="自訂建議">
    - **判斷累積**：比較 `PreBalance` 與 `PostBalance` — 當 `PostBalance > PreBalance` 表示錢包在累積
    - **時間範圍**：加上 `where: {Block: {Time: {since: "2025-03-01T00:00:00Z"}}}` 限定區間
    - **更長歷史**：將 `count` 提高到最多 `10000` 以取得完整餘額歷史
  </Accordion>
</AccordionGroup>

<Info>
`PostBalance` 與 `PreBalance` 的差額即每筆事件的淨變動。正數表示代幣流入；負數表示流出。
</Info>

## 如何取得代幣的大戶持倉？

依餘額排序取得代幣的頂部持倉者。

```graphql
query {
  Solana {
    TokenHolders(
      limit: {count: 100}
      tokenAddress: {is: "TOKEN_ADDRESS"}
    ) {
      Token { Address }
      Holder { Address }
      LatestBalance
      LatestBalanceUSD
      FirstSeen
      LastSeen
    }
  }
}
```

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Token.Address` | 代幣地址 |
    | `Holder.Address` | 持倉者錢包地址 |
    | `LatestBalance` | 目前代幣餘額 |
    | `LatestBalanceUSD` | 目前餘額的美元價值 |
    | `FirstSeen` | 該持倉者首次取得此代幣的時間 |
    | `LastSeen` | 涉及此代幣的最近一筆交易 |
  </Accordion>
  <Accordion title="自訂建議">
    - **大戶門檻**：加上 `where: {LatestBalanceUSD: {gt: 10000}}` 僅顯示大額持倉者
    - **活躍持倉者**：加上 `where: {LastSeen: {after: "2025-03-01T00:00:00Z"}}` 篩選近期仍有活動者
    - **持倉人數**：使用 `count` 指標取得總持倉者數
  </Accordion>
</AccordionGroup>

<Tip>
**TokenHolders** Cube 屬於 DWS（Summary）層資料表 — 已預先彙總，查詢遠比掃描單筆餘額更新快。適合用於持倉排名與分佈分析。
</Tip>

## 如何取得錢包的損益？

取得錢包各代幣的損益資料，包含買／賣成交量與交易次數。

```graphql
query {
  Solana {
    WalletTokenPnL(
      limit: {count: 20}
      walletAddress: {is: "WALLET_ADDRESS"}
    ) {
      Wallet { Address }
      Token { Address }
      BuyVolumeUSDState
      SellVolumeUSDState
      BuyCountState
      SellCountState
      FirstTradeState
      LastTradeState
    }
  }
}
```

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Wallet.Address` | 錢包地址 |
    | `Token.Address` | 代幣地址 |
    | `BuyVolumeUSDState` | 買入此代幣累計花費的美元 |
    | `SellVolumeUSDState` | 賣出此代幣累計收到的美元 |
    | `BuyCountState` | 買入交易次數 |
    | `SellCountState` | 賣出交易次數 |
    | `FirstTradeState` | 首筆交易時間戳 |
    | `LastTradeState` | 最近一筆交易時間戳 |
  </Accordion>
  <Accordion title="自訂建議">
    - **計算 PnL**：`SellVolumeUSDState - BuyVolumeUSDState` 即各代幣已實現損益
    - **仍持倉部位**：篩選 `SellCountState < BuyCountState` 可找出錢包仍持有的代幣
    - **高頻交易代幣**：依 `BuyCountState` 或 `SellCountState` 排序找出最常交易的代幣
  </Accordion>
</AccordionGroup>

<Info>
**WalletTokenPnL** Cube 使用 `walletAddress` 選擇器（非 `ownerAddress`）。提供累計交易指標 — 適合投組分析與績效排行榜。
</Info>
