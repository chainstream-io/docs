**DEXTrades** Cube 收錄逐筆 DEX 兌換事件，是最細緻的交易資料。可用來查詢單筆交易、分析錢包活動、追蹤代幣價格，以及找出頂尖交易者。

<Note>
以下範例皆使用 `network: sol`（Solana）。若為其他支援鏈，請改為 `eth`、`bsc` 或 `polygon`。
</Note>

---

## 如何取得最新的 DEX 交易？

取得 Solana 上最近 10 筆 DEX 交易，包含區塊資訊、交易雜湊、買賣細節與 DEX 協定。

```graphql
query {
  Solana {
    DEXTrades(
      limit: {count: 10}
      orderBy: {descending: Block_Time}
    ) {
      Block { Time, Slot }
      Transaction { Hash }
      Trade {
        Buy {
          Currency { MintAddress }
          Amount
          PriceInUSD
          Account { Owner }
        }
        Sell {
          Currency { MintAddress }
          Amount
          Account { Owner }
        }
        Dex { ProgramAddress, ProtocolName }
      }
      Pool { Address }
    }
  }
}
```

<Tip>
[在 GraphQL IDE 開啟](https://ide.chainstream.io) — 貼上上方查詢即可互動執行，並享有自動完成與 schema 探索。
</Tip>

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Block.Time` | 區塊時間戳（ISO 8601） |
    | `Block.Slot` | Solana slot 編號（Solana 專用） |
    | `Transaction.Hash` | 鏈上交易雜湊 — 可用於在瀏覽器查詢該筆交易 |
    | `Trade.Buy.Currency.MintAddress` | 買入資產的代幣地址 |
    | `Trade.Buy.PriceInUSD` | 成交當下買入代幣的美元價格 |
    | `Trade.Buy.Account.Owner` | 買方錢包地址 |
    | `Trade.Dex.ProtocolName` | DEX 名稱（例如 Raydium、Orca、Jupiter） |
    | `Pool.Address` | 該筆交易執行所在的流動性池地址 |
  </Accordion>
  <Accordion title="自訂建議">
    - **切換鏈**：將 `network: sol` 改為 `network: eth`、`network: bsc` 或 `network: polygon`
    - **增加筆數**：將 `count: 10` 改為最多 `10000`
    - **加入時間篩選**：加上 `where: {Block: {Time: {after: "2025-03-01T00:00:00Z"}}}` 以限定時間範圍
    - **依 DEX 篩選**：加上 `where: {Trade: {Dex: {ProtocolName: {is: "Raydium"}}}}` 以限定特定協定
  </Accordion>
</AccordionGroup>

## 如何取得特定代幣的交易？

透過 `tokenAddress` 選擇器傳入代幣地址，即可取得該代幣的交易。

```graphql
query {
  Solana {
    DEXTrades(
      limit: {count: 10}
      tokenAddress: {is: "TOKEN_ADDRESS"}
      orderBy: {descending: Block_Time}
    ) {
      Block { Time }
      Trade {
        Buy { Amount, PriceInUSD, Account { Owner } }
        Sell { Currency { MintAddress }, Amount }
        Dex { ProtocolName }
      }
      Pool { Address }
    }
  }
}
```

<Tip>
請將 `TOKEN_ADDRESS` 換成實際的代幣 mint 地址（例如 Solana 上 USDC 為 `EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v`）。依名稱查地址可參考 [代幣 metadata](/zh-Hant/graphql/examples/ohlc-stats#how-do-i-look-up-token-information)。
</Tip>

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Trade.Buy.Amount` | 買入的代幣數量 |
    | `Trade.Buy.PriceInUSD` | 成交時每單位代幣價格 |
    | `Trade.Buy.Account.Owner` | 執行買入的錢包 |
    | `Trade.Sell.Currency.MintAddress` | 賣出資產的代幣地址（交易對的另一邊） |
    | `Trade.Dex.ProtocolName` | DEX 協定名稱 |
  </Accordion>
  <Accordion title="自訂建議">
    - **依最小金額篩選**：加上 `where: {Trade: {Buy: {Amount: {gt: 1000}}}}` 僅顯示大額交易
    - **依價格區間篩選**：加上 `where: {Trade: {Buy: {PriceInUSD: {gte: 0.001, lte: 1.0}}}}` 限定價格帶
    - **排除可疑交易**：預設已套用 `IsSuspect = false`，機器人／MEV 交易已排除
  </Accordion>
</AccordionGroup>

## 如何取得某錢包的所有交易？

依特定錢包地址取得其所有交易。

```graphql
query {
  Solana {
    DEXTrades(
      limit: {count: 20}
      walletAddress: {is: "WALLET_ADDRESS"}
      orderBy: {descending: Block_Time}
    ) {
      Block { Time }
      Trade {
        Buy { Currency { MintAddress }, Amount, PriceInUSD }
        Sell { Currency { MintAddress }, Amount }
        Dex { ProtocolName }
      }
      Transaction { Hash, FeeInNative }
    }
  }
}
```

<Info>
`walletAddress` 選擇器會比對買方或賣方為該錢包的交易。
</Info>

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Trade.Buy.Currency.MintAddress` | 買入的代幣 |
    | `Trade.Sell.Currency.MintAddress` | 賣出的代幣 |
    | `Transaction.FeeInNative` | 以原生代幣支付的 gas 手續費（Solana 上為 SOL） |
  </Accordion>
  <Accordion title="自訂建議">
    - **僅限單一代幣**：可與 `tokenAddress: {is: "TOKEN_ADDRESS"}` 併用，只看該錢包對該代幣的交易
    - **加入時間窗**：加上 `where: {Block: {Time: {after: "2025-03-01T00:00:00Z"}}}` 限定近期交易
    - **提高 limit**：設 `count: 100` 可取得更多歷史（上限 10,000）
  </Accordion>
</AccordionGroup>

## 如何取得代幣的目前價格？

從最近一筆非可疑交易取得代幣最新價格。

```graphql
query {
  Solana {
    DEXTrades(
      limit: {count: 1}
      tokenAddress: {is: "TOKEN_ADDRESS"}
      where: {IsSuspect: {eq: false}}
      orderBy: {descending: Block_Time}
    ) {
      Trade {
        Buy { PriceInUSD, PriceInNative }
      }
      Block { Time }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Trade.Buy.PriceInUSD` | 最近一筆交易的美元價格 |
    | `Trade.Buy.PriceInNative` | 以鏈原生代幣（SOL、ETH、BNB）計價 |
    | `Block.Time` | 該筆交易時間戳 — 可看出價格有多新 |
  </Accordion>
  <Accordion title="自訂建議">
    - **多筆價格**：提高 `count` 可取得多筆近期價格以利平均
    - **跨鏈**：使用 `network: eth` 可在 Ethereum 取得同一代幣價格（若該鏈有該代幣）
  </Accordion>
</AccordionGroup>

<Tip>
若需要長期可靠的代幣價格，建議改用 [Pairs](/zh-Hant/graphql/examples/ohlc-stats#how-do-i-get-k-line-ohlc-candlestick-data) Cube — 提供預先計算的 K 線，含每分鐘開高低收。
</Tip>

## 如何找出某代幣的頂尖交易者？

透過彙總找出頂尖交易者。此查詢依買方錢包分組，回傳總買入次數與成交量。

```graphql
query {
  Solana {
    DEXTrades(
      limit: {count: 100}
      tokenAddress: {is: "TOKEN_ADDRESS"}
      where: {IsSuspect: {eq: false}}
    ) {
      Trade {
        Buy {
          Account { Owner }
          Amount
          PriceInUSD
        }
      }
      count
      sum(of: Trade_Buy_Amount)
    }
  }
}
```

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Trade.Buy.Account.Owner` | 錢包地址（分組鍵） |
    | `count` | 該錢包的交易筆數 |
    | `sum(of: Trade_Buy_Amount)` | 該錢包買入的代幣總量 |
  </Accordion>
  <Accordion title="自訂建議">
    - **依成交量排序**：結果會依維度欄位分組 — 選較少維度可得到較高層級的彙總
    - **限定時段的排行榜**：加上 `where: {Block: {Time: {after: "2025-03-01T00:00:00Z"}}}` 限定區間
    - **排除小額交易**：在 `where` 中加上 `Trade: {Buy: {Amount: {gt: 100}}}`
  </Accordion>
</AccordionGroup>

<Info>
當指標欄位（`count`、`sum`）與維度欄位並列時，API 會依所選維度自動分組。完整說明見 [Metrics & Aggregation](/zh-Hant/graphql/schema/metrics-aggregation)。
</Info>
