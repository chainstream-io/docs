本頁說明聚合後的交易與供應量資料。**Cube 名稱**為 **Trading** chain group 底下的 **Pairs** 與 **Tokens**（跨鏈 OHLC 與成交統計）。市值與價格隨時間變化則來自 **Solana** 或 **EVM** 底下的 **TokenSupplyUpdates**。

- **Pairs**（Trading，DWM）——每分鐘 K 線／蠟燭圖資料（OHLC、成交量、成交筆數）
- **Tokens**（Trading，DWM）——每分鐘成交統計，含買／賣拆解與不重複交易人數
- **TokenSupplyUpdates**（Solana／EVM，DWD）——增發／銷毀事件，含供應量、價格、市值與 FDV 等欄位
- **Token 中繼資料**——目前沒有獨立的 **TokenSearch** Cube；請在 **Pairs**／**Tokens** 上使用 **Token**（及相關）維度，或搭配 **DEXPools**、**TokenHolders**、**DEXTradeByTokens** 等 Cube 做探索與篩選

<Note>
**Trading** **沒有** `network` 參數。請以 `where: { Market: { Network: { is: "sol" } } }`（或 `eth`、`bsc`、`polygon`）依鏈篩選。**Solana** 與 **EVM** 群組則使用慣用包裝（`Solana { ... }`、`EVM(network: eth) { ... }`）。
</Note>

## 如何取得 K 線（OHLC）蠟燭圖資料？

查詢代幣的蠟燭圖資料——每分鐘的開、高、低、收盤價，以及 USD 成交量與成交筆數。請在 **Trading** 內使用 **Pairs** Cube。

```graphql
query {
  Trading {
    Pairs(
      tokenAddress: { is: "TOKEN_ADDRESS" }
      where: { Market: { Network: { is: "sol" } } }
      limit: { count: 24 }
      orderBy: { descending: Block_Time }
    ) {
      Interval { Time { Start } }
      Token { Address }
      Market { Network }
      Price {
        Ohlc {
          Open
          High
          Low
          Close
        }
      }
      Volume { Usd }
      Stats { TradeCount }
    }
  }
}
```

<Tip>
[在 GraphQL IDE 中開啟](https://ide.chainstream.io)——貼上上方查詢即可互動執行，並享有自動完成與 schema 探索。
</Tip>

<Tip>
將 `TOKEN_ADDRESS` 換成代幣 mint 或合約位址。每一列代表一分鐘區間——`limit: { count: 60 }` 約一小時，`count: 1440` 約 24 小時。若省略 `Market` 篩選，可於單一結果集中跨所有鏈查詢。
</Tip>

<AccordionGroup>
  <Accordion title="重點欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Interval.Time.Start` | 分鐘區間起點（與蠟燭時間相同） |
    | `Token.Address` | 代幣位址 |
    | `Market.Network` | 鏈識別（`sol`、`eth`、`bsc`、…） |
    | `Price.Ohlc.Open` | 區間開盤價 |
    | `Price.Ohlc.High` | 區間內最高價 |
    | `Price.Ohlc.Low` | 區間內最低價 |
    | `Price.Ohlc.Close` | 區間收盤價 |
    | `Volume.Usd` | 本區間 USD 成交總額 |
    | `Stats.TradeCount` | 本區間成交筆數 |
  </Accordion>
  <Accordion title="自訂技巧">
    - **更長時間範圍**：以較大的 `limit` 多取列，或在客戶端聚合；蠟燭以分鐘粒度儲存
    - **成交量篩選**：例如 `where: { Volume: { Usd: { gt: 100 } }, Market: { Network: { is: "sol" } } }` 略過低量區間
    - **時間範圍**：例如 `where: { Block: { Time: { since: "2026-03-27T00:00:00Z" } }, Market: { Network: { is: "sol" } } }`
  </Accordion>
</AccordionGroup>

<Info>
**Pairs** Cube 為 **DWM**（聚合）模型——資料已預先按分鐘計算。繪圖時比掃描原始成交快得多。
</Info>

## 如何取得代幣的成交統計？

取得每分鐘成交統計，含買／賣筆數、不重複買方／賣方人數與成交量。請在 **Trading** 內使用 **Tokens** Cube。

```graphql
query {
  Trading {
    Tokens(
      tokenAddress: { is: "TOKEN_ADDRESS" }
      where: { Market: { Network: { is: "sol" } } }
      limit: { count: 24 }
      orderBy: { descending: Block_Time }
    ) {
      Interval { Time { Start } }
      Token { Address }
      Market { Network }
      Stats {
        TradeCount
        BuyCount
        SellCount
        UniqueBuyers
        UniqueSellers
      }
      Volume { Usd }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="重點欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Interval.Time.Start` | 分鐘區間起點（與蠟燭時間相同） |
    | `Stats.TradeCount` | 本區間總成交筆數 |
    | `Stats.BuyCount` | 買方成交筆數 |
    | `Stats.SellCount` | 賣方成交筆數 |
    | `Volume.Usd` | USD 成交總額 |
    | `Stats.UniqueBuyers` | 不重複買方錢包數 |
    | `Stats.UniqueSellers` | 不重複賣方錢包數 |
  </Accordion>
  <Accordion title="自訂技巧">
    - **買賣力道**：比較 `Stats.BuyCount` 與 `Stats.SellCount`
    - **不重複交易人**：`Stats.UniqueBuyers` 與 `Stats.UniqueSellers` 可看出量是分散或集中
    - **活動熱度圖**：查詢整天（`count: 1440`）並依 `Interval.Time.Start`（或 `Block.Time`）繪圖
    - **買賣成交量**：需要 USD 拆解時，**Tokens** 紀錄亦提供 `Volume.BuyVolumeUSD` 與 `Volume.SellVolumeUSD`
  </Accordion>
</AccordionGroup>

<Tip>
對同一代幣、鏈與時間窗同時查 **Pairs** 與 **Tokens**，可組儀表板——OHLC 搭配流量與參與者指標。
</Tip>

## 如何取得市值、價格與供應量隨時間變化？

舊版摘要 Cube **TokenMarketCap** 已不在目前 schema 使用。請改以 **TokenSupplyUpdates**（Solana 或 EVM）：每一列對應影響供應的事件，**TokenSupplyUpdate** 指標含價格、市值、FDV 與總供應量。

### Solana

```graphql
query {
  Solana {
    TokenSupplyUpdates(
      tokenAddress: { is: "TOKEN_ADDRESS" }
      limit: { count: 24 }
      orderBy: { descending: Block_Time }
    ) {
      Block { Time }
      TokenSupplyUpdate {
        Currency {
          MintAddress
          Decimals
          Symbol
          Name
        }
        PriceInUSD
        MarketCapInUSD
        TotalSupply
        FDVInUSD
        PostBalance
      }
      Transaction { Signature }
    }
  }
}
```

### EVM（以 Ethereum 為例）

```graphql
query {
  EVM(network: eth) {
    TokenSupplyUpdates(
      tokenAddress: { is: "TOKEN_ADDRESS" }
      limit: { count: 24 }
      orderBy: { descending: Block_Time }
    ) {
      Block { Time }
      TokenSupplyUpdate {
        Currency {
          MintAddress
          Decimals
          Symbol
          Name
        }
        PriceInUSD
        MarketCapInUSD
        TotalSupply
        FDVInUSD
        PostBalance
      }
      Transaction { Hash }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="重點欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Block.Time` | 事件時間 |
    | `TokenSupplyUpdate.Currency.*` | 代幣識別（mint／合約、decimals、symbol、name） |
    | `TokenSupplyUpdate.PriceInUSD` | 本次更新時的 USD 價格 |
    | `TokenSupplyUpdate.MarketCapInUSD` | 市值 |
    | `TokenSupplyUpdate.TotalSupply` | 總供應量 |
    | `TokenSupplyUpdate.FDVInUSD` | 完全稀釋估值（FDV） |
    | `TokenSupplyUpdate.PostBalance` | 事件後與供應相關的餘額 |
  </Accordion>
  <Accordion title="自訂技巧">
    - **最新快照**：`limit: { count: 1 }` 並 `orderBy: { descending: Block_Time }`
    - **跨鏈比較**：在 `Solana` 與 `EVM(network: bsc)`（或其他支援網路）下執行相同查詢形狀
    - **更多脈絡**：供應量與流動性相關範例亦可參考 [流動性池與流動性](/zh-Hant/graphql/examples/pools-liquidity#how-do-i-get-token-supply-and-market-cap)
  </Accordion>
</AccordionGroup>

<Info>
**TokenSupplyUpdates** 為 **DWD**（事件層級）。適合與 mint／burn 活動相關的歷史估值與供應變化，而非單一靜態「市值」摘要列。
</Info>

## 代幣搜尋／中繼資料在哪裡？

**TokenSearch** Cube **並非**目前 API 的一部分。若要取得代幣脈絡：

- **Pairs** 與 **Tokens** 會一併暴露 **Token** 維度（例如 **Token.Address**）與 OHLC／統計——已知位址且要聚合交易資料時可使用。
- 若要更豐富的中繼資料、持有人數、池子或探索，請依鏈在 **Solana** 或 **EVM** 下使用 **DEXPools**、**TokenHolders**、**DEXTradeByTokens** 等 Cube。
