本頁涵蓋用於 DEX 資金池與流動性分析的 Cube：

- **DEXPoolEvents**（DWD）— 流動性新增／移除事件，含每筆事件的準備金與價格
- **DEXPools**（DWS）— 資金池**快照**（目前流動性與價格，定期更新）
- **TokenSupplyUpdates**（DWD）— 代幣供給、市值與 FDV 更新

<Note>
範例使用 **Solana** 與 **EVM** 的 [Chain Groups](/zh-Hant/graphql/schema/chain-groups)：`Solana { ... }` 用於 Solana，`EVM(network: eth | bsc | polygon) { ... }` 用於 EVM 鏈（含 Polygon）。
</Note>

---

## 如何取得某代幣的流動性池？

找出快照中將代幣列為 **token A** 的 DEX 池。若資產僅作為 token B 出現，需再以 `tokenB`（或 `where`）查第二趟；若要**事件層級**且含完整交易對脈絡，請用 **DEXPoolEvents**。

```graphql
query {
  Solana {
    DEXPools(
      limit: {count: 10}
      tokenA: {is: "TOKEN_ADDRESS"}
    ) {
      Pool {
        Address
        ProgramAddress
        TokenAAddress
        TokenBAddress
        LiquidityUSD
        PriceAtoB
        PriceBtoA
        LastUpdated
      }
    }
  }
}
```

<Tip>
[在 GraphQL IDE 開啟](https://ide.chainstream.io) — 貼上上方查詢即可互動執行，並享有自動完成與 schema 探索。
</Tip>

<Tip>
請將 `TOKEN_ADDRESS` 換成代幣 mint（Solana）或合約地址（EVM）。再以 `tokenB: {is: "TOKEN_ADDRESS"}` 重複查詢，可納入該代幣為報價端的池。
</Tip>

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Pool.Address` | 流動性池／市場地址 |
    | `Pool.ProgramAddress` | DEX 程式或 factory 地址 |
    | `Pool.TokenAAddress` | 基礎端代幣地址 |
    | `Pool.TokenBAddress` | 報價端代幣地址 |
    | `Pool.LiquidityUSD` | 池子總流動性（美元，快照） |
    | `Pool.LastUpdated` | 此快照列最後更新時間 |
  </Accordion>
  <Accordion title="自訂建議">
    - **依流動性篩選**：加上 `where: {Pool: {LiquidityUSD: {gt: 10000}}}` 僅顯示超過某美元深度門檻的池
    - **特定 DEX**：加上 `where: {Pool: {ProgramAddress: {is: "DEX_PROGRAM_OR_FACTORY_ADDRESS"}}}`
    - **更多池**：提高 `count` 以探索該代幣的更多池
    - **逐事件準備金**：使用 **DEXPoolEvents** 搭配 `tokenAddress`／`poolAddress` 與 `orderBy: {descending: Block_Time}` 取得加／減流動歷史
  </Accordion>
</AccordionGroup>

## 如何取得特定資金池的詳情？

依地址讀取單一池的**最新快照**。**DEXPools** 不提供每列的區塊時間軸 — 歷史準備金序列請用 **DEXPoolEvents**。

```graphql
query {
  Solana {
    DEXPools(
      limit: {count: 1}
      poolAddress: {is: "POOL_ADDRESS"}
    ) {
      Pool {
        Address
        TokenAAddress
        TokenBAddress
        ProgramAddress
        LiquidityUSD
        PriceAtoB
        PriceBtoA
        LastUpdated
      }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Pool.TokenAAddress` / `Pool.TokenBAddress` | 交易對代幣地址 |
    | `Pool.LiquidityUSD` | 總流動性（美元，快照） |
    | `Pool.LastUpdated` | 此池快照最後刷新時間 |
  </Accordion>
  <Accordion title="自訂建議">
    - **準備金歷史**：查詢 **DEXPoolEvents**，`poolAddress: {is: "POOL_ADDRESS"}` 並 `orderBy: {descending: Block_Time}`
    - **事件時間範圍**：在 **DEXPoolEvents** 加上 `where: {Block: {Time: {since: "2025-03-01T00:00:00Z"}}}` 限定加／減流區間
  </Accordion>
</AccordionGroup>

<Info>
`poolAddress` 選擇器依 `Pool.Address` 篩選，**DEXPools**（快照）與 **DEXPoolEvents**（事件）皆可用。
</Info>

## 如何對池排序或讀取快照流動性？

**DEXPools**（DWS）儲存**目前狀態**列（通常每數分鐘刷新）。可用它依 `LiquidityUSD` 排序，並比較不同 DEX 程式，無需掃描原始事件。

```graphql
query {
  Solana {
    DEXPools(
      limit: {count: 20}
      tokenA: {is: "TOKEN_ADDRESS"}
    ) {
      Pool {
        Address
        ProgramAddress
        TokenAAddress
        TokenBAddress
        LiquidityUSD
        LastUpdated
      }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `Pool.Address` | 池地址 |
    | `Pool.ProgramAddress` | DEX 程式／factory |
    | `Pool.TokenAAddress` / `Pool.TokenBAddress` | 交易對代幣 |
    | `Pool.LiquidityUSD` | 快照流動性（美元） |
    | `Pool.LastUpdated` | 快照新鮮度 |
  </Accordion>
  <Accordion title="自訂建議">
    - **依流動性排序**：客戶端依 `LiquidityUSD` 排序，或若 schema 支援池快照排序則使用 `orderBy`
    - **活躍池**：優先選擇 `LastUpdated` 較新的池
    - **比較 DEX**：依 `Pool.ProgramAddress` 分組或篩選
  </Accordion>
</AccordionGroup>

<Tip>
**時間序列**的準備金與每筆流動性事件請用 **DEXPoolEvents**（DWD）。**最新**池狀態與探索請用 **DEXPools**（DWS）。
</Tip>

## 如何取得代幣供給與市值？

使用 **TokenSupplyUpdates** 取得與供給事件連動的供給量、市值、價格與 FDV。

```graphql
query {
  Solana {
    TokenSupplyUpdates(
      limit: {count: 1}
      tokenAddress: {is: "TOKEN_ADDRESS"}
      orderBy: {descending: Block_Time}
    ) {
      TokenSupplyUpdate {
        Currency { MintAddress, Decimals }
        PostBalance
        MarketCapInUSD
        PriceInUSD
        FDVInUSD
        TotalSupply
      }
      Block { Time }
    }
  }
}
```

<AccordionGroup>
  <Accordion title="主要欄位">
    | 欄位 | 說明 |
    |:------|:------------|
    | `TokenSupplyUpdate.Currency.MintAddress` | 代幣地址 |
    | `TokenSupplyUpdate.Currency.Decimals` | 代幣小數位數 |
    | `TokenSupplyUpdate.PostBalance` | 事件後的目前供給 |
    | `TokenSupplyUpdate.MarketCapInUSD` | 美元市值 |
    | `TokenSupplyUpdate.PriceInUSD` | 供給事件當下的代幣價格 |
    | `TokenSupplyUpdate.FDVInUSD` | 完全稀釋估值 |
    | `TokenSupplyUpdate.TotalSupply` | 代幣總供給 |
  </Accordion>
  <Accordion title="自訂建議">
    - **供給歷史**：提高 `count` 觀察供給隨時間變化（鑄造／銷毀事件）
    - **市值歷史**：查多筆供給更新以繪製市值走勢
    - **與價格交叉驗證**：將 `PriceInUSD` 與 `TotalSupply` 併用以核對市值計算
  </Accordion>
</AccordionGroup>

<Info>
本頁以 **TokenSupplyUpdates** 說明與供給連動的市值、價格與 FDV。更多代幣指標與摘要見 [OHLC 與統計](/zh-Hant/graphql/examples/ohlc-stats)。
</Info>
