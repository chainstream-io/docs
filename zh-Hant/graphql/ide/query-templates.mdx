## 什麼是查詢範本

查詢範本是預先寫好的 GraphQL 查詢，涵蓋 Solana、Ethereum 與 BSC 上最常見的分析情境。每個範本對應特定 Cube，並已帶入正確的查詢結構、欄位與篩選參數——替換佔位符後即可執行。

範本可協助你：

- **透過範例學習 schema**——實際看到欄位、篩選與排序如何運作
- **快速起步**，無須從零撰寫查詢
- **發現**你可能尚未留意的 Cube

---

## 如何開啟範本

<Steps>
  <Step title="開啟側邊欄">
    點選側邊欄切換鈕，或按 **Ctrl/Cmd+B** 開啟左側面板。
  </Step>
  <Step title="選擇 Queries 分頁">
    側邊欄有兩個分頁：**Queries**（範本）與 **Saved**（已儲存查詢）。請選 **Queries**。
  </Step>
  <Step title="依鏈與 Cube 瀏覽">
    範本以樹狀組織：**鏈**（Solana / Ethereum / BSC）→ **Cube**（DEXTrades、Transfers 等）→ **範本**。
  </Step>
  <Step title="點擊載入">
    點選任一範本即可載入至查詢編輯器。佔位符會標示以利替換。
  </Step>
</Steps>

---

## 範本組織方式

範本採兩層階層：

```
Chain (Solana / Ethereum / BSC)
 └── Cube
      └── Template
```

每條鏈包含相同的 Cube 與範本集合，並預先設定正確的 `network` 值（`sol`、`eth` 或 `bsc`）。

## 佔位符

範本使用大寫佔位符，表示執行前須替換為實際鏈上地址。

| 佔位符 | 說明 | 範例 |
|:--|:--|:--|
| `TOKEN_ADDRESS` | 代幣合約／mint 地址 | `EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v`（Solana 上的 USDC） |
| `WALLET_ADDRESS` | 錢包公開地址 | `5Q544fKrFoe6tsEbD7S8EmxGTJYAKtTVhAW5Q5pge4j1` |
| `POOL_ADDRESS` | DEX 流動性池地址 | `58oQChx4yWmvKdwLLZzBi4ChoCc2fqCUWBkwMihLYQo2` |

<Note>
佔位符在查詢中會以字面值出現（例如 `MintAddress: {is: "TOKEN_ADDRESS"}`）。請將整段佔位字串（含引號）替換為你的實際地址。
</Note>

## 範例：使用範本

以下示範如何在 Solana 上使用 **DEXTrades → 最新成交** 範本：

<Steps>
  <Step title="開啟側邊欄並選擇範本">
    導覽至 **Solana → DEXTrades → 最新成交** 並點擊載入。
  </Step>
  <Step title="檢視載入的查詢">
    編輯器會填入：
    ```graphql
    query {
      Solana {
        DEXTrades(
          limit: {count: 10}
          orderBy: {descending: Block_Time}
        ) {
          Block { Time Slot }
          Transaction { Hash }
          Trade {
            Buy {
              Currency { MintAddress Symbol Name }
              Amount
              PriceInUSD
            }
            Sell {
              Currency { MintAddress Symbol Name }
              Amount
            }
            Dex { ProtocolName }
          }
        }
      }
    }
    ```
  </Step>
  <Step title="執行">
    此範本無佔位符——會取得最近 10 筆成交。按 **Ctrl/Cmd+Enter** 執行。
  </Step>
  <Step title="迭代調整">
    可修改查詢以加入篩選、變更 limit，或選取不同欄位。schema 自動完成可協助你探索可用選項。
  </Step>
</Steps>
