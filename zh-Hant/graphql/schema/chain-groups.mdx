## 概覽

ChainStream GraphQL 將 25 個 Cube 分為三個 **Chain Group（鏈群組）**。每個群組代表一條區塊鏈生態，並決定可用的 Cube、`network` 參數的行為，以及有哪些鏈專屬欄位。

```graphql
type ChainStream {
  EVM(network: Network!, dataset: Dataset, aggregates: Aggregates) { ... }
  Solana(dataset: Dataset, aggregates: Aggregates) { ... }
  Trading(dataset: Dataset, aggregates: Aggregates) { ... }
}
```

---

## EVM

**EVM** 群組包含所有 EVM 相容鏈的 Cube。查詢時**必須**提供 `network` 引數以指定要查哪條鏈。

### 可用網路

| 網路 ID | 區塊鏈 |
|:-----------|:-----------|
| `eth` | Ethereum |
| `bsc` | BNB Chain (BSC) |
| `polygon` | Polygon |

<Warning>
**Polygon 資料可用性：** 目前僅 **Prediction Markets** 相關 Cube（`PredictionTrades`、`PredictionManagements`、`PredictionSettlements`）在 Polygon 有資料。其餘 Cube（DEXTrades、Blocks、Transfers 等）仍在部署中——對它們下查詢可能回傳錯誤。
</Warning>

### Cubes

**與 Solana 共用：**
`DEXTrades`、`DEXTradeByTokens`、`Transfers`、`BalanceUpdates`、`DEXPoolEvents`、`TokenSupplyUpdates`、`Blocks`、`Transactions`、`TransactionBalances`、`DEXPools`、`TokenHolders`、`WalletTokenPnL`

**僅 EVM：**
- `Events` — 智慧合約事件日誌（已解碼的 topic 與 data）
- `Calls` — 內部呼叫 trace（CALL、DELEGATECALL、CREATE 等）
- `MinerRewards` — 區塊獎勵明細（static、dynamic、uncle、burned fees）
- `DEXPoolSlippages` — 流動池價格滑點分析
- `Uncles` — Uncle 區塊資料（主要為 Ethereum PoW 歷史）
- `PredictionTrades` — 預測市場成交 *(主要為 Polygon)*
- `PredictionManagements` — 預測市場管理事件 *(主要為 Polygon)*
- `PredictionSettlements` — 預測市場結算 *(主要為 Polygon)*

### 查詢範例

```graphql
query {
  EVM(network: eth, dataset: combined) {
    DEXTrades(
      limit: {count: 10}
      orderBy: {descending: Block_Time}
    ) {
      Block { Time Number }
      Transaction { Hash }
      Trade {
        Buy { Currency { SmartContract Symbol } Amount PriceInUSD }
        Sell { Currency { SmartContract Symbol } Amount }
        Dex { ProtocolName }
      }
    }
  }
}
```

## Solana

**Solana** 群組包含 Solana 鏈的 Cube。網路為隱式（`sol`）——**不需要** `network` 引數。

### Cubes

**與 EVM 共用：**
`DEXTrades`、`DEXTradeByTokens`、`Transfers`、`BalanceUpdates`、`DEXPoolEvents`、`TokenSupplyUpdates`、`Blocks`、`Transactions`、`TransactionBalances`、`DEXPools`、`TokenHolders`、`WalletTokenPnL`

**僅 Solana：**
- `Instructions` — 程式 instruction 資料（程式位址、方法、帳戶、log）
- `InstructionBalanceUpdates` — instruction 層級的餘額變動
- `Rewards` — 驗證者與質押獎勵
- `DEXOrders` — DEX 訂單簿事件（限價單、取消、成交）

### 查詢範例

```graphql
query {
  Solana(dataset: realtime) {
    DEXTrades(
      limit: {count: 10}
      orderBy: {descending: Block_Time}
    ) {
      Block { Time Slot }
      Transaction { Hash }
      Trade {
        Buy { Currency { MintAddress Symbol } Amount PriceInUSD }
        Sell { Currency { MintAddress Symbol } Amount }
        Dex { ProgramAddress ProtocolName }
      }
    }
  }
}
```

### 跨鏈欄位命名

下表為各鏈慣用的欄位名稱。不過 Record 型別會以**超集**包含**所有鏈**的欄位——每個 Cube 的 Record 都同時有 `MintAddress` 與 `SmartContract`，且對應同一底層資料。在任何鏈上都可任選其一使用。

| 概念 | 慣用（Solana） | 慣用（EVM） | 跨鏈？ |
|:--------|:---------------------|:-------------------|:-------------|
| 代幣位址 | `MintAddress` | `SmartContract` | 兩者在所有鏈皆可用 |
| 交易 ID | `Signature` / `Hash` | `Hash` | 兩者在所有鏈皆可用 |
| 區塊識別 | `Slot` | `Number` | 兩者在所有鏈皆可用 |
| 程式／合約 | `ProgramAddress` | `SmartContract` | 兩者在所有鏈皆可用 |
| 付費者 | `FeePayer` | `From` | 兩者在所有鏈皆可用 |

<Tip>
這表示你可以用**同一套查詢範本**套用到所有鏈，而不必改欄位名稱。例如 `Currency { MintAddress }` 在 Solana 與 EVM 查詢都能用。
</Tip>

## Trading

**Trading** 群組提供**跨鏈預先彙總**的交易分析。它將所有支援鏈的資料合併到具備 `chain` 維度、可篩選的統一 materialized view。

### Cubes

- `Pairs` — OHLC K 線（開／高／低／收、成交量、成交筆數）
- `Tokens` — 單一代幣交易統計（成交量、買賣拆分、不重複交易者數）

### 與 EVM／Solana 群組的主要差異

1. **沒有 `network` 引數** — Trading 群組跨鏈合併資料。請在資料內用 `chain` 維度依網路篩選。
2. **跨鏈資料** — 單一查詢可同時回傳 `sol`、`eth`、`bsc` 等資料。
3. **預先彙總** — 資料由 DWM 層以分鐘粒度物化。

### 範例：跨鏈 OHLC

```graphql
query {
  Trading(aggregates: yes) {
    Pairs(
      where: { Token: { Address: { is: "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v" } } }
      limit: {count: 60}
      orderBy: {descending: Block_Time}
    ) {
      Token { Address }
      Market { Network }
      Interval { Time }
      Price { Ohlc { Open High Low Close } }
      Volume { Usd Native }
      Stats { TradeCount BuyCount SellCount }
    }
  }
}
```

### 範例：依鏈的代幣交易統計

```graphql
query {
  Trading {
    Tokens(
      where: {
        Token: { Address: { is: "0xdac17f958d2ee523a2206206994597c13d831ec7" } }
        Market: { Network: { is: "eth" } }
      }
      limit: {count: 30}
      orderBy: {descending: Block_Time}
    ) {
      Interval { Time }
      Volume { Usd BuyVolumeUSD SellVolumeUSD }
      Stats { TradeCount UniqueBuyers UniqueSellers }
    }
  }
}
```

## Chain Group 對照

| 特性 | EVM | Solana | Trading |
|:--------|:----|:-------|:--------|
| **`network` 引數** | 必填（`eth`、`bsc`、`polygon`） | 不需要（隱式 `sol`） | 不需要（跨鏈） |
| **獨有 Cube** | Events、Calls、MinerRewards、Uncles、DEXPoolSlippages、Prediction* | Instructions、InstructionBalanceUpdates、Rewards、DEXOrders | Pairs、Tokens |
| **資料粒度** | 單筆事件（DWD）+ 彙總（DWM／DWS） | 單筆事件 + instruction 層級 | 預先彙總（DWM） |
| **`dataset` 支援** | 有（多數 Cube） | 有（多數 Cube） | 有 |
| **`aggregates` 支援** | 有 | 有 | 有 |
