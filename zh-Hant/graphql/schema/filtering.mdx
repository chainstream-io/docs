在 ChainStream GraphQL 中篩選資料有兩種方式：

1. **Selector shortcuts（選擇器捷徑）**——頂層參數如 `tokenAddress`，為常見篩選提供簡便寫法
2. **`where` 參數**——巢狀篩選物件，支援完整運算子與任意維度篩選

<Warning>
**最佳實務：務必加上時間篩選。** 對於 DWD（明細）類 Cube，如 DEXTrades、Transfers、BalanceUpdates，若查詢未帶 `Block.Time` 篩選，可能掃描極大的表分區。請一律加入 `where: {Block: {Time: {after: "..."}}}` 以限制掃描範圍，並避免 OLAP 引擎觸及記憶體上限。
</Warning>

同一查詢中可同時使用兩者。

---

## Selector shortcuts（選擇器捷徑）

Selectors 是 Cube 欄位上的**便利參數**，對應常見的 `where` 篩選模式。它們接受與 `where` 欄位相同的篩選輸入型別（例如含 `is`、`in`、`like` 等的 `StringFilter`），而非純字串。

| Selector | 對應至 | 適用 Cube |
|:---------|:--------|:-------------|
| `tokenAddress` | 各 Cube 的主要代幣地址欄 | DEXTrades、Transfers、BalanceUpdates、DEXPools、TokenSupplyUpdates、Pairs、Tokens、DEXPoolEvents、TokenHolders |
| `walletAddress` | 錢包／帳戶擁有者地址 | DEXTrades、WalletTokenPnL |
| `poolAddress` | 池合約地址 | DEXTrades、DEXPools |
| `senderAddress` | 轉帳發送方地址 | Transfers |
| `receiverAddress` | 轉帳接收方地址 | Transfers |
| `ownerAddress` | Token 帳戶擁有者地址 | BalanceUpdates |
| `dexProgram` | DEX 程式／合約地址 | DEXTrades |
| `date` | 區塊時間戳（DateTime 篩選） | DEXTrades、Transfers、BalanceUpdates、DEXPools |

**以下兩則查詢等價：**

<Tabs>
  <Tab title="使用 Selector">
```graphql
query {
  Solana {
    DEXTrades(
      tokenAddress: {is: "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"}
      limit: { count: 10 }
    ) {
      Block { Time }
      Trade { Buy { Amount PriceInUSD } }
    }
  }
}
```
  </Tab>
  <Tab title="使用 where 篩選">
```graphql
query {
  Solana {
    DEXTrades(
      where: {
        Trade: {
          Buy: {
            Currency: {
              MintAddress: {
                is: "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"
              }
            }
          }
        }
      }
      limit: { count: 10 }
    ) {
      Block { Time }
      Trade { Buy { Amount PriceInUSD } }
    }
  }
}
```
  </Tab>
</Tabs>

<Tip>
Selectors 支援所有篩選運算子——不僅限於完全相等。例如 `tokenAddress: {in: ["ADDR_1", "ADDR_2"]}` 可匹配多個代幣，`date: {after: "2025-01-01T00:00:00Z"}` 可依時間篩選。
</Tip>

## `where` 參數

`where` 接受巢狀輸入物件，結構鏡射 Cube 的維度階層。每個葉節欄位對應一個具型別運算子的**篩選 primitive**。

### 結構

```graphql
where: {
  DimensionGroup: {
    DimensionField: {
      operator: value
    }
  }
}
```

**範例**——篩選區塊時間在某日之後，且買入數量大於 1000 的 DEXTrades：

```graphql
query {
  Solana {
    DEXTrades(
      where: {
        Block: { Time: { after: "2025-01-01T00:00:00Z" } }
        Trade: { Buy: { Amount: { gt: 1000 } } }
      }
      limit: { count: 20 }
    ) {
      Block { Time }
      Trade { Buy { Amount PriceInUSD } }
    }
  }
}
```

當 `where` 同一層級指定多個欄位時，會以 **AND** 邏輯組合。

## 篩選 primitive 型別

每個葉維度會對應下列其中一種篩選輸入型別：

### StringFilter

用於地址、雜湊、協定名稱等文字欄位。

| 運算子 | 型別 | 說明 |
|:---------|:-----|:------------|
| `is` | `String` | 完全相等 |
| `not` | `String` | 不相等 |
| `in` | `[String!]` | 符合清單中任一值 |
| `notIn` | `[String!]` | 排除清單中所有值 |
| `like` | `String` | SQL LIKE 模式比對（`%` 萬用字元） |

```graphql
where: {
  Trade: {
    Dex: {
      ProtocolName: { in: ["Raydium", "Orca"] }
    }
  }
}
```

### IntFilter / FloatFilter

用於數量、價格、計數等數值欄位。

| 運算子 | 型別 | 說明 |
|:---------|:-----|:------------|
| `eq` | `Number` | 等於 |
| `ne` | `Number` | 不等於 |
| `gt` | `Number` | 大於 |
| `gte` | `Number` | 大於或等於 |
| `lt` | `Number` | 小於 |
| `lte` | `Number` | 小於或等於 |
| `in` | `[Number!]` | 符合清單中任一值 |
| `notIn` | `[Number!]` | 排除清單中所有值 |

```graphql
where: {
  Trade: {
    Buy: {
      PriceInUSD: { gte: 0.001, lte: 1.0 }
    }
  }
}
```

### DateTimeFilter

用於時間戳欄位。值為 ISO 8601 字串。

| 運算子 | 型別 | 說明 |
|:---------|:-----|:------------|
| `after` | `DateTime` | 在此時間戳之後（不含邊界） |
| `before` | `DateTime` | 在此時間戳之前（不含邊界） |
| `since` | `DateTime` | 自此刻起（含邊界） |
| `till` | `DateTime` | 直到此刻（含邊界） |
| `between` | `[DateTime!, DateTime!]` | 介於兩個時間戳之間（兩端皆含） |

```graphql
where: {
  Block: {
    Time: { since: "2025-03-01T00:00:00Z", till: "2025-03-31T23:59:59Z" }
  }
}
```

<Note>
`after`／`before` 為**不含邊界**（嚴格不等）。`since`／`till` 為**含邊界**。`between` 接受兩個元素的陣列，兩端皆含。
</Note>

### BoolFilter

用於布林欄位。

| 運算子 | 型別 | 說明 |
|:---------|:-----|:------------|
| `eq` | `Boolean` | 等於 `true` 或 `false` |

```graphql
where: {
  IsSuspect: { eq: true }
}
```

## 以 `any` 表達 OR 邏輯

預設 `where` 內所有條件以 AND 組合。若要 OR，請使用 `any` 陣列欄位——每個元素皆為完整篩選物件，符合**任一**條件的紀錄都會回傳。

```graphql
query {
  Solana {
    DEXTrades(
      where: {
        any: [
          { Trade: { Buy: { Currency: { MintAddress: { is: "TOKEN_A" } } } } }
          { Trade: { Buy: { Currency: { MintAddress: { is: "TOKEN_B" } } } } }
        ]
      }
      limit: { count: 20 }
    ) {
      Trade {
        Buy { Currency { MintAddress } Amount }
      }
    }
  }
}
```

<Info>
`any` 可與其他頂層 `where` 欄位並用。`any` 內條件先以 OR 組合，再與同層其他條件以 AND 組合。
</Info>

## 預設篩選

部分 Cube 會自動套用預設篩選。你可在 `where` 子句中明確設定以覆寫。

| Cube | 預設篩選 | 效果 |
|:-----|:---------------|:-------|
| DEXTrades | `IsSuspect = false` | 預設排除 bot／MEV 成交 |

若要**納入** suspect 成交，請明確設定篩選：

```graphql
query {
  Solana {
    DEXTrades(
      where: { IsSuspect: { eq: true } }
      limit: { count: 10 }
    ) {
      Block { Time }
      Trade { Buy { Amount } }
      IsSuspect
    }
  }
}
```

若完全不在 `where` 指定 `IsSuspect`——預設 `false` 仍會生效。若要查詢**不論** suspect 狀態的**所有**成交，請使用 OR：

```graphql
where: {
  any: [
    { IsSuspect: { eq: true } }
    { IsSuspect: { eq: false } }
  ]
}
```

## 合併 Selectors 與 `where`

Selectors 與 `where` 以 AND 合併。因此可用 selector 鎖定主要實體，再以 `where` 細化條件：

```graphql
query {
  Solana {
    DEXTrades(
      tokenAddress: {is: "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"}
      where: {
        Block: { Time: { after: "2025-03-01T00:00:00Z" } }
        Trade: { Buy: { Amount: { gt: 100 } } }
      }
      orderBy: {descending: Block_Time}
      limit: { count: 50 }
    ) {
      Block { Time }
      Transaction { Hash }
      Trade {
        Buy { Amount PriceInUSD }
        Sell { Currency { MintAddress } Amount }
        Dex { ProtocolName }
      }
    }
  }
}
```

此查詢取得 Solana 上 USDC 最近 50 筆成交（買入數量大於 100），並依時間遞減排序。
