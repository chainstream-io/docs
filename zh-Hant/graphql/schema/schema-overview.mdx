## 動態 Schema 產生

ChainStream GraphQL schema 在啟動時由 **activecube-rs**（Rust 函式庫）**動態產生**，將 **Cube** 定義編譯為完整型別的 [async-graphql](https://github.com/async-graphql/async-graphql) schema。每個 Cube 對應由 OLAP 表支撐的分析資料模型，activecube-rs 會自動產生：

- Cube 的**頂層 Query 欄位**（巢狀於其 Chain Group 下）
- **Record 型別**（`{Cube}Record`），代表可選取的維度
- **Filter 輸入**（`{Cube}Filter`），結構對應維度階層
- **OrderBy 列舉**（`{Cube}OrderBy`），涵蓋各維度路徑的升序／降序變體

因此 schema 永遠與底層資料模型同步——無需手寫 SDL 維護。

<Info>
由於 schema 自 Cube 定義產生，Rust 中新增的任何資料模型在部署後都會自動反映於 GraphQL 端點。
</Info>

## 根查詢結構

根查詢型別名為 `ChainStream`。Cube 依三個 **Chain Groups** 組織，各為頂層欄位：

```graphql
type ChainStream {
  EVM(network: Network!, dataset: Dataset, aggregates: Aggregates) {
    DEXTrades(...): [DEXTradesRecord!]!
    Transfers(...): [TransfersRecord!]!
    BalanceUpdates(...): [BalanceUpdatesRecord!]!
    Blocks(...): [BlocksRecord!]!
    Transactions(...): [TransactionsRecord!]!
    Events(...): [EventsRecord!]!
    Calls(...): [CallsRecord!]!
    # ... more EVM Cubes
  }

  Solana(dataset: Dataset, aggregates: Aggregates) {
    DEXTrades(...): [DEXTradesRecord!]!
    Instructions(...): [InstructionsRecord!]!
    DEXOrders(...): [DEXOrdersRecord!]!
    # ... more Solana Cubes
  }

  Trading(dataset: Dataset, aggregates: Aggregates) {
    Pairs(...): [PairsRecord!]!
    Tokens(...): [TokensRecord!]!
  }
}
```

沒有 `Mutation` 或 `Subscription` 型別——GraphQL API 僅供唯讀分析查詢。

## Chain Groups

Cube 依目標區塊鏈生態分成三組：

| Chain Group | `network` 參數 | 可用網路 | 說明 |
|:--|:--|:--|:--|
| **EVM** | 必填 | `eth`、`bsc`、`polygon` | 所有 EVM 相容鏈共用的 Cube |
| **Solana** | 不需要 | `sol`（隱含） | Solana 專用 Cube（含 Instructions、DEXOrders 等） |
| **Trading** | 不需要 | 跨鏈（`sol`、`eth`、`bsc`） | 預先聚合的交易分析（OHLC K 線、代幣統計），內含 `chain` 維度 |

<Note>
**EVM** 群組必須傳入 `network` 以選擇鏈。**Solana** 與 **Trading** 不需要 `network`——Solana 為隱含，Trading 則在資料內含 `chain` 維度。
</Note>

完整各群組所含 Cube 請見 [Chain Groups](/zh-Hant/graphql/schema/chain-groups)。

## Chain Group 參數

每個 Chain Group 接受兩個可選參數，控制資料來源行為：

### Dataset

`dataset` 控制查詢資料的時間範圍：

| 值 | 說明 |
|:------|:------------|
| `combined` | 完整區間——同時查詢近期與歷史資料**（預設）** |
| `realtime` | 僅近期資料（約最近 24 小時） |
| `archive` | 歷史資料，上限為保留 TTL |

```graphql
query {
  Solana(dataset: realtime) {
    DEXTrades(limit: {count: 10}, orderBy: {descending: Block_Time}) {
      Block { Time }
      Trade { Buy { Amount } }
    }
  }
}
```

### Aggregates

`aggregates` 控制是否使用預先聚合（DWM／DWS）表：

| 值 | 說明 |
|:------|:------------|
| `yes` | 在可用時優先使用預先聚合表**（適用 Cube 的預設）** |
| `no` | 僅使用原始明細表 |
| `only` | 僅使用預先聚合表（較快但欄位較受限） |

```graphql
query {
  Trading(aggregates: only) {
    Pairs(
      where: { Token: { Address: { is: "0x..." } } }
      limit: {count: 100}
    ) {
      Interval { Time }
      Price { Ohlc { Open High Low Close } }
      Volume { Usd }
    }
  }
}
```

<Tip>
詳細用法、支援的表與效能建議請見 [Dataset & Aggregates](/zh-Hant/graphql/schema/dataset-aggregates)。
</Tip>

## 常見參數模式

在 Chain Group 內，每個 Cube 欄位接受同一組標準參數，外加可選的 Cube 專屬 **selectors**：

| 參數 | 型別 | 必填 | 說明 |
|:---------|:-----|:---------|:------------|
| `where` | `{Cube}Filter` | 否 | 巢狀篩選物件，對應維度階層 |
| `limit` | `LimitInput` | 否 | 分頁：`{count: Int, offset: Int}` |
| `orderBy` | `{Cube}OrderBy` | 否 | 排序列舉（`{Path}_ASC`／`{Path}_DESC`） |
| *selectors* | Filter 輸入 | 否 | 捷徑篩選（例如 `tokenAddress: {is: "..."}`） |

```graphql
query {
  Solana {
    DEXTrades(
      tokenAddress: {is: "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"}
      where: { Block: { Time: { after: "2026-01-01T00:00:00Z" } } }
      limit: { count: 50, offset: 0 }
      orderBy: {descending: Block_Time}
    ) {
      Block { Time }
      Trade { Buy { Amount PriceInUSD } }
    }
  }
}
```

### LimitInput

```graphql
input LimitInput {
  count: Int   # Number of rows to return
  offset: Int  # Number of rows to skip (for pagination)
}
```

預設 `count` 依 Cube 而異（多為 25）。多數 Cube 上限為 10,000。

## 每個 Cube 的產生型別

對每個 Cube，activecube-rs 會產生三個配套型別：

<CardGroup cols={3}>
  <Card title="Record 型別" icon="table">
    `{Cube}Record`——回傳型別，含所有可選維度與指標。欄位結構鏡射 Cube 的維度階層。
  </Card>
  <Card title="Filter 輸入" icon="filter">
    `{Cube}Filter`——巢狀輸入物件，各維度對應篩選 primitive（`StringFilter`、`IntFilter`、`DateTimeFilter` 等）。
  </Card>
  <Card title="OrderBy 列舉" icon="arrow-down-a-z">
    `{Cube}OrderBy`——各維度路徑在升序與降序的列舉變體（例如 `Block_Time_ASC`、`Trade_Buy_Amount_DESC`）。
  </Card>
</CardGroup>

**DEXTrades 範例：**

```graphql
# Record type (return shape)
type DEXTradesRecord {
  Block: DEXTradesBlockRecord
  Transaction: DEXTradesTransactionRecord
  Trade: DEXTradesTradeRecord
  Pool: DEXTradesPoolRecord
  IsSuspect: Boolean
  count: Int
  sum(of: DEXTradesSumOf!): Float
}

# Filter input
input DEXTradesFilter {
  Block: DEXTradesBlockFilter
  Transaction: DEXTradesTransactionFilter
  Trade: DEXTradesTradeFilter
  Pool: DEXTradesPoolFilter
  IsSuspect: BoolFilter
  any: [DEXTradesFilter!]  # OR logic
}

# OrderBy enum (partial)
enum DEXTradesOrderBy {
  Block_Time_ASC
  Block_Time_DESC
  Trade_Buy_Amount_ASC
  Trade_Buy_Amount_DESC
  # ...
}
```

## Introspection

Schema 支援標準 GraphQL introspection。可使用 `__schema` 與 `__type` 查詢探索型別、欄位與參數：

<Tabs>
  <Tab title="列出所有 Cube">
```graphql
query {
  __schema {
    queryType {
      fields {
        name
        description
        args { name type { name } }
      }
    }
  }
}
```
  </Tab>
  <Tab title="檢視某 Cube 型別">
```graphql
query {
  __type(name: "DEXTradesRecord") {
    name
    fields {
      name
      type { name kind ofType { name } }
    }
  }
}
```
  </Tab>
  <Tab title="列出 Filter 運算子">
```graphql
query {
  __type(name: "StringFilter") {
    name
    inputFields {
      name
      type { name kind }
    }
  }
}
```
  </Tab>
</Tabs>

<Tip>
[GraphQL IDE](https://ide.chainstream.io) 會自動抓取 introspection schema 以驅動自動完成與內嵌文件。你可互動探索完整 schema，無須手寫 introspection 查詢。
</Tip>
