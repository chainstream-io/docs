## MCP Prompts

ChainStream MCP 伺服器在 `registerAllPrompts` 中註冊 **4 個** prompt。引數名與是否必填與傳給 `server.prompt` 的 Zod 引數模式一致。

### `token_analysis`

**程式碼中的描述：** 多維度評估代幣的深度分析模板

| 引數 | 型別 | 必填 | Zod 說明 |
|-----------|------|----------|-----------------|
| `chain` | string | 是 | 區塊鏈（solana, ethereum, bsc） |
| `token` | string | 是 | 代幣地址或符號 |

---

### `wallet_profiling`

**程式碼中的描述：** 分析交易模式與績效的錢包畫像模板

| 引數 | 型別 | 必填 | Zod 說明 |
|-----------|------|----------|-----------------|
| `chain` | string | 是 | 區塊鏈 |
| `address` | string | 是 | 錢包地址 |

---

### `trading_strategy`

**程式碼中的描述：** 根據自然語言描述生成並回測交易策略

| 引數 | 型別 | 必填 | Zod 說明 |
|-----------|------|----------|-----------------|
| `description` | string | 是 | 自然語言策略描述 |
| `chain` | string | 否 | 目標區塊鏈 |
| `token` | string | 否 | 目標代幣 |

---

### `risk_assessment`

**程式碼中的描述：** 針對代幣或交易頭寸的綜合風險評估

| 引數 | 型別 | 必填 | Zod 說明 |
|-----------|------|----------|-----------------|
| `chain` | string | 是 | 區塊鏈 |
| `token` | string | 是 | 代幣地址 |

## MCP Resources

伺服器在 `registerAllResources` 中註冊 **4 個** resource：三個固定 URI 與一個 `ResourceTemplate`。

### `chainstream://chains`

**程式碼中的資源名：** `chains`  
**MIME 型別：** `application/json`

處理函式返回程式碼中的 `CHAINS_DATA` —— 完整 JSON 負載如下：

```json
{ "chains": [{ "id": "solana", "name": "Solana", "status": "active", "avgBlockTime": "400ms" }, { "id": "ethereum", "name": "Ethereum", "status": "active", "avgBlockTime": "12s" }, { "id": "bsc", "name": "BNB Smart Chain", "status": "active", "avgBlockTime": "3s" }] }
```

為便於閱讀而格式化：

```json
{
  "chains": [
    { "id": "solana", "name": "Solana", "status": "active", "avgBlockTime": "400ms" },
    { "id": "ethereum", "name": "Ethereum", "status": "active", "avgBlockTime": "12s" },
    { "id": "bsc", "name": "BNB Smart Chain", "status": "active", "avgBlockTime": "3s" }
  ]
}
```

---

### `chainstream://docs/api_reference`

**程式碼中的資源名：** `api-reference`  
**MIME 型別：** `text/markdown`

正文由 `API_REFERENCE` 常量生成（基礎 URL、Data API 與 DeFi API 端點列表）。

---

### `chainstream://docs/query_examples`

**程式碼中的資源名：** `query-examples`  
**MIME 型別：** `text/markdown`

正文由 `QUERY_EXAMPLES` 常量生成（面向智慧體的常見查詢模式）。

---

### `chainstream://chains/{chain}/dexes`

**程式碼中的資源名：** `chain-dexes`  
**型別：** `ResourceTemplate('chainstream://chains/{chain}/dexes', { list: undefined })`  
**MIME 型別：** `application/json`

處理函式解析 `variables.chain` 並以 JSON 響應：`{ "chain": "<resolved>", "dexes": string[] }`，其中 `dexes` 為 `DEXES_BY_CHAIN[chain]`，未知鏈則為 `[]`。

示例 URI：`chainstream://chains/solana/dexes`。

## 支援的鏈

程式碼中 `CHAINS_DATA` / `DEXES_BY_CHAIN` 配置覆蓋 **3** 條鏈：

| 識別符號 | 展示名稱（來自 `chains` resource） |
|------------|--------------------------------------|
| `solana` | Solana |
| `ethereum` | Ethereum |
| `bsc` | BNB Smart Chain |

## 按鏈劃分的 DEX

取值與 `resources/index.ts` 中的 `DEXES_BY_CHAIN` 一致（協議 id 與程式碼中相同，小寫）：

| 鏈 | `dexes` 陣列 |
|-------|----------------|
| `solana` | `jupiter`, `raydium`, `pumpfun`, `moonshot`, `candy` |
| `ethereum` | `kyberswap` |
| `bsc` | `kyberswap` |
