ChainStream MCP Server 支援所有 API Reference 中定義的介面，AI 可以直接呼叫這些工具獲取鏈上資料。

<Note>
MCP 工具的引數和返回值與 REST API 完全一致，詳細欄位說明請參考 [API Reference](/zh-Hant/api-reference/authentication/authenticate)。
</Note>

## Token 工具

### token_search

搜尋代幣。

```
搜索 PEPE 相关的代币
```

### token_get

獲取代幣詳細資訊。

```
查看代币 So11111111111111111111111111111111111111112 的详细信息
```

### token_get_metadata

獲取代幣後設資料（名稱、符號、Logo 等）。

```
获取这个代币的 Logo 和社交媒体链接
```

### token_get_stats

獲取代幣統計資料（交易量、買賣次數等）。

```
查看这个代币过去 24 小时的交易统计
```

### token_get_price

獲取代幣當前價格。

```
BONK 现在的价格是多少？
```

### token_get_prices

獲取代幣歷史價格。

```
查看 ETH 过去一周的价格走势
```

### token_get_candles

獲取代幣 K 線資料。

```
获取 SOL 最近 100 根 1 小时 K 线
```

### token_get_holders

獲取代幣持有者資訊。

```
这个代币有多少持有者？
```

### token_get_top_holders

獲取代幣 Top 持有者列表。

```
查看这个代币的前 10 大持有者
```

### token_get_pools

獲取代幣相關的流動池。

```
这个代币在哪些 DEX 有流动池？
```

### token_get_security

獲取代幣安全檢查結果。

```
帮我检查这个代币是否安全
```

### token_get_creation

獲取代幣建立資訊。

```
这个代币是什么时候创建的？创建者是谁？
```

### token_get_mint_and_burn

獲取代幣鑄造和銷燬記錄。

```
查看这个代币的铸造和销毁历史
```

### token_list

獲取代幣列表（支援分頁和篩選）。

```
列出 Solana 上最近创建的代币
```

### token_get_traders_tag

獲取代幣交易者標籤分佈。

```
分析这个代币的交易者都是什么类型的
```

### token_get_liquidity_snapshots

獲取代幣流動性快照。

```
查看这个代币过去一周的流动性变化
```

### token_dev_get_tokens

獲取開發者建立的所有代幣。

```
查看这个开发者还创建过哪些代币
```

## WebSocket 訂閱

MCP 服務也支援 WebSocket 實時訂閱，可以獲取以下實時資料：

### 訂閱型別

| 型別 | 說明 | 示例提問 |
|------|------|----------|
| K線 | 實時價格 K 線 | "訂閱這個代幣的 1 分鐘 K 線" |
| 代幣統計 | 實時交易統計 | "實時監控這個代幣的交易資料" |
| 新代幣 | 新代幣建立通知 | "通知我 Solana 上的新代幣" |
| 代幣後設資料 | 新代幣後設資料 | "獲取新代幣的詳細資訊" |
| 錢包餘額 | 餘額變動 | "監控這個錢包的餘額變化" |
| 錢包交易 | 交易通知 | "這個錢包有新交易時通知我" |
| 交易活動 | 實時交易流 | "顯示這個代幣的實時交易" |
| 流動池 | 池子狀態 | "監控這個池子的流動性變化" |
| 排行榜 | 熱門代幣更新 | "實時更新熱門代幣排行" |

<Note>
WebSocket 訂閱的詳細引數和返回格式請參考 [WebSocket API 文件](/zh-Hant/api-reference/endpoint/websocket/api)。
</Note>

## 提示詞最佳實踐

### 明確指定鏈

```
✅ 好：查询 0x... 在以太坊上的余额
❌ 差：查询 0x... 的余额
```

### 提供完整地址

```
✅ 好：分析 0x1234567890abcdef1234567890abcdef12345678
❌ 差：分析 0x1234...5678
```

### 指定時間範圍

```
✅ 好：查看最近 7 天的交易记录
❌ 差：查看最近的交易
```

### 分步驟提問

複雜任務拆分為多個簡單問題：

```
✅ 好：
1. 首先查询这个地址的持仓
2. 然后分析它的交易历史
3. 最后判断它的投资风格

❌ 差：
告诉我这个地址的一切信息
```

### 利用上下文追問

```
用户：查询 0x... 的持仓
AI：[返回持仓列表]
用户：这些代币中哪个涨幅最大？
用户：分析持有 PEPE 的原因
```
