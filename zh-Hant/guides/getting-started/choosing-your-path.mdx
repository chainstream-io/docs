## 資料使用者

### 適用場景

需要獲取鏈上資料用於展示、分析或交易決策的開發者。

**典型用例：**
- 在 dApp 中展示代幣實時價格
- 構建行情看板或資料分析工具
- 監控特定錢包的持倉變化
- 開發交易機器人獲取市場資料

**典型使用者：**
- DeFi 應用開發者
- 行情資料展示平臺
- 鏈上資料分析師
- 交易機器人開發者

### 推薦路徑

<Steps>
  <Step title="理解資料模型">
    閱讀 [資料概念](/zh-Hant/guides/data-concepts/realtime-streaming) 瞭解 ChainStream 的資料結構
  </Step>
  <Step title="呼叫代幣 API">
    檢視 [Token API 參考](/zh-Hant/api-reference/endpoint/token/v1/token-chain-tokenaddress-get) 獲取價格和代幣資訊
  </Step>
  <Step title="實戰：價格預警">
    跟隨 [價格預警機器人教程](/zh-Hant/playbooks/tutorials/build-price-alert-bot) 構建完整應用
  </Step>
</Steps>

### 核心能力

你將使用到以下能力：
- 實時價格資料（WebSocket 推送）
- 多鏈代幣資訊查詢
- 交易和持倉資料
- Smart Money 錢包追蹤

<Note>
**預計時間**：完成推薦路徑約需 1-2 小時
</Note>

## GraphQL 分析師

### 適用場景

需要對鏈上 OLAP 資料進行靈活、宣告式查詢並具備聚合能力的開發者和分析師。

**典型用例：**
- 分析多鏈 DEX 交易模式
- 用聚合的鏈上指標構建自定義看板
- 查詢代幣持有人分佈和錢包 PnL
- 對交易、轉賬和池進行即席分析查詢

**典型使用者：**
- 鏈上資料分析師
- 量化研究員
- 看板和分析工具開發者
- DeFi 協議團隊

### 推薦路徑

<Steps>
  <Step title="瞭解 GraphQL API">
    閱讀 [GraphQL 概覽](/zh-Hant/graphql/getting-started/overview) 瞭解 OLAP 分析型 API
  </Step>
  <Step title="執行第一個查詢">
    跟隨 [第一個查詢指南](/zh-Hant/graphql/getting-started/first-query) 使用 GraphQL IDE
  </Step>
  <Step title="探索 Data Cubes">
    瞭解 [12 個 Data Cubes](/zh-Hant/graphql/schema/cubes)，涵蓋交易、轉賬、餘額等
  </Step>
  <Step title="嘗試查詢示例">
    瀏覽 [DEX 交易示例](/zh-Hant/graphql/examples/dex-trades) 和其他查詢示例
  </Step>
</Steps>

### 核心能力

你將使用到以下能力：
- 單一 GraphQL 端點查詢所有鏈上資料
- 12 個 Data Cubes 涵蓋 DEX 交易、轉賬、餘額、池、OHLC 和代幣指標
- 內建聚合功能（count、sum、avg、min、max、uniq）
- 靈活的過濾、排序和分頁
- 瀏覽器端 GraphQL IDE，內建模板和程式碼匯出

<Note>
**預計時間**：完成推薦路徑約需 1-2 小時
</Note>

## 合規使用者

### 適用場景

需要評估交易風險或分析地址安全性的團隊。

**典型用例：**
- 檢測使用者充值是否來自高風險地址
- 分析錢包地址的歷史行為
- 生成交易風險評估報告
- 整合風控流程到業務系統

**典型使用者：**
- 交易所合規團隊
- 錢包安全團隊
- 風控系統開發者

### 推薦路徑

<Steps>
  <Step title="理解 KYT/KYA">
    閱讀 [安全合規概述](/zh-Hant/guides/security-compliance/overview) 瞭解風控能力
  </Step>
  <Step title="呼叫風險評估 API">
    檢視 [KYT API 參考](/zh-Hant/api-reference/endpoint/kyt/v1/kyt-transfer-post) 瞭解交易風險評估介面
  </Step>
  <Step title="實戰：充值風控">
    跟隨 [充值風控整合教程](/zh-Hant/playbooks/tutorials/integrate-deposit-check) 實現完整流程
  </Step>
</Steps>

### 核心能力

你將使用到以下能力：
- KYT 交易風險評估（返回風險分數和標籤）
- KYA 地址畫像分析（地址型別、歷史行為）
- 風險標籤體系（可配置的風險規則）

<Note>
**預計時間**：完成推薦路徑約需 2-3 小時
</Note>

## AI 開發者

### 適用場景

希望讓 AI Agent 能夠查詢和理解鏈上資料的開發者。

**典型用例：**
- 讓 Claude/GPT 回答鏈上資料問題
- 構建能理解加密市場的 AI 助手
- 在 AI 工作流中整合鏈上資料查詢

**典型使用者：**
- AI Agent 開發者
- 自動化交易系統開發者
- LLM 應用開發者
- MCP 整合開發者

### 推薦路徑

<Steps>
  <Step title="理解 AI 能力">
    閱讀 [AI 基礎設施概述](/zh-Hant/guides/ai-infrastructure/overview) 瞭解 MCP 協議
  </Step>
  <Step title="配置 MCP Server">
    按照 [MCP 配置指南](/zh-Hant/guides/ai-infrastructure/mcp-server/setup-guide) 完成接入
  </Step>
  <Step title="實戰：AI 助理">
    跟隨 [AI 交易助理教程](/zh-Hant/playbooks/tutorials/ai-agent-with-mcp) 構建完整應用
  </Step>
</Steps>

### 核心能力

你將使用到以下能力：
- MCP Server 整合（支援 Claude Desktop 等客戶端）
- AI 友好的工具集（結構化輸出）
- 自然語言查詢（如"查詢 ETH 最近 24 小時價格"）

<Note>
**預計時間**：完成推薦路徑約需 1-2 小時
</Note>

## AI Agent 開發者

### 適用場景

構建需要資料訪問和 DeFi 執行能力的 AI Agent，透過 CLI 或 Agent Skills 實現。

**典型用例：**
- 構建能查詢鏈上資料並執行交易的 AI Agent
- 自動化代幣研究和投資組合管理
- 將 ChainStream 整合到 CI/CD 流水線
- 在 Cursor、Claude Code 或 Codex 中使用 Agent Skills

**典型使用者：**
- AI Agent 開發者
- 自動化鏈上工作流的 DevOps 工程師
- Cursor / Claude Code 高階使用者

### 推薦路徑

<Steps>
  <Step title="安裝 CLI">
    閱讀 [CLI 概覽](/zh-Hant/guides/cli/overview) 並執行 `npx @chainstream-io/cli login`
  </Step>
  <Step title="安裝 Agent Skills">
    按照 [Skills 安裝指南](/zh-Hant/guides/ai-infrastructure/agent-skills/installation) 配置你的平臺
  </Step>
  <Step title="探索資料查詢">
    學習 [chainstream-data skill](/zh-Hant/guides/ai-infrastructure/agent-skills/chainstream-data) 進行代幣研究和錢包分析
  </Step>
  <Step title="執行 DeFi（可選）">
    學習 [chainstream-defi skill](/zh-Hant/guides/ai-infrastructure/agent-skills/chainstream-defi) 進行兌換和代幣建立
  </Step>
</Steps>

### 核心能力

你將使用到以下能力：
- CLI 命令用於代幣、市場、錢包和 DeFi 操作
- Agent Skills 用於結構化 AI 工作流
- x402 自動支付實現無縫訂閱管理
- MCP Server 整合用於 AI 聊天助手

<Note>
**預計時間**：完成推薦路徑約需 30-60 分鐘
</Note>
