## 版本策略

### API 版本

- **主版本** (v1, v2)：包含破壞性變更
- **次版本** (v1.1, v1.2)：新功能，向後相容
- **補丁** (v1.1.1)：Bug 修復

### SDK 版本

SDK 遵循語義化版本 (SemVer)：
- **Major**：破壞性變更
- **Minor**：新功能
- **Patch**：Bug 修復

### 廢棄策略

- 廢棄的端點至少保留 6 個月
- 廢棄前會在響應頭中新增警告
- 重大變更會提前 30 天通知
