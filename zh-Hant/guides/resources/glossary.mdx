## 區塊鏈術語

基礎區塊鏈概念和通用術語。

### A

<Accordion title="Address（地址）">
區塊鏈上的唯一識別符號，用於傳送和接收資產。以太坊地址以 `0x` 開頭，長度為 42 個字元。

示例：`0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045`
</Accordion>

<Accordion title="ABI (Application Binary Interface)">
應用二進位制介面。智慧合約的介面定義，描述了合約的函式和資料結構，用於與合約互動。
</Accordion>

### B

<Accordion title="Block（區塊）">
區塊鏈中儲存交易資料的基本單位。每個區塊包含一組交易、時間戳和前一區塊的雜湊值。

| 欄位 | 說明 |
|------|------|
| Block Number | 區塊高度 |
| Block Hash | 區塊雜湊 |
| Timestamp | 出塊時間 |
| Transactions | 包含的交易 |
</Accordion>

<Accordion title="Block Confirmation（區塊確認）">
交易被打包進區塊後，後續產生的區塊數量。確認數越多，交易越不可逆。

| 鏈 | 建議確認數 |
|----|-----------|
| Ethereum | 12-32 |
| BSC | 15 |
| Solana | 32 |
</Accordion>

<Accordion title="Bridge（跨鏈橋）">
允許資產和資料在不同區塊鏈之間轉移的協議。常見跨鏈橋包括 LayerZero、Wormhole 等。
</Accordion>

### C

<Accordion title="Chain（鏈）">
區塊鏈網路。ChainStream 支援多條鏈，包括 Ethereum、BSC、Polygon、Arbitrum、Solana 等。

鏈標識示例：
- `ethereum` - 以太坊主網
- `bsc` - BNB Chain
- `arbitrum` - Arbitrum One
- `solana` - Solana
</Accordion>

<Accordion title="Contract Address（合約地址）">
智慧合約部署在區塊鏈上的唯一地址。與普通地址（EOA）不同，合約地址由程式碼控制。
</Accordion>

### E

<Accordion title="EOA (Externally Owned Account)">
外部擁有賬戶。由私鑰控制的普通錢包地址，與智慧合約地址相對。
</Accordion>

<Accordion title="ERC-20">
以太坊代幣標準。定義了代幣的基本功能介面，如 `transfer`、`approve`、`balanceOf` 等。
</Accordion>

<Accordion title="ERC-721">
以太坊 NFT 標準。定義了非同質化代幣的介面規範。
</Accordion>

<Accordion title="EVM (Ethereum Virtual Machine)">
以太坊虛擬機器。執行智慧合約程式碼的執行環境。Polygon、Arbitrum、Base 等都是 EVM 相容鏈。
</Accordion>

### G

<Accordion title="Gas">
執行區塊鏈交易所需支付的手續費單位。Gas 費 = Gas Used × Gas Price。

| 術語 | 說明 |
|------|------|
| Gas Limit | 交易最大 Gas 消耗 |
| Gas Used | 實際消耗的 Gas |
| Gas Price | 每單位 Gas 的價格 |
| Gwei | Gas Price 單位，1 Gwei = 10⁻⁹ ETH |
</Accordion>

### H

<Accordion title="Hash（雜湊）">
將任意長度資料轉換為固定長度字串的加密函式。區塊鏈中常用於交易雜湊、區塊雜湊等。

示例：`0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef`
</Accordion>

### L

<Accordion title="Layer 1 (L1)">
第一層區塊鏈。獨立執行的主鏈，如 Ethereum、Solana、Bitcoin。
</Accordion>

<Accordion title="Layer 2 (L2)">
第二層擴充套件方案。構建在 L1 之上的擴容解決方案，如 Arbitrum、Optimism、Base、zkSync。
</Accordion>

### M

<Accordion title="Mempool（記憶體池）">
待處理交易的暫存區。交易在被打包進區塊前，會在 mempool 中等待。
</Accordion>

### N

<Accordion title="NFT (Non-Fungible Token)">
非同質化代幣。具有唯一性的數字資產，常用於數字藝術、遊戲道具等。
</Accordion>

<Accordion title="Node（節點）">
執行區塊鏈軟體的計算機，參與網路驗證和資料儲存。
</Accordion>

<Accordion title="Nonce">
交易序號。每個地址傳送交易時的遞增計數器，防止重放攻擊。
</Accordion>

### R

<Accordion title="RPC (Remote Procedure Call)">
遠端過程呼叫。與區塊鏈節點互動的協議，用於傳送交易、查詢資料等。
</Accordion>

### S

<Accordion title="Smart Contract（智慧合約）">
在區塊鏈上自動執行的程式程式碼。一旦部署，程式碼邏輯不可更改（除非設計了升級機制）。
</Accordion>

### T

<Accordion title="Token（代幣）">
區塊鏈上的數字資產。包括原生代幣（如 ETH）和合約代幣（如 USDC）。
</Accordion>

<Accordion title="Transaction（交易）">
區塊鏈上的狀態變更操作，如轉賬、合約呼叫等。

| 欄位 | 說明 |
|------|------|
| TxHash | 交易雜湊，唯一標識 |
| From | 傳送方地址 |
| To | 接收方地址 |
| Value | 轉賬金額 |
| Input Data | 合約呼叫資料 |
</Accordion>

<Accordion title="Transaction Hash / TxHash（交易雜湊）">
交易的唯一識別符號。可用於在區塊瀏覽器上查詢交易詳情。
</Accordion>

### W

<Accordion title="Wallet（錢包）">
儲存和管理加密資產的工具。可以是軟體（如 MetaMask）或硬體（如 Ledger）。
</Accordion>

## DeFi 術語

去中心化金融相關概念。

### A

<Accordion title="AMM (Automated Market Maker)">
自動做市商。DEX 中用於提供流動性和定價的演算法機制，如 Uniswap 的恆定乘積公式 `x * y = k`。
</Accordion>

<Accordion title="APR (Annual Percentage Rate)">
年化利率。不考慮複利的年化收益百分比。
</Accordion>

<Accordion title="APY (Annual Percentage Yield)">
年化收益率。考慮複利後的年化收益百分比。

計算公式：`APY = (1 + APR/n)^n - 1`，其中 n 為複利次數。
</Accordion>

<Accordion title="ATH (All-Time High)">
歷史最高價。代幣或資產曾達到的最高價格。
</Accordion>

<Accordion title="ATL (All-Time Low)">
歷史最低價。代幣或資產曾達到的最低價格。
</Accordion>

### C

<Accordion title="CEX (Centralized Exchange)">
中心化交易所。由中心化機構運營的加密貨幣交易平臺。

示例：Binance、Coinbase、OKX、Bybit
</Accordion>

<Accordion title="Collateral（抵押品）">
借貸協議中用於擔保借款的資產。抵押率通常需要超過 100%。
</Accordion>

### D

<Accordion title="DEX (Decentralized Exchange)">
去中心化交易所。無需中心化機構運營的交易平臺，透過智慧合約自動執行交易。

示例：Uniswap、SushiSwap、Raydium、Jupiter
</Accordion>

<Accordion title="dApp (Decentralized Application)">
去中心化應用。執行在區塊鏈上的應用程式。
</Accordion>

<Accordion title="DeFi (Decentralized Finance)">
去中心化金融。基於區塊鏈的金融服務和應用，包括交易、借貸、衍生品等。
</Accordion>

### F

<Accordion title="Farming / Yield Farming（流動性挖礦）">
透過向 DeFi 協議提供流動性來獲取代幣獎勵的行為。
</Accordion>

<Accordion title="FDV (Fully Diluted Valuation)">
完全稀釋估值。假設所有代幣都已發行的市場總價值。

計算公式：`FDV = 代币价格 × 总供应量`
</Accordion>

<Accordion title="Flash Loan（閃電貸）">
無需抵押的即時借貸，必須在同一區塊/交易內還款。常用於套利、清算等場景。
</Accordion>

### H

<Accordion title="Health Factor（健康因子）">
借貸協議中衡量頭寸安全性的指標。當健康因子 &lt; 1 時，頭寸可能被清算。
</Accordion>

### I

<Accordion title="Impermanent Loss（無常損失）">
流動性提供者因價格波動而相對於簡單持有資產產生的損失。

| 價格變化 | 無常損失 |
|---------|---------|
| ±25% | -0.6% |
| ±50% | -2.0% |
| ±100% | -5.7% |
</Accordion>

### L

<Accordion title="Liquidation（清算）">
當借貸頭寸的抵押率低於閾值時，協議自動出售抵押品償還債務的過程。
</Accordion>

<Accordion title="Liquidity（流動性）">
市場中可供交易的資產數量。高流動性意味著大額交易不會顯著影響價格。
</Accordion>

<Accordion title="Liquidity Pool（流動性池）">
DEX 中存放交易對資產的智慧合約，為交易提供流動性。
</Accordion>

<Accordion title="LP (Liquidity Provider)">
流動性提供者。向 DEX 池子提供資產的使用者，獲取交易手續費作為回報。
</Accordion>

<Accordion title="LP Token">
流動性提供者代幣。代表使用者在流動性池中份額的憑證，可用於贖回資產或質押挖礦。
</Accordion>

### M

<Accordion title="Market Cap（市值）">
代幣的市場總價值。

計算公式：`Market Cap = 代币价格 × 流通供应量`
</Accordion>

<Accordion title="MEV (Maximal Extractable Value)">
最大可提取價值。礦工或驗證者透過交易排序、插入或審查獲取的額外價值。

常見 MEV 型別：
- Sandwich Attack（三明治攻擊）
- Front-running（搶跑）
- Back-running（尾隨）
- Liquidation（清算）
</Accordion>

### O

<Accordion title="Oracle（預言機）">
為智慧合約提供鏈外資料的服務，如 Chainlink、Pyth。常用於獲取價格資料。
</Accordion>

### S

<Accordion title="Slippage（滑點）">
交易預期價格與實際成交價格的差異。在流動性較低的池子中，滑點可能較大。
</Accordion>

<Accordion title="Staking（質押）">
將代幣鎖定在協議中以獲取獎勵或參與治理。
</Accordion>

<Accordion title="Swap（交換）">
在 DEX 上進行的代幣兌換操作。
</Accordion>

### T

<Accordion title="TVL (Total Value Locked)">
總鎖倉價值。鎖定在 DeFi 協議中的資產總價值，是衡量協議規模的重要指標。
</Accordion>

### W

<Accordion title="Whale（巨鯨）">
持有大量加密資產的地址或投資者。通常指持有某代幣 1% 以上供應量的地址。
</Accordion>

### Y

<Accordion title="Yield（收益）">
投資 DeFi 協議獲得的回報，包括交易費分成、代幣獎勵、利息等。
</Accordion>

## ChainStream 專有術語

ChainStream 產品和服務相關概念。

### A

<Accordion title="Access Token（訪問令牌）">
用於驗證 API 請求的 JWT（JSON Web Token）令牌。

**獲取流程**：
1. 在 Dashboard 建立 App，獲得 Client ID 和 Client Secret
2. 使用 Client ID 和 Client Secret 呼叫認證服務生成 Access Token
3. 在 API 請求頭中使用 `Authorization: Bearer <Access Token>`

**認證域名**：`dex.asia.auth.chainstream.io`

**Scope（許可權範圍）**：
| Scope | 說明 |
|-------|------|
| `webhook.read` | Webhook API 讀取許可權 |
| `webhook.write` | Webhook API 寫入許可權 |
| `kyt.read` | KYT API 讀取許可權 |
| `kyt.write` | KYT API 寫入許可權 |

安全提示：
- 不要在客戶端程式碼中暴露 Client Secret
- 不要將憑據提交到版本控制
- 使用環境變數儲存敏感資訊
- Access Token 有過期時間，需定期重新整理

參見 [認證文件](/zh-Hant/api-reference/authentication/authenticate)
</Accordion>

### C

<Accordion title="CU (Compute Unit)">
計算單元。ChainStream 用於衡量 API 請求複雜度的計量單位，不同端點消耗不同 CU。
</Accordion>

### K

<Accordion title="KYA (Know Your Address)">
瞭解你的地址。ChainStream 提供的地址風險評估服務，分析地址的歷史行為和關聯風險。
</Accordion>

<Accordion title="KYT (Know Your Transaction)">
瞭解你的交易。ChainStream 提供的交易風控服務，評估交易的資金來源和風險等級。

風險等級：

| 等級 | 說明 |
|------|------|
| Low | 低風險，可自動處理 |
| Medium | 中等風險，建議人工稽核 |
| High | 高風險，建議攔截 |
| Severe | 嚴重風險，必須攔截 |
</Accordion>

### R

<Accordion title="Rate Limit（速率限制）">
API 對請求頻率的限制。ChainStream 根據套餐提供不同的速率限制。

響應頭：
- `X-RateLimit-Limit` - 限制總量
- `X-RateLimit-Remaining` - 剩餘量
- `X-RateLimit-Reset` - 重置時間
</Accordion>

### S

<Accordion title="Smart Money">
聰明錢。ChainStream 識別的在市場中持續獲得超額收益的地址。

型別：
- Top Traders（頂級交易者）
- Early Investors（早期投資者）
- Institutional Wallets（機構錢包）
- KOL Wallets（KOL 錢包）

參見 [Smart Money 方法論](/zh-Hant/guides/data-concepts/smart-money-methodology)
</Accordion>

<Accordion title="Smart Score">
智慧評分。ChainStream 對錢包地址的綜合評分，反映其交易能力和歷史表現。

| 分數區間 | 等級 |
|---------|------|
| 90-100 | 頂級 Smart Money |
| 75-89 | 優質交易者 |
| 60-74 | 良好交易者 |
| &lt; 60 | 普通交易者 |
</Accordion>

### W

<Accordion title="Webhook">
網路鉤子。事件觸發時向指定 URL 傳送通知的機制。

**支援的事件型別**：
| 事件 | 說明 |
|------|------|
| `sol.token.created` | Solana 鏈上新代幣建立 |
| `sol.token.migrated` | Solana 鏈上代幣畢業（從 Bonding Curve 遷移到 DEX） |

<Note>
更多事件型別正在開發中。
</Note>

**管理端點**：
- `POST /v1/webhook/endpoint` - 建立 Webhook
- `GET /v1/webhook/endpoint` - 列出 Webhook
- `DELETE /v1/webhook/endpoint/{id}` - 刪除 Webhook

參見 [Webhook 基礎](/zh-Hant/playbooks/frameworks/webhook-fundamentals)
</Accordion>

<Accordion title="WebSocket">
一種網路協議，支援實時雙向通訊。ChainStream 透過 WebSocket 提供實時資料流。

**端點**：`wss://realtime-dex.chainstream.io/connection/websocket`

**連線方式**：
```
wss://realtime-dex.chainstream.io/connection/websocket?token=YOUR_ACCESS_TOKEN
```

**支援的訂閱型別**：
- 代幣 K 線（`dex-token-candles`）
- 代幣統計（`dex-token-stats`）
- 代幣持有者（`token-holders`）
- 新代幣（`dex-new-token`）
- 錢包餘額（`wallet-balance`）
- 錢包交易（`wallet-trade`）

<Note>
使用 SDK 時無需手動處理認證，SDK 已內建支援。
</Note>

參見 [WebSocket API](/zh-Hant/api-reference/endpoint/websocket/introduction)
</Accordion>

## AI 術語

AI 和 MCP 相關概念。

### A

<Accordion title="Agent（代理）">
能夠自主執行任務的 AI 系統。AI Agent 可以使用工具、訪問外部資料、做出決策。
</Accordion>

<Accordion title="Agent Skills（代理技能）">
結構化的指令包（`SKILL.md` 檔案），用於教 AI 程式設計助手如何使用特定能力。ChainStream 提供兩個 Skill：`chainstream-data`（只讀資料查詢）和 `chainstream-defi`（DeFi 執行）。Skills 包含決策樹、工作流、安全規則和錯誤恢復 — AI Agent 自主執行所需的一切。

參見 [Agent Skills 介紹](/zh-Hant/guides/ai-infrastructure/agent-skills/introduction)
</Accordion>

### C

<Accordion title="Context（上下文）">
AI 模型在生成回答時參考的資訊，包括對話歷史、系統提示、外部資料等。
</Accordion>

<Accordion title="Context Window（上下文視窗）">
AI 模型一次能處理的最大 Token 數量。更大的上下文視窗允許處理更多資訊。
</Accordion>

### L

<Accordion title="LLM (Large Language Model)">
大語言模型。基於大量文字資料訓練的 AI 模型，如 Claude、GPT 等。
</Accordion>

### M

<Accordion title="MCP (Model Context Protocol)">
模型上下文協議。Anthropic 開發的開放協議，允許 AI 模型與外部資料來源和工具互動。

ChainStream MCP Server 實現了該協議，讓 AI 能夠直接查詢鏈上資料。

參見 [MCP 介紹](/zh-Hant/guides/ai-infrastructure/mcp-server/introduction)
</Accordion>

<Accordion title="MCP Server">
MCP 服務端。實現 MCP 協議、提供工具和資料的服務。ChainStream MCP Server 提供鏈上資料查詢能力。
</Accordion>

<Accordion title="MCP Client">
MCP 客戶端。使用 MCP 協議呼叫工具的應用，如 Claude Desktop、Cursor。
</Accordion>

<Accordion title="MPP（微支付協議）">
使用 Tempo 網路（鏈 ID 4217）進行 USDC.e 支付的協議。是 x402 的替代方案，適合偏好在 Tempo 上付款的 AI Agent。Gas 直接以 USDC.e 支付，不需要 ETH。

參見 [x402 支付](/zh-Hant/guides/cli/x402-payment)
</Accordion>

### P

<Accordion title="Prompt（提示詞）">
給 AI 模型的輸入文字，引導模型生成期望的輸出。
</Accordion>

### T

<Accordion title="SKILL.md">
Agent Skills 的標準檔案格式。包含 YAML 頭部（name、description）和結構化 Markdown，含決策樹、工作流、安全規則和參考文件。被 Cursor、Claude Code、Codex 等 AI 程式設計助手使用。
</Accordion>

<Accordion title="TEE (Trusted Execution Environment)">
可信執行環境。提供硬體級別安全保護的計算環境，確保程式碼和資料的機密性和完整性。ChainStream CLI 使用 TEE 進行錢包金鑰管理 — 私鑰不會離開安全飛地。

參見 [CLI 認證](/zh-Hant/guides/cli/authentication)
</Accordion>

<Accordion title="Token（詞元）">
AI 模型處理文字的基本單位。一個 Token 約等於 4 個英文字元或 1-2 箇中文字元。

<Note>
此處 Token 指 AI 領域的詞元，與區塊鏈代幣是不同概念。
</Note>
</Accordion>

<Accordion title="Tool（工具）">
MCP 中定義的可被 AI 呼叫的功能。ChainStream 提供的工具包括 `get_token_balance`、`analyze_wallet` 等。
</Accordion>

### X

<Accordion title="x402">
由 [Coinbase](https://github.com/coinbase/x402) 推出的基於 HTTP 402（Payment Required）的開放支付協議。AI Agent 可透過 Base 網路上的 USDC 穩定幣按次付費呼叫 API。ChainStream 支援 x402 作為認證方式 — 無需 API Key 的 Agent 可透過簽名鏈上授權（EIP-3009 或 Permit2）為每次工具呼叫付費，由 Facilitator 服務驗證。無需註冊賬戶。
</Accordion>
